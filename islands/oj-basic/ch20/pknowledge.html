一、复习回顾：函数的调用流程

在学递归前，我们要记住：当一个函数被调用时，电脑会暂停手里的活，跑到那个函数里去执行，执行完了再回来。

🥞 函数调用栈：就像叠盘子

二、递归概念：函数自己调用自己？

递归 (Recursion) 是一种奇妙的魔法。它指的是：一个函数在它的内部，又调用了它自己。

🪆 俄罗斯套娃

在编程里，递归就是：
把大问题变成长得一样、但规模更小的问题。

比如：想知道 5!，就先问 4! 是多少... 直到问 1!。

三、递归出口：停止“套娃”的条件

如果函数一直调用自己，电脑就会像原地转圈的陀螺，最后“头晕”崩溃（这叫栈溢出 RecursionError）。

🚨 警告：无限递归
如果不写出口，程序会一直运行直到报错！一定要告诉函数：“什么时候停下来！”
🧙‍♂️ 递归两要素：
  1. 递归出口 (Base Case)： 最小的问题，可以直接回答（比如 1! = 1）。
  2. 递归公式 (Recursive Step)： 如何把大问题变小（比如 n! = n * (n-1)!）。
def story():
    if 没人听了: return # 1. 出口：没人听就不讲了
    print("从前有座山...")
    story()             # 2. 公式：继续讲同样的故事

四、应用案例：斐波那契数列 (兔子的繁衍)

数列：1, 1, 2, 3, 5, 8, 13... （每个数是前两个数之和）。

🌳 递归树：计算 f(4)
def f(n):
    if n == 1 or n == 2: 
        return 1 # 出口
    return f(n - 1) + f(n - 2) # 公式

五、应用案例：阶乘宝箱

计算 n! (即 1×2×3×...×n)。我们可以理解为：n! = n × (n-1)!

fact(4)
= 4 ×
fact(3)
= 3 ×
fact(2)
...

一直拆到 fact(1) = 1，然后再带着结果一层层返回！

def fact(n):
    if n == 1:
        return 1 # 出口：1的阶乘就是1
    return n * fact(n - 1) # 公式：n乘以更小的阶乘

六、应用案例：汉诺塔 (Tower of Hanoi)

有三根柱子 (A, B, C)，要把 A 柱上的圆盘全部移到 C 柱，大盘不能压在小盘上。

三步走魔法：
1. 把 n-1 个盘子从 A 移到 B (借助 C)
2. 把最大的盘子从 A 移到 C
3. 把 n-1 个盘子从 B 移到 C (借助 A) 
def hanoi(n, A, B, C):
    if n == 1: 
        print(A, "->", C)
        return
    hanoi(n - 1, A, C, B) # 1. 移走上面的
    print(A, "->", C)     # 2. 移动底盘
    hanoi(n - 1, B, A, C) # 3. 移回上面的

一、复习回顾：变量存储的局限性

以前我们要存一个数字，会申请一个变量（箱子）。

但是，如果老师要存全班 50 位同学的成绩，难道要定义 a1, a2, a3 ... a50 这样 50 个变量吗？

痛点： 单个变量处理大量数据太麻烦，还容易丢！我们需要一个“大柜子”。

二、列表初识：神奇的伸缩储物柜

在 Python 中，这个大柜子叫做 列表 (List)。

Python 列表的特点：

  • 📦 有序集合： 数据排排坐，每个都有位置。
  • 🪄 动态伸缩： 不像 C++ 数组必须定死大小，Python 列表可以随时变长变短，像魔法一样！
  • 🔢 编号 (下标)： 每个格子都有一个编号，方便查找。
🧙‍♂️ 老师的比喻：
C++ 的数组像学校固定的**储物柜**（大小固定）。
Python 的列表像一列**火车**，随时可以加挂新的车厢（添加元素）！

四、列表访问：下标从0开始的秘密

要拿到列表里的东西，必须知道它的下标 (Index)。

最重要的规则：计算机的数数是从 0 开始的！

🚨 红色警报：IndexError: list index out of range
如果你定义了 a = [10, 20, 30]，那么合法的下标是 0, 1, 2。
如果你试图访问 a[3]，Python 会直接报错，告诉你“下标越界”了！ 
scores = [90, 80, 70]
print(scores[0])  # 输出 90
scores[1] = 85    # 修改第二个格子的值
print(scores)     # 输出 [90, 85, 70]

五、列表遍历：体育课的点名时间

列表最强大的地方在于，它可以配合 for 循环，一次性处理几百上千个数据。

遍历 (Traversal)：就是把列表里的每个元素都访问一遍。

方式一：直接遍历 (推荐✨)

Python 特有的魔法！不用管下标，直接拿东西。

for score in scores:
    print(score)

就像老师直接喊名字：“张三、李四、王五”。

方式二：下标遍历 (传统)

配合 len() 和 range() 使用。

for i in range(len(scores)):
    print(scores[i])

就像老师喊学号：“1号、2号、3号”。

# 实战：读入 n 个数，存入列表
n = int(input())
nums = []

for i in range(n):
    val = int(input())
    nums.append(val) # 一个个加进去

print(nums)

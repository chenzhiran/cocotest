一、复习回顾：变量存储的局限性

以前我们要存一个数字，会申请一个变量（箱子）。

但是，如果老师要存全班 50 位同学的成绩，难道要定义 a1, a2, a3 ... a50 这样 50 个变量吗？

痛点： 单个变量处理大量数据太麻烦，还容易丢！我们需要一个“大柜子”。

二、数组初识：一排贴了编号的储物箱

数组 (Array) 就是：

  • 📦 相同类型 数据的集合（比如全是整数，或者全是小数）。
  • 📏 在内存中是 连续存放 的（挨着住）。
  • 🔢 每个格子都有一个 编号 (下标)。
🧙‍♂️ 老师的比喻：
就像学校的快递柜。这是一个整体（柜子），里面有很多小格子，每个格子都有编号，专门用来存东西。

三、定义初始化：批量制造变量的魔法

怎么告诉计算机，我要申请一个有 100 个格子的柜子呢？

语法公式：
类型 数组名[大小]; 
int a[5];  // 申请一个能放5个整数的柜子，名字叫 a
int b[100]; // 申请一个能放100个整数的柜子，名字叫 b

如何往里面放东西？(初始化)

1. 完全初始化
int a[3] = {1, 2, 3};
1 2 3
2. 部分初始化
int a[5] = {6, 8};
6 8 0 0 0
3. 全部清零
int a[5] = {0};
0 0 0 0 0

重点： 如果只给前面几个格子赋值，剩下的格子会自动变成 0！这招常用来清空数组。

四、数组访问：下标从0开始的秘密

要拿到数组里的东西，必须知道它的下标 (Index)。

最重要的规则：C++ 的下标是从 0 开始的！

🚨 红色警报：数组越界 (Out of Bounds)
如果你定义了 int a[5]，那么合法的下标是 0, 1, 2, 3, 4。
如果你试图访问 a[5] 或者 a[100]，程序可能会崩溃或者拿到垃圾数据！千万小心！ 
int a[5];
a[0] = 10;  // 把10放入第1个格子
a[1] = 20;  // 把20放入第2个格子
cin >> a[2]; // 输入一个数放入第3个格子
cout << a[0] + a[1]; // 计算并输出

五、数组遍历：体育课的点名时间

数组最强大的地方在于，它可以配合 for 循环，一次性处理几百上千个数据。

遍历 (Traversal)：就是把数组里的每个元素都访问一遍。

模式： 下标 i 从 0 走到 n-1。 
int n;
int a[105]; // 开大一点，防止不够用

cin >> n; // 1. 先问有多少个数

// 2. 循环输入
for (int i = 0; i < n; i++) {
    cin >> a[i]; 
}

// 3. 循环输出 (或者处理)
for (int i = 0; i < n; i++) {
    cout << a[i] << " ";
}

为什么 i < n 而不是 i <= n？

因为下标是从 0 开始的！如果有 n 个数，最后一个下标是 n-1。比如 5 个数，下标是 0, 1, 2, 3, 4。

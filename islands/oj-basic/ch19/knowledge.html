一、复习回顾：结构化程序的思想

到目前为止，我们写的代码都堆在 main 函数里。这就像把家里所有的东西（衣服、碗筷、书本）都堆在客厅里，太乱了！

结构化思想就是：把大问题拆成小问题，把大任务分给不同的人去做。

二、函数定义：像积木一样搭建程序

函数 (Function) 就是一段有名字的代码块，专门用来完成特定的任务（比如判断素数、求最大值）。

// 定义一个打招呼的函数
void sayHello() {
    cout << "Hello, World!" << endl;
}

int main() {
    sayHello(); // 呼叫函数干活
    return 0;
}
🧙‍♂️ 老师提示：
如果函数不需要返回任何结果（比如只是打印一句话），返回值类型写 void（空）。

三、参数传递：形参与实参的替身游戏

当我们调用函数时，经常需要给它一些数据（原材料）。这里有一个非常重要的概念：值传递。

main()
x = 5

实参 (本体)

➡️ 复制 ➡️
func(int a)
a = 5

形参 (替身)

如果在函数里把 a 变成了 100，main 里的 x 不会变！
因为你打的是“影分身”，本体毫发无伤。 

// 这里的 a, b 是形参 (形式参数)
int add(int a, int b) {
    return a + b;
}

int main() {
    int x = 10, y = 20;
    // 这里的 x, y 是实参 (实际参数)
    int sum = add(x, y); 
}

四、返回值：任务完成的报告

函数做完事情后，通常需要把结果告诉 main 函数。这就是 return 的作用。

注意： 一旦执行了 return 语句，函数就会立刻结束，后面的代码都不会执行了！

五、作用域：局部变量与全局变量的地盘

变量是在哪里定义的，它的“地盘”（作用域）就在哪里。

🌍 全局作用域 (客厅)

int global_x = 100; (大家都能用)

🏠 函数 func() (卧室)

int local_a = 5; (只有 func 能用)

可以使用 global_x ✅

🏠 函数 main() (书房)

int local_b = 10; (只有 main 能用)

可以使用 global_x ✅

可以使用 local_a ❌

🧙‍♂️ 老师建议：
尽量少用全局变量（客厅别乱放东西），多用局部变量和参数传递，这样程序更安全、更整洁。

一、复习回顾：结构化程序的思想

到目前为止，我们写的代码都堆在主程序里。这就像把家里所有的东西（衣服、碗筷、书本）都堆在客厅里，太乱了！

结构化思想就是：把大问题拆成小问题，把大任务分给不同的人（函数）去做。

二、函数定义：def 的魔法

在 Python 中，我们用 def (define 的缩写) 来创造一个新函数。

关键点： Python 没有花括号 {}，它靠缩进（空格）来知道哪些代码属于这个函数。

# 定义一个打招呼的函数
def say_hello():
    print("Hello, Python World!")

# 主程序调用
say_hello() # 必须加括号！

三、参数传递：给函数喂原料

函数就像一个果汁机，你给它水果（参数），它才能工作。

🍎

实参 (Main里的值)

➡️ 传递 ➡️
def func(x):

形参 (函数里的代号)

# 定义带参数的函数
def greet(name):
    print("你好, " + name)

# 调用时传入实际的值
greet("Alice") 
greet("Bob")

四、返回值：工厂产出成品

函数做完事情后，如果需要把结果告诉主程序，就要用 return。

def add(a, b):
    result = a + b
    return result # 把结果扔回去

total = add(5, 10) # 接住结果
print(total)

五、作用域：局部与全局的地盘

变量是在哪里定义的，它的“地盘”（作用域）就在哪里。

  • 局部变量： 在函数里面定义的，只有函数自己能用（像卧室里的牙刷）。
  • 全局变量： 在函数外面定义的，大家都能看（像客厅的电视）。
🚨 Python 特别规定：
如果你想在函数里修改全局变量，必须先用 global 关键字声明！否则 Python 会以为你在创建一个同名的新变量。
🌍 全局作用域 (客厅)

score = 0

🏠 函数 add_score()

(我想改客厅的分数...)

global score 🔑 拿到钥匙！

score += 10 ✅ 成功修改

score = 0

def win_game():
    global score # 我要改外面的那个score！
    score += 1
    print("赢了！当前分数：", score)

win_game()

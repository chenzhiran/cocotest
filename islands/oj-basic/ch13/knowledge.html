一、复习回顾：数组查找与最值

在之前的课程中，我们学会了如何在数组中找到特定的数（线性查找），以及如何找到最大的数（擂台法）。

关键点： 遍历数组时，下标范围是 0 到 n-1。越界是最大的敌人！

二、元素移动：乾坤大挪移 (插入与删除)

数组里的元素是“挨着住”的。如果你想在中间插一个人，或者把一个人拉出来，后面的人都需要动一动！

1. 插入元素 (Insert)

场景： 在排好的队伍中，让新同学插进第 k 个位置。

操作： 从最后一名开始，依次往后退一步，给新同学腾出空位。

// 在第 k 个位置插入 value (假设数组够大)
// 必须从后往前搬运，否则会覆盖数据！
for (int i = n; i > k; i--) {
    a[i] = a[i-1];
}
a[k] = value; // 填入新值
n++;          // 总人数+1

2. 删除元素 (Delete)

场景： 第 k 个同学离开了。

操作： 后面的同学依次往前补位，把空缺填满。

// 删除第 k 个位置的元素
// 从前往后搬运
for (int i = k; i < n - 1; i++) {
    a[i] = a[i+1];
}
n--; // 总人数-1

三、桶排序思想：统计数字出现的次数

如果我们要统计全班同学每个人投了多少票（编号 1-10），怎么做最快？

我们可以准备 10 个贴了标签的桶，拿到一张票（比如 3 号），就往 3 号桶里丢一颗石子（计数+1）。

💡 核心思想：
利用数组的下标来代表具体的数字，数组的值代表出现的次数。
count[x]++ => 数字 x 出现了一次。 
int counts[11] = {0}; // 准备10个桶，初始化为0
int x;
for(int i=0; i<n; i++) {
    cin >> x;
    counts[x]++; // 关键：对应编号的桶，计数+1
}

四、开关灯问题：模拟状态的批量切换

有 n 盏灯，开始都是关着的。我们要进行多次操作，每次按下某些灯的开关（开变关，关变开）。

我们可以用数组 a[i] 来表示第 i 盏灯的状态：

  • 0：关灯 ⚫
  • 1：开灯 🟡

0 (Off)

➡️ 按开关 ➡️

1 (On)

// 切换状态的魔法公式
// 方法1：用 if
if (light[i] == 0) light[i] = 1;
else light[i] = 0;

// 方法2：数学魔法 (1变0, 0变1)
light[i] = 1 - light[i]; 

// 方法3：逻辑非 (适合bool类型)
light[i] = !light[i];

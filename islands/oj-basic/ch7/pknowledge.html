一、复习回顾：If-Else 家族

我们已经学习了强大的 if-elif-else，它就像一个万能的岔路口。但在某些简单的情况下，写好几行代码会显得有点啰嗦。今天我们来认识两位新朋友，帮我们把代码变简洁！

二、三目运算符：一行代码的艺术

有时候，if-else 只是为了给一个变量赋不同的值。Python 有一种独特的“倒装句”写法，读起来就像英语句子一样自然。

🪄 魔法对比：求两个数的最大值

👴 传统 If-Else 写法：

if a > b:
    max_val = a
else:
    max_val = b

✨ 一行魔法写法：

max_val = a if a > b else b
# 翻译：如果a大于b，就是a，否则就是b

三、Match-Case：Python的“开关箱” (3.10+)

如果你面对的是一个自动售货机：按下 1 出可乐，按下 2 出雪碧……这种针对具体数值的选择，在 Python 3.10 版本之后，我们终于有了 match-case 语句！（以前只能用很多 if-elif）

Match (匹配)
🔑 key
Case 1: 🥤 可乐
Case 2: 🥤 雪碧
Case _: 💧 矿泉水
choice = int(input())

match choice:
    case 1:
        print("可乐")
    case 2:
        print("雪碧")
    case 3:
        print("果汁")
    case _:
        print("没货了，给你水") # _ 代表其他所有情况(Default)
🧙‍♂️ 老师小贴士：
match-case 是 Python 的新魔法。如果你的 Python 版本比较老（比如在某些旧的 OJ 网站上），可能不支持哦！那时我们还是用 if-elif-else 吧。

四、代码优化：让程序更优雅

写代码不仅是给机器看的，更是给人看的。Python 被称为“最优雅的语言”，我们要学会写得漂亮。

✨ 优雅指南：
  • 利用布尔值的直接判断： 不要写 if flag == True:，直接写 if flag:。
  • 链式比较： Python 特有技能！可以写 0 < x < 10，不用写 0 < x and x < 10。
❌ 啰嗦的写法
if score >= 60 and score < 100:
    print("Pass")

if is_vip == True:
    print("Welcome")
✅ 优雅的写法
if 60 <= score < 100:
    print("Pass")

if is_vip:
    print("Welcome")

五、易错辨析：赋值(=)与相等(==)

在 Python 中，混淆 = 和 == 虽然不像 C++ 那样容易造成隐蔽的逻辑错误（Python 通常会直接报错），但依然是初学者的头号大敌。

🚨 警报：SyntaxError

如果你在 if 条件里写了 if a = 5:，Python 会非常生气地给你一个 SyntaxError（语法错误）。它在告诉你：“这里不能赋值！请用 == 来比较！”

一、复习回顾：If-else 与 Switch 的对比

我们已经学习了强大的 if-else，它就像一个万能的岔路口。但在某些情况下，它会显得有点啰嗦。今天我们来认识两位新朋友，帮我们把代码变简洁！

二、三目运算符：极致简约的一行代码 (?:)

有时候，if-else 只是为了给一个变量赋不同的值，写 4 行代码太累了！这时候，三目运算符 闪亮登场。

它是 C++ 中唯一需要三个参与者的运算符，像一个简易的天平。

🪄 魔法对比：求两个数的最大值

👴 传统 If-Else 写法：

if (a > b) {
    max = a;
} else {
    max = b;
}

✨ 三目运算符写法：

max = (a > b) ? a : b;
// 翻译：a大于b吗？
// 是就给a，不是就给b

三、Switch语句：精准的开关控制箱

如果你面对的是一个自动售货机：按下 1 出可乐，按下 2 出雪碧，按下 3 出果汁……这种针对具体数值的选择，用 switch 更方便！

Switch (开关)
🔑 key
Case 1: 🥤 可乐
Case 2: 🥤 雪碧
Case 3: 🍹 果汁
Default: 💧 矿泉水
int choice;
cin >> choice;

switch (choice) {
    case 1:
        cout << "可乐";
        break; // 🛑 拿到可乐就停下，别往下走了！
    case 2:
        cout << "雪碧";
        break;
    case 3:
        cout << "果汁";
        break;
    default:
        cout << "没货了，给你水"; // 其他所有情况
}
⚠️ 致命陷阱：break (刹车)
如果不写 break，程序就会像刹车失灵一样，一直往下执行（穿透），直到结束！这叫“Case 穿透”。

四、代码优化：让程序更优雅、更易读

写代码不仅是给机器看的，更是给人看的。好的代码就像一篇优美的文章。

🧹 代码整洁之道：
  • 变量命名： 别老是用 a, b, c。用 score, age, sum 更有意义。
  • 缩进对齐： 就像排队一样，整齐的缩进能让人一眼看清层次。
  • 注释： 给关键代码写上解释，方便自己和别人阅读。
❌ 糟糕的代码
int a; cin>>a;
if(a>60) cout<<"ok";
else cout<<"no";
✅ 优雅的代码
int score;
cin >> score;
if (score > 60) {
    cout << "Pass";
} else {
    cout << "Fail";
}

五、易错辨析：赋值(=)与相等(==)的陷阱

这是 C++ 中最著名的“坑”，99% 的同学都掉进去过！

🚨 警报：

在 if 的括号里，如果你写了 if (a = 5)，C++ 不会报错，它会先把 5 给 a，然后认为这个条件是真的（因为 5 不是 0）。这会导致严重的逻辑错误！

永远记住：判断相等用两个等号 ==！

3.1 快速排序

Quick Sort：分治法的巅峰之作 ⚡️

🏃‍♂️ 体育老师的分队魔法

假设全班同学排成一排。体育老师想按个子高矮排队，但他不想一个一个比。他用了一个绝招：

老师随便拉出一个同学（比如小明）站在中间，大喊一声：

“比小明矮的站左边！比小明高的站右边！”

哗啦一下，队伍分成了两波。虽然左边那波内部还没排好，但它们肯定都比右边那波矮！接着，老师对左边和右边的小队重复同样的命令……这就是分治法！

🎨 详细图解：哨兵出动

我们把被选中的那个数叫做“基准数” (Pivot)。通常我们选最左边的数。

我们要排序：[ 6, 1, 2, 7, 9, 3, 4, 5, 10, 8 ]

Step 1: 选基准 6
Step 2: 哨兵出动

规则：右哨兵 j 先走，找比 6 小的；左哨兵 i 后走，找比 6 大的。找到了就交换！

Step 3: 基准归位

哨兵们继续走、继续换，直到它们碰头。碰头的位置，就是基准数 6 应该待的地方！

然后，再分别对“绿色队”和“蓝色队”做同样的事情（递归）。

💻 编程赛道

注意： 代码中为了方便，我们通常直接选中间的数作为基准（mid），这样写代码更不容易出错，而且速度更稳！

#include <iostream> using namespace std; int a[1005]; // 快速排序函数：处理从 l 到 r 的范围 void quick_sort(int l, int r) { if (l >= r) return; // 只有一个数或没有数，不用排 // 1. 选基准 (这里选中间的数，防止最坏情况) int pivot = a[(l + r) / 2]; int i = l, j = r; // 2. 哨兵出发 while (i <= j) { // 左边找比基准大的 while (a[i] < pivot) i++; // 右边找比基准小的 while (a[j] > pivot) j--; // 找到了就交换 if (i <= j) { swap(a[i], a[j]); i++; j--; } } // 3. 递归：现在分成了两队，继续排 quick_sort(l, j); // 排左边那队 (l ~ j) quick_sort(i, r); // 排右边那队 (i ~ r) } int main() { int n; cin >> n; for (int i = 0; i < n; i++) cin >> a[i]; quick_sort(0, n - 1); for (int i = 0; i < n; i++) cout << a[i] << " "; return 0; }
def quick_sort(arr, l, r): if l >= r: return # 递归出口 # 1. 选基准 pivot = arr[(l + r) // 2] i, j = l, r # 2. 哨兵划分 while i <= j: # 左找大 while arr[i] < pivot: i += 1 # 右找小 while arr[j] > pivot: j -= 1 # 交换 if i <= j: arr[i], arr[j] = arr[j], arr[i] i += 1 j -= 1 # 3. 递归处理左右两边 quick_sort(arr, l, j) quick_sort(arr, i, r) # 测试 data = [6, 1, 2, 7, 9, 3, 4, 5, 10, 8] quick_sort(data, 0, len(data) - 1) print(data)

🏆 为什么它是最快的？

🚀 平均速度 O(nlogn)

因为它每次都把队伍切成两半，就像折纸一样，折的次数很少。在绝大多数情况下，它都是比赛的冠军！

⚠️ 倒霉的时候 O(n²)

如果每次选的基准数都是最小的（运气极差），它就退化成冒泡排序了。所以我们通常选中间的数当基准来避免这种情况。

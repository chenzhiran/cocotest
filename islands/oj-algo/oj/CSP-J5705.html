📜 任务清单 (题目描述)

在一个果园里，多多已经摘下了所有果子，并根据果子的不同种类分成了不同的果堆。

现在，多多想要将所有果子合并成一大堆。每次合并两堆果子都会消耗一些体力，消耗的体力等于两堆果子的重量之和。

最终，当所有果堆合并成一堆时，合并过程中所有的体力消耗之和就是多多的总体力消耗。请你计算出多多需要消耗的最小体力值。

果园输入与输出规则：

  • 第一行是一个整数 n，表示果子的种类数量（也就是初始状态下果堆的数量）。
  • 第二行包含 n 个正整数，用空格分隔，第 i 个整数代表第 i 堆果子的数目。
  • 输出一个整数，表示将所有果堆完全合并成一堆时，所能达到的最小总体力消耗。
📥 魔法输入阵：
(3堆果子各自的数量)
3
1 2 9
📤 奇迹输出阵：
(花费的最小体力消耗之和)
15

数据规模限制：

  • 对于 100% 的数据，保证有 1 ≤ n ≤ 10,000
  • 每种果子的初始数量满足 1 ≤ ai ≤ 20,000
  • 最终生成的最小体力消耗值小于 2^31（即在标准 32 位有符号整型范围内）

🧠 魔法道具箱 (核心图解：精打细算的贪心法则)

想要总体力最省，核心秘诀就是：永远挑当前全场重量最小的两堆果子进行合并！

为什么？因为越早被合并的果堆，在接下来的后续合并中就会被“重复计算”越多次。让重量越小的果堆多搬几次，重量最大的果堆少搬几次，整体消耗自然就降到了最低！

🗺️ 样例贪心合并动态模拟（输入：1 2 9） 🗺️
🟢 初始果园状态： 有三堆果子
1 (最小)
2 (次小)
9
🔮 第一轮合并： 选出最小的 1 和 2 合并。
（本轮搬运消耗体力：1 + 2 = 3 ； 累计总体力：3）
3 (新诞生且当前最小)
9 (当前次小)
🔮 第二轮合并： 选出剩下的 3 和 9 合并。
（本轮搬运消耗体力：3 + 9 = 12 ； 累计总体力：3 + 12 = 15）
最终大果堆：12

* 成果清点：将每一轮合并时新诞生的果堆重量全部加起来：3 + 12 = 15，这就是完美契合贪心法则的最小体力答案！

✨ 最小堆数据结构的妙用

由于合并过程中会不断诞生新的果堆，我们需要能动态地、极快地找出当前最小的两堆。优先队列（最小堆 / Min-Heap）就是最完美的魔法工具：

  1. 🔸 果子入堆： 把所有初始果堆的重量通通塞进最小堆中。
  2. 🔸 成双取出： 只要堆内的果堆数量不小于 2，就连续取出堆顶的两个最小元素，分别设为 a 和 b。
  3. 🔸 合二为一： 产生新消耗 a + b，将其加到最终体力答案中，并把这个新堆 a + b 重新放回最小堆。
  4. 🔸 完成搬运： 重复上述步骤，直到堆里只剩下一堆果子，此时输出累计的总体力值即可。
策略公式：每次提取最小的两个元素 → 累加合并权值 → 重新插入堆中 → 直至规模化为一

📘 C++ 魔法秘籍 (知识点)

  • 小顶堆的建立： C++ 中的 priority_queue 默认是大顶堆。为了将其改造为最小堆，我们需要显式声明三个参数：priority_queue<int, vector<int>, greater<int>>。
  • 自动排序维护： 每次使用 q.push() 把合并后的新堆丢进队列时，堆内部会以极高的对数时间复杂度自动调整，确保堆顶永远是全场最小值。
  • 标准弹堆三步走： 通过 q.top() 读取堆顶，再用 q.pop() 将其移除，逻辑条理清晰，执行非常稳健。

💻 C++ 终极法术 (标准小顶堆模拟)

#include <iostream>
#include <queue>
#include <vector>

using namespace std;

// 全局变量定义：n表示种类数，x表示每次输入的重量，ans存储累计消耗的体力
int n, x, ans; 

// 🌟 核心道具：定义一个最小堆（小顶堆），用于高效动态存取每堆果子的数量
priority_queue<int, vector<int>, greater<int> > q; 

int main() {
    // ⚡ 提速法阵：解除输入输出流的绑定，加快读取速度
    ios::sync_with_stdio(false);
    cin.tie(NULL);

    cin >> n; // 输入果子的种类数
    for (int i = 1; i <= n; i++) {
        cin >> x;   // 输入每种果子的初始数量
        q.push(x);  // 将其加入最小堆，队列内部会自动按照从小到大排序
    }

    // 🔄 只要最小堆中有两个或以上的果堆，就持续进行贪心合并
    while (q.size() >= 2) {
        // 🔮 精准拔筹：取出当前全场重量最小的两堆果子
        int a = q.top(); q.pop(); // 提取并弹出最小的果堆
        int b = q.top(); q.pop(); // 提取并弹出第二小的果堆
        
        ans += a + b;   // 💥 多多付出了 a + b 的体力，将其累加进总消耗中
        q.push(a + b);  // 将合并后产生的新果堆重新放回最小堆中
    }

    cout << ans << endl; // 🏆 输出最终所需的最小总体力消耗
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • 原生库 heapq 模块： Python 为我们直接提供了高效的堆算法操作库 heapq。区别于 C++ 的复杂声明，Python 中的 heapq 默认就是纯正的最小堆结构。
  • 堆化法术 heapify： 可以直接把普通的乱序列表传入 heapq.heapify(fruits_list)，它可以在极其惊人的 O(n) 线性时间内把列表在原地直接改造成合规的最小堆。
  • 高效弹出与推入： 用 heapq.heappop(heap) 获取并移除最小值，用 heapq.heappush(heap, item) 推入新元素，完美契合算法思路。

💻 Python 终极法术 (heapq内置最小堆版)

import sys
import heapq

def main():
    # 📥 极速块读：一次性把文件里所有的魔法数字全部吸入内存切片中
    input_data = sys.stdin.read().split()
    if not input_data:
        return
        
    n = int(input_data[0])
    # 将接下来的 n 个果堆数字提取出来，并转换为整型列表
    fruits = [int(x) for x in input_data[1:n+1]]
    
    # ⚙️ 核心觉醒：在线性 O(n) 时间内直接把基础列表原地重构为标准的最小堆
    heapq.heapify(fruits)
    
    total_stamina = 0  # 🏆 总体力消耗计数器
    
    # 🔄 只要果园里还剩下至少两堆或以上的果子，就持续施展双堆合并
    while len(fruits) >= 2:
        # 🔮 顺藤摸瓜：弹出当前全场分量最轻的两堆果子
        a = heapq.heappop(fruits)
        b = heapq.heappop(fruits)
        
        current_merge_cost = a + b
        total_stamina += current_merge_cost  # 累加这一步合并时所付出的体力
        
        # 将合二为一的全新果堆重新放入最小堆中，供后续轮次筛选
        heapq.heappush(fruits, current_merge_cost)
        
    # 🏆 打印输出多多完全合并果堆后的最小体力开销
    print(total_stamina)

if __name__ == '__main__':
    main()

💡 技巧和重点详细总结 (算法官的结题心得)

  • 🌟 经典的哈夫曼思想模型： 这道经典问题在算法界有一个非常响亮的名字——哈夫曼树路径权重最小化问题。它与“哈夫曼编码问题”完全是孪生兄弟，核心思想都是通过每次合并两个最小项来构建一颗高度最优的二叉树。
  • 🌟 为什么普通的快排很难通关？： 如果我们每次合并完，都用普通的快速排序（sort）重新给数组排一次序，那么每一次都需要消耗 O(n log n) 的代价，多轮合并下来的整体复杂度会高达 O(n^2 log n)，面对 10,000 的顶格测试数据必然会超时被秒杀。而利用最小堆，每次弹入弹出的调整代价只有稳定的 O(log n)，效率极高！
  • 🌟 精彩绝伦的时间复杂度： 初始建立堆（或者 Python 堆化）开销控制在 O(n) 以内。随后总共进行 n - 1 轮贪心合并，每轮对堆进行弹出和插入操作的开销为 O(log n)。因此，算法整体的时间复杂度被完美定格在 O(n log n)。程序可在几毫秒内瞬间秒杀全部测试点，稳拿满分！

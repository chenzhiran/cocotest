📜 任务清单 (题目描述)

已知海岸线是一条无限长的直线（x 轴），大海位于 x 轴上方（y > 0），陆地位于下方。

海上有 n 个小岛，每个小岛坐标为 (x, y)。雷达只能安装在海岸线上，覆盖半径为 d 且呈圆形。

你的任务是计算出能够安全覆盖所有岛屿所需的最少雷达数量。如果无论怎么放都无法覆盖到某个孤立岛屿，则输出 -1 表示无解。

大殿多组测试阵法（输入与输出）：

  • 输入包含多组独立的数据。每组的第一行是两个整数：n（岛屿数）和 d（雷达覆盖半径）。
  • 接下来的 n 行，每行两个整数表示每个岛屿的坐标 (x_i, y_i)。
  • 相邻两组数据之间会用一个空行隔开。当读到 0 0 时，代表大阵闭合，输入结束。
  • 对于每一组数据，请严格按照格式输出：Case i: 结果。若无解，结果为 -1。

🧠 魔法道具箱 (核心图解：从二维圆到一维区间的降维法术)

初看这道题，雷达是个二维的圆形，看起来无从下手。但由于雷达的中心必须放在海岸线（x 轴）上，这就给了我们降维打击的机会！

📍 离海岸垂直距离为 y 的小岛
/ \ 斜边半径为 d，垂直边为 y
⚡ 雷达必须安放的 x 轴区间 [x - dis, x + dis] ⚡

根据直角三角形的勾股定理：如果想要覆盖这个小岛，雷达在 x 轴上的安放位置与小岛的横坐标 x 的距离，绝对不能超过 dis = sqrt(d * d - y * y)！

换句话说，为了能罩住这头小岛，我们必须在海岸线的 [x - dis, x + dis] 这个区间内放置至少一个雷达。这样，题目就变成了一个标准的用最少的点覆盖所有区间的经典贪心问题！

✨ 几何贪心四步破咒法

  1. 🔸 判绝境（无解检查）： 如果某个小岛的垂直高度 y 大于雷达半径 d，说明雷达的手太短了，无论放哪都够不着！直接判定此组为无解，输出 -1。
  2. 🔸 划区间（降维映射）： 利用勾股定理，计算每个小岛在 x 轴上映射出的合法放置区间 [左端点 l, 右端点 r]。
  3. 🔸 定乾坤（按右端点升序排序）： 把所有区间按照右端点（r）从小到大进行升序排列。
  4. 🔸 贪心扎针（选点覆盖）： 从左到右审阅区间：第一个雷达任性地放在第一个区间的右端点。接下来如果遇到某个区间的左端点大于当前雷达位置，说明当前雷达照顾不到了，必须在当前区间的右端点增设一个新雷达，以此类推。
核心策略：将雷达定在区间右端点的“危险悬崖边缘”，给后续区间留出最大的容纳可能

📘 C++ 魔法秘籍 (知识点)

  • 结构体数组建模： 使用 struct Point 分别用双精度浮点数 double 记录每个岛屿计算出来的左右边界，防止精度丢失。
  • 多组数据清空重置： 面对输入含有多组数据的题目，切记每一轮循环都要使用 memset 或初始化指令将全局数组和控制变量（如 flag、ans）重置，避免残留信息污染下一组结果。
  • 数学库函数 sqrt： 引入标准头文件 <cmath>（或万能头），利用内置开平方根函数进行精确计算。

💻 C++ 终极法术 (区间右端点递推版)

#include <iostream>
#include <vector>
#include <cmath>
#include <algorithm>
#include <cstring>
#include <cstdio>

using namespace std;

const int MAXN = 1010;

// 🔮 定义一个结构体存储每个岛屿投影到海岸线上的可覆盖区间
struct Point {
    double l, r; // 分别表示可覆盖的最左端点和最右端点
} a[MAXN];

// ⚖️ 比较函数：按照区间的右端点进行升序排序（从小到大）
bool cmp(Point aa, Point bb) {
    return aa.r < bb.r;
}

int main() {
    int n, d, t = 0; // n为岛屿数，d为雷达半径，t为测试案例编号计数器
    
    // 🔄 多组数据循环输入，直到读到 n = 0 且代表结束
    while (cin >> n && n != 0) {
        t++;
        int ans = 1;             // 初始化所需雷达数，默认至少需要 1 个
        cin >> d;                // 输入雷达覆盖半径
        memset(a, 0, sizeof(a)); // 🧼 处女座清空：重置每组对应的结构体空间
        int flag = 0;            // 标识是否遇到绝境（无解情况）
        
        for (int i = 1; i <= n; i++) {
            int x, y;
            scanf("%d%d", &x, &y); // 读入每个小岛的平面坐标
            
            // 🛑 细节死穴：如果岛屿垂直距离 y 大于雷达半径 d，绝对够不着！
            if (y > d) {
                cout << "Case " << t << ": -1" << endl;
                flag = 1;
                // 注意：即使无解也必须把当前组剩下的输入全部读完，所以不能用 return
            }
            
            if (flag == 0) {
                // 📐 勾股定理显神威：计算雷达在 x 轴上可覆盖的偏离距离
                double dis = sqrt(d * d - y * y);
                a[i].l = x - dis; // 可安放的左侧极限位置
                a[i].r = x + dis; // 可安放的右侧极限位置
            }
        }
        
        // 若在此组数据的读入中已发现无解情况，直接跳过计算阶段，开启下一组
        if (flag == 1) continue; 
        
        // ⏱️ 贪心核心：对所有有效的一维区间进行右端点升序快排
        sort(a + 1, a + n + 1, cmp);
        
        double now = a[1].r; // 🎯 贪心策略：将第一个雷达部署在第一个区间的最大右端点
        
        // 🔍 线性扫描剩下的所有区间
        for (int i = 2; i <= n; i++) {
            // 如果下一个区间的左起始点跑到了当前雷达的覆盖范围右边
            if (now < a[i].l) { 
                now = a[i].r; // 必须设立新雷达，且同样安放在当前区间的最大右端点
                ans++;        // 雷达数量加 1
            }
        }
        
        // 🏆 打印本轮测试的最终完美布局答案
        cout << "Case " << t << ": " << ans << endl;
    }
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • 多组未知空行的切割： 题目提到多组数据且其中可能有空行。在 Python 中，使用 sys.stdin.read().split() 是最优解。它会自动帮我们忽略一切空格、制表符和换行空行，把所有有意义的字符串按顺序做成一维列表，处理起来极其惬意。
  • 数学开根号 math.sqrt： 必须先引入 Python 自带的数学魔法库 import math，方可施展精准计算。
  • 元组列表的高效排序： 通过 intervals.sort(key=lambda x: x) 可以极为优雅、精简地实现按区间的第二个元素（右边界）做原地升序排列。

💻 Python 终极法术 (全自动快读切片版)

import sys
import math

def main():
    # 📥 降维吸入法：一口气把全场所有的数字信息统统切片吞入内存
    input_data = sys.stdin.read().split()
    if not input_data:
        return
        
    data_idx = 0
    case_num = 0
    
    # 🔄 当还有数据可以读取时，持续推进大循环
    while data_idx < len(input_data):
        n = int(input_data[data_idx])
        d = float(input_data[data_idx+1])
        data_idx += 2
        
        # 🏁 终焉信号：如果读到了 n = 0 且 d = 0，说明大阵到头，直接结束
        if n == 0 and d == 0:
            break
            
        case_num += 1
        intervals = []
        possible = True
        
        # 🔍 依次读取当前组的所有岛屿坐标
        for _ in range(n):
            x = int(input_data[data_idx])
            y = int(input_data[data_idx+1])
            data_idx += 2
            
            # 🛑 绝境死穴检查
            if y > d:
                possible = False
                # 注意：即便这里已经绝望，循环依然要继续把这组剩下的坐标读完，推进指针
                
            if possible:
                # 📐 几何降维：计算一维覆盖轴线半径
                dis = math.sqrt(d * d - y * y)
                intervals.append((x - dis, x + dis))
                
        # 🏆 如果已经判定失守，直接输出结果并进入下一组案例
        if not possible:
            print(f"Case {case_num}: -1")
            continue
            
        # ⏱️ 排序核心：利用匿名函数按照区间的右端点做升序排序
        intervals.sort(key=lambda x: x[1])
        
        ans = 1
        current_radar = intervals[0][1] # 针扎在第一个区间的最大右边界
        
        # 🔍 单向指针扫描剩下的全部区间
        for start, end in intervals[1:]:
            # 如果新保护区的左边界超过了当前雷达能够辐射的范围
            if start > current_radar:
                current_radar = end # 在新防线的右边界增设雷达阵眼
                ans += 1
                
        print(f"Case {case_num}: {ans}")

if __name__ == '__main__':
    main()

💡 技巧和重点详细总结 (调度法师的防御笔记)

  • 🌟 绝妙的转化思想： 这道题的最高级智慧在于将“点覆盖圆”巧妙转换为了“点覆盖一维区间”。在算法设计和高等竞赛中，面对高维（二维、三维）的复杂结构，如果能够通过题目中给出的某些特殊边界限制（例如本题雷达必须在 x 轴上），将其强行压缩降维到一维直线上解决，往往能瞬间瓦解难题。
  • 🌟 为什么必须按右端点升序排序？： 很多初学者会疑惑为什么不能按左端点排序。让我们用贪心逻辑来思考：按右端点从小到大排序，意味着我们优先解决当前最早结束、最迫切需要被覆盖的那个区间。为了让这个雷达不仅能罩住它，还能尽可能照顾到后面的区间，把它钉在它的右端点（极限边缘）是最明智的选择。如果按左端点排序，你就无法保证部署位置能让前面的迫切区间受惠，逻辑就会崩塌。
  • 🌟 时间复杂度的降维打击： 计算区间耗时 O(n)，最核心的花销在排序这一步，耗时 O(n log n)。最后的贪心选点单向扫描只需要 $O(n)$ 复杂度。由于题目给出的规模每组数据不超过 n = 1000，程序可以在几毫秒内以秋风扫落叶之势秒杀通关，稳拿满分！

📜 任务清单 (题目描述)

骑士团需要使用尽量少的闭区间 [a_i, b_i] 来完全覆盖目标线段 [s, t]。

请你计算并输出所需的最少区间数量。如果穷尽所有屏障也无法将其完全覆盖，则输出“无法完全覆盖线段。”。

大殿调度法则：

  • 输入的第一行包含一个整数 n，表示已知区间的数量。
  • 接下来的 n 行中，每行包含两个整数 a_i 和 b_i，表示一个闭区间的起始和结束点。
  • 最后一行包含两个整数 s 和 t，表示需要覆盖的目标线段。
📥 魔法输入阵：
(3个区间与目标 [2, 6])
3
1 4
3 6
5 7
2 6
📤 奇迹输出阵：
(最少需要的区间数)
2

数据规模限制：

  • 1 ≤ n ≤ 100,000
  • -1,000,000 ≤ a_i, b_i, s, t ≤ 1,000,000

🧠 魔法道具箱 (核心图解：起点过界，终点最远)

面对“完全覆盖”的任务，直觉告诉我们：每次挑选屏障时，它的起点必须能够接上当前已经覆盖的终点，且在此基础上，它的终点必须延伸得越远越好！

这就是经典的区间覆盖贪心策略。

目标防线
屏障1 [1, 4]
第1步：选它！覆盖到4
屏障2 [3, 6]
第2步：衔接成功！完美到6

* 方案解析：第一步在所有包含起点 2 的区间里，[1, 4] 能够延伸到 4（最远）；当前覆盖终点变为 4。第二步寻找起点小于等于 4 的所有区间中终点最远的，[3, 6] 能延伸到 6。成功抵达目标终点 6，总共使用 2 个区间！

✨ 贪心覆盖四步决胜法

  1. 🔸 边界裁剪（预处理）： 剔除所有完全在目标线段 [s, t] 之外的无效区间。对于有重叠的区间，将其左右边界裁剪到 [s, t] 范围内。
  2. 🔸 按起点升序排序： 将所有区间按照左端点（起始点）从小到大排序，方便我们按从左到右的顺序依次审阅。
  3. 🔸 双重循环探最远： 外层控制当前已覆盖到的终点位置 current_end（初始为 s）。内层循环遍历所有左端点小于等于 current_end 的区间，在它们之中找出右端点最远的一个，记为 furthest_end。
  4. 🔸 步步推进防断层： 如果最远能到的 furthest_end 无法超越 current_end，说明中间有断层，无法完全覆盖，直接宣判无解！否则，把 current_end 更新为 furthest_end，计数器加 1。直到 current_end 达到或超过 t。
贪心核心：在能衔接的区间中，永远选择能伸展到最远那一端的“最强屏障”

📘 C++ 魔法秘籍 (知识点)

  • 结构体打包： 利用 struct Interval 自定义包含 start 和 end 的复合数据类型。
  • 区间预处理裁剪： 在排序前利用 max(interval.start, s) 和 min(interval.end, t) 进行裁剪，大幅度减少后续复杂干扰，是防止溢出的绝妙技巧。
  • 单向推进指针： 使用 size_t i 记录遍历位置。由于区间已经过排序，i 指针在整个算法中绝不回头，确保了极高的运行性能。

💻 C++ 终极法术 (区间覆盖贪心版)

#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

// 🔮 定义区间结构体，用于存储区间的起始和结束点
struct Interval {
    int start;
    int end;
};

// ⚖️ 比较函数：按照区间的起始点进行升序排序
bool compare(Interval a, Interval b) {
    return a.start < b.start;
}

int main() {
    // ⚡ 提速指令，应对十万级大数据输入
    ios::sync_with_stdio(false);
    cin.tie(NULL);

    int n; 
    cin >> n;

    vector<Interval> intervals(n); 
    for (int i = 0; i < n; ++i) {
        cin >> intervals[i].start >> intervals[i].end;
    }

    int s, t; 
    cin >> s >> t;

    // 🛠️ 预处理：去掉所有完全在[s, t]外的部分，并将重叠部分裁剪到[s, t]内
    vector<Interval> valid_intervals;
    for (auto& interval : intervals) {
        if (interval.end < s || interval.start > t) {
            continue;
        }
        interval.start = max(interval.start, s);
        interval.end = min(interval.end, t);
        valid_intervals.push_back(interval);
    }

    // ⏱️ 排序核心：按照区间的起始点进行升序排序
    sort(valid_intervals.begin(), valid_intervals.end(), compare);

    int current_end = s;  // ⏳ 当前已覆盖的区间终点（初始为s）
    int count = 0;        // 🏆 记录所选区间的数量
    size_t i = 0;         // 🔍 遍历区间的单向不回头指针

    // 当还没有覆盖到终点 t，且还有可用区间时，持续推进
    while (current_end < t && i < valid_intervals.size()) {
        int furthest_end = current_end; // 🚀 寻找当前能达到的最远终点
        
        // 内层循环：把所有能与当前防线衔接的区间（起点 <= current_end）全部扫一遍
        while (i < valid_intervals.size() && valid_intervals[i].start <= current_end) {
            furthest_end = max(furthest_end, valid_intervals[i].end);
            ++i;
        }
        
        // 🛑 细节恶魔：如果最远终点没法向前推进，说明中间出现断层无解
        if (furthest_end == current_end) {
            cout << "无法完全覆盖线段。" << endl;
            return 0;
        }
        
        current_end = furthest_end; // 更新当前防御战线的最远终点
        ++count;                    // 成功选定一道金色屏障
    }

    // 🏆 最终校验：看最远终点是否真正抵御并越过了终点 t
    if (current_end >= t) {
        cout << count << endl;
    } else {
        cout << "无法完全覆盖线段。" << endl;
    }

    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • sys.stdin.read 极速分块： 面对庞大的十万级区间读入，切忌使用常规循环中的 input()。运用 sys.stdin.read().split() 将全部数据一次性读入并分块，处理效率将得到百倍的飙升。
  • 二维列表的高效排序： Python 在对列表排序时具有天然的优雅性。使用匿名函数 valid_intervals.sort(key=lambda x: x[0]) 可以秒级按照区间的左端点完成升序排列。
  • 巧妙的双层循环逻辑： 内部的 while 循环利用下标迭代，不仅做到了逻辑上的精简，更实现了时间复杂度上的单次线性扫描。

💻 Python 终极法术 (线性双指针扫描版)

import sys

def main():
    # 📥 一气呵成，以极速形式将战场内的全部数据捕捉进内存
    input_data = sys.stdin.read().split()
    if not input_data:
        return
        
    n = int(input_data[0])
    intervals = []
    
    idx = 1
    for _ in range(n):
        start = int(input_data[idx])
        end = int(input_data[idx+1])
        intervals.append([start, end])
        idx += 2
        
    s = int(input_data[idx])
    t = int(input_data[idx+1])
    
    # 🛠️ 预处理：清洗区间，去除无关区间，并将重叠区间裁剪到 [s, t] 范围内
    valid_intervals = []
    for start, end in intervals:
        if end < s or start > t:
            continue
        start = max(start, s)
        end = min(end, t)
        valid_intervals.append([start, end])
        
    # ⏱️ 排序核心：以左端点(起点)为基准进行升序排列
    valid_intervals.sort(key=lambda x: x[0])
    
    current_end = s  # ⏳ 当前已成功覆盖到的最远防御边界
    count = 0        # 🏆 记录所选区间的数量
    i = 0            # 🔍 遍历区间的单向索引指针
    m = len(valid_intervals)
    
    # 当还没有覆盖到目标终点 t，且还有未检查的区间时，继续循环
    while current_end < t and i < m:
        furthest_end = current_end  # 🚀 初始化当前轮次能到达的最远终点
        
        # 内层循环：贪心地扫完所有能与当前防线衔接（起点 <= current_end）的区间
        while i < m and valid_intervals[i][0] <= current_end:
            furthest_end = max(furthest_end, valid_intervals[i][1])
            i += 1
            
        # 🛑 细节恶魔：如果这一轮下来最远边界无法推进，说明遇到无法跨越的断层
        if furthest_end == current_end:
            print("无法完全覆盖线段。")
            return
            
        current_end = furthest_end  # 扩展边界到最远可达的地方
        count += 1                  # 计入屏障数量
        
    # 🏆 最终校验：判定是否将整个防线全线安全收复
    if current_end >= t:
        print(count)
    else:
        print("无法完全覆盖线段。")

if __name__ == '__main__':
    main()

💡 技巧和重点详细总结 (骑士团长的经验笔记)

  • 🌟 为什么必须按左端点（起点）排序？： 在前一题“魔法时间表”中，我们为了选择尽可能多且“互不重叠”的区间，采取了按终点排序的策略。而本题的目标是“完全覆盖”，也就是说区间之间允许且通常需要重叠衔接。为了确保我们能从左向右不留空隙地推进，必须保证新加入的区间起点不出现断层，因此必须按照起点升序排序，才能有条不紊地寻找衔接者。
  • 🌟 内层循环不回头的精妙设计： 初学者常常误以为双重 while 循环的时间复杂度会退化为平方级。但在本代码中，仔细观察可以发现，下标变量 i 在内层循环递增后，外层循环并不会重置它。也就是说，每个区间从头到尾只被扫描了刚好一次！这种双指针线性扫描的技术让程序极度丝滑。
  • 🌟 时间复杂度的终极表现： 区间预处理消耗 O(n) 的时间，排序需要 O(n log n)，而随后的单向指针双重循环仅仅花费 O(n)。因此，整体的算法时间复杂度被完美控制在 O(n log n)。即便面对 n = 100,000 的顶格测试，程序也能在几十毫秒内瞬间破阵，堪称完美！

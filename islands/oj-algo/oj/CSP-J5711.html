📜 任务清单 (题目描述)

国王站在最前面不能移动，其余 n 位大臣可以任意调整排队顺序。

每位大臣能获得金币数量的计算公式为：前面所有人的左手数字乘积，除以他自己右手上的数字，并向下取整。

你的任务是帮助安排大臣的顺序，使得所有大臣中获得金币最多的那个人，他所得的金币数降到最低。请输出这个最小的“最大奖励值”。

大殿队列规则：

  • 第 1 行输入一个整数 n，表示大臣的人数。
  • 第 2 行输入两个整数 a 和 b，代表最前面无法移动的国王左手和右手的数字。
  • 接下来 n 行，每行两个整数 a 和 b，分别代表一位大臣左手和右手的正整数。
  • 输出包含一个整数，表示重新排列后，获奖赏最多的大臣能够拿到的最少金币数。

(3个大臣，第2行为国王)
3

📤 奇迹输出阵：

2

数据范围轻度解析：

  • 对于 100% 的数据：1 ≤ n ≤ 1000， 0 < a, b < 10000
  • 重点警示： 1000 个小于 10000 的数连续相乘，结果会极其庞大，远远超出标准 64 位整型的范围！

🧠 魔法道具箱 (核心图解：左右手联合乘积的微积分)

面对极多可能的排列（n最大为1000），全排列绝对会超时。本题的终极破阵核心是区间交换论证的贪心大魔术：

我们可以将每一位大臣看作一张“危险卡片”，其危险系数与他自己的左手数字和右手数字的乘积（a * b）牢牢绑定。让乘积越小、也就是越安全的卡片排在前面，就能奇迹般地让整体最大奖励值压到最低！

⚔️ 证明小劇場：为什么按 a * b 升序排序一定最优？
假设两个相邻的大臣甲和乙，甲在前面，乙在后面。
设他们前面所有人左手数字的乘积总和是 P。
此时，甲拿到的金币是：P / 甲的右数
乙拿到的金币是：(P * 甲的左数) / 乙的右数
如果我们将甲乙两人的位置对调，并进行数学大小推演，可以惊奇地发现：只要满足“甲的左数 * 甲的右数 < 乙的左数 * 乙的右数”，对调前的最大奖励就一定比对付对调后的最大奖励来得小！
核心定理：国王稳坐阵头，大臣以 左手数 * 右手数 升序排队，前缀积路步步推进！

📘 C++ 魔法秘籍 (知识点)

  • 高精度乘除模拟： 由于前缀连乘积会造成极其严重的数值溢出，我们在 C++ 中需要使用 vector<int> 来手动模拟高精度大整型运算。
  • 低位在前的储存法： 在进行大整数乘法时，把个位放在数组的 位置（低位在前）可以极其方便地处理进位问题。但在最后除法以及输出时，要注意逆向恢复。
  • 前导零剥离： 在高精度除法中，由于除数可能较大，结果数组的高位可能会产生多余的零，必须利用循环配合 erase 指令将其彻底洗刷。

💻 C++ 终极法术 (手动高精度数组版)

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

// 🔮 定义大臣结构体：存储左右手板数字
struct Minister {
    int a, b;
};

// ⚖️ 贪心准则：按照左手和右手数字的乘积做升序排列
bool cmp(const Minister& x, const Minister& y) {
    return (long long)x.a * x.b < (long long)y.a * y.b;
}

// 🔢 高精度乘法：大整数 num 乘以单体基本整型 x
vector<int> mul(const vector<int>& num, int x) {
    vector<int> res;
    int carry = 0;
    for (int i = 0; i < num.size(); ++i) {
        long long tmp = (long long)num[i] * x + carry;
        res.push_back(tmp % 10);
        carry = tmp / 10;
    }
    while (carry) {
        res.push_back(carry % 10);
        carry /= 10;
    }
    return res;
}

// 🔢 高精度除法：大整数 num 除以单体基本整型 x，返回商数组
vector<int> div(const vector<int>& num, int x) {
    vector<int> q; 
    int cur = 0;
    // 从高位开始往下依次借位做除法
    for (int i = num.size() - 1; i >= 0; --i) {
        cur = cur * 10 + num[i];
        if (cur >= x) {
            q.push_back(cur / x);
            cur %= x;
        } else {
            q.push_back(0);
        }
    }
    // 🧼 洗掉可能产生的多余前导零
    while (q.size() > 1 && q[0] == 0) {
        q.erase(q.begin());
    }
    // 反转数组，使其重新变回低位在前的规范，方便后续比大小
    reverse(q.begin(), q.end());
    return q;
}

// ⚖️ 高精度大数比较：判定 a 大于 b 是否成立
bool greaterThan(const vector<int>& a, const vector<int>& b) {
    if (a.size() != b.size()) return a.size() > b.size();
    for (int i = a.size() - 1; i >= 0; --i) {
        if (a[i] != b[i]) return a[i] > b[i];
    }
    return false;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, kingL, kingR;
    cin >> n >> kingL >> kingR; 
    
    vector<Minister> mins(n);
    for (int i = 0; i < n; ++i) {
        cin >> mins[i].a >> mins[i].b;
    }
    
    // ⏱️ 依照乘积贪心大法给所有大臣排队
    sort(mins.begin(), mins.end(), cmp);

    // 初始化大数乘积容器，将国王的左手放入
    vector<int> prod;
    int tmp = kingL;
    while (tmp) {
        prod.push_back(tmp % 10);
        tmp /= 10;
    }
    if (prod.empty()) prod.push_back(0);

    vector<int> maxReward(1, 0); // 初始化最大惩罚值为 0

    // 🔍 单向扫描，挨个结算赏金
    for (const auto& m : mins) {
        vector<int> reward = div(prod, m.b); // 结算当前大臣的奖金
        if (greaterThan(reward, maxReward)) {
            maxReward = reward; // 刷新最高金额记录
        }
        prod = mul(prod, m.a); // 将当前大臣的左手乘入前缀大数，留给下一个人
    }

    // 🏆 高位在前逆序打印最终的最优大数值答案
    for (int i = maxReward.size() - 1; i >= 0; --i) {
        cout << maxReward[i];
    }
    cout << endl;

    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • 天生自带高精度： 恭喜 Python 选手！Python 的 int 类型在底层是全自动高精度大数维护。无论是乘以几万还是几亿，都可以直接进行乘法和整除（//）运算，完全没有任何爆内存或整型溢出的危险！
  • 一句话复合键值排序： 仅仅需要一句极其简短的 ministers.sort(key=lambda x: x * x) 即可实现按两个元素的乘积升序进行排序。
  • 极简前缀推进： 伴随循环直接执行简练的自乘更新操作，阅读体验直接拉满。

💻 Python 终极法术 (原生高精度极简版)

import sys

def main():
    # 📥 快速分块读入大厅内的全部数据
    input_data = sys.stdin.read().split()
    if not input_data:
        return
        
    n = int(input_data[0])
    king_l = int(input_data[1])
    # king_r = int(input_data) 按题意读入，但国王由于在最前，其右手不参与除法运算
    
    ministers = []
    idx = 3
    for _ in range(n):
        a = int(input_data[idx])
        b = int(input_data[idx+1])
        ministers.append((a, b))
        idx += 2

    # ⏱️ 核心贪心：使用 lambda 匿名表达式，直接按照左右手数字乘积从小到大升序排序
    ministers.sort(key=lambda x: x[0] * x[1])

    prod = king_l     # 🔮 初始化前缀大数，将国王左手数字装入
    max_reward = 0    # 🏆 累计追踪各轮次产生的最大奖励值

    # 🔍 顺序遍历每一位排好队的大臣
    for a, b in ministers:
        reward = prod // b  # 💥 Python原生整除大数法术，向下取整
        if reward > max_reward:
            max_reward = reward  # 捕捉最大值
        prod *= a  # 刷新前缀乘积，将其推入下一轮

    # 🏆 直接大方打印出降到最低的限制级最大奖励数
    print(max_reward)

if __name__ == "__main__":
    main()

💡 技巧和重点详细总结 (小小军师的复盘日志)

  • 🌟 贪心之魂的深度反思： 这道题在算法竞赛中有一个非常大名鼎鼎的本源名字叫做“国王游戏”。许多初学者第一次见到它时，会下意识地觉得：“既然是求最大值中的最小值，是不是应该按右手降序或者左手升序来简单排序？” 千万别拍脑袋想当然！ 这种牵扯到前缀累乘与后续除法关联的问题，唯有利用严密的“相邻项微调交换法”在草稿纸上进行公式约分化简，才能剥离出 a * b 这一具有黄金法则效果的单一评判键值。
  • 🌟 语言特性的奇妙博弈： 本题非常生动地展示了不同编程语言之间的利弊。C++ 拥有极高的执行效率，但由于原生不支持千位级别的大整数，我们被迫写了长达几十行的手工业级高精度向量模拟函数，非常考验基本功。而 Python 在高精度计算领域的内置全自动优越性，让我们能用短短二十行不到的代码直接优雅通关！
  • 🌟 绝对稳健的时间复杂度： 算法唯一的算力大开销在于最初对大臣的排序，需要消耗 O(n log n) 的时间。随后的线性单向扫描循环只需要 O(n) 复杂度。即便是面对 100% 顶格的 n = 1000 规模测试，程序也能在零点几秒内瞬间轻松破阵，稳拿满分！

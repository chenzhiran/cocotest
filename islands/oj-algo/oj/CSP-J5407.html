📜 任务清单 (题目描述)

厨神小猪 Hanke 准备了正好 10 种配料。每种配料必须放，而且只能放 1 克、2 克 或 3 克。

烤鸡的“美味值”就是这 10 种配料克数的总和。现在给你一个目标的美味值 n，请你找出所有能凑成这个美味值的“十味配方”。

配方输出规则：

  • 第一行必须先大声宣布：一共找到了多少种配方（方案总数）。
  • 接下来的每一行写出一个配方（10 个数字）。
  • 配方必须按“字典序”从小到大排列（比如 1 1 1... 必须排在 1 1 2... 前面）。
  • 如果无论如何都凑不出这个美味值，只需输出一行 0 即可。

🧠 魔法道具箱 (核心思路：DFS 深度优先搜索)

面对 10 种配料，我们需要一个个盘子去尝试。这正是深度优先搜索 (DFS) 最擅长的舞台！我们把问题想象成面前摆着 10 个空盘子。

✨ 试错与记录的艺术

我们走到第 1 个盘子前，先放 1 克，然后走向第 2 个盘子也放 1 克……一直走到第 10 个盘子。如果总和恰好等于 n，太棒了，我们把这个配方抄写在小本本上！如果不对，我们就退回到上一个盘子，把 1 克换成 2 克继续试。

为什么天然就是字典序？ 因为我们在每个盘子前，总是按照 1克 -> 2克 -> 3克 的顺序去尝试，所以找出来的配方天然就是从小到大排列的，无需额外排序！

✂️ 神奇的剪枝魔法

别傻乎乎地去试所有情况！我们有 10 个盘子，每个盘子最少放 1 克，最多放 3 克。所以：

  • 最低美味值： 10 × 1 = 10 克。
  • 最高美味值： 10 × 3 = 30 克。

如果你给我的目标美味值是 9，或者 31，我连盘子都不用碰，闭着眼睛就知道绝对凑不出来！直接输出 0 结束战斗！

📘 C++ 魔法秘籍 (知识点)

  • 🗂️ 二维魔法篮子 vector<vector<int>>： 题目要求我们先输出配方的总数，再输出配方。这意味着我们不能边找边输出，必须把找到的配方先存起来！我们可以用一个大篮子（外层 vector）装很多个小篮子（每个小篮子装 10 个数字）。
  • 🔄 状态记录器 recipe： 使用一个普通的数组来记录我们当前 10 个盘子里分别放了多少克调料。一旦第 10 个盘子放完且总和正确，就把这个数组抄进小篮子里。

💻 C++ 终极法术 (代码详解)

#include <iostream>
#include <vector>
using namespace std;

int n;
// 🗂️ 大篮子：用来存放所有成功的配方
vector<vector<int>> solutions; 
// 📝 记录表：记录当前正在尝试的 10 个盘子的克数
int recipe[10]; 

// 🌲 DFS 探险法术
// step: 当前走到第几个盘子了 (0 到 9)
// sum: 目前盘子里已经放了多少克配料了
void dfs(int step, int sum) {
    // 🎯 如果 10 个盘子都放满了 (step 达到 10)
    if (step == 10) {
        if (sum == n) { // 检查美味值是否等于目标 n
            vector<int> sol;
            // 把当前成功的记录表抄进小篮子 sol 中
            for (int i = 0; i < 10; ++i) sol.push_back(recipe[i]);
            // 把小篮子放进大篮子 solutions 里
            solutions.push_back(sol);
        }
        return; // 盘子放完了，不管成没成功都要退回上一步
    }
    
    // 🧂 站在当前盘子前，依次尝试放入 1克、2克、3克
    for (int i = 1; i <= 3; ++i) {
        recipe[step] = i; // 记录放入的克数
        // 走向下一个盘子 (step+1)，并把当前克数加到总和中 (sum+i)
        dfs(step + 1, sum + i);
    }
}

int main() {
    cin >> n;
    
    // ✂️ 究极剪枝魔法：如果目标不在 10 到 30 之间，直接放弃
    if (n < 10 || n > 30) {
        cout << 0 << endl;
        return 0;
    }
    
    // 从第 0 个盘子开始尝试，当前总和为 0
    dfs(0, 0);
    
    // 📢 第一行输出：配方的总数量！
    cout << solutions.size() << endl;
    
    // 📜 逐行输出大篮子里的每一个配方
    for (const auto& sol : solutions) {
        for (int i = 0; i < 10; ++i) {
            if (i > 0) cout << " "; // 保证数字之间有空格
            cout << sol[i];
        }
        cout << endl;
    }
    return 0;
}

🌟 C++ 重点复习小贴士

  • 💡 为什么不用 pop_back() 回溯？ 因为我们用的是固定大小的数组 recipe，每次循环的 recipe[step] = i; 会直接覆盖掉上一次尝试的值。所以我们不需要像使用 vector 那样去手动 pop 弹出元素，代码更加简洁！

🐍 Python 魔法秘籍 (知识点)

  • 📸 列表的影分身 path[:]： 在 Python 中，列表传递的是引用。如果直接把 path 存进答案库，后面回溯修改 path 时，答案库里的也会跟着变空！所以必须用 path[:] 拍一张“照片”（创建副本）存起来！
  • ✂️ 巧妙的边界剪枝： 在进入核心递归之前，先写上 if n < 10 or n > 30:，这几行简单的代码能直接省去无数不必要的计算。

💻 Python 终极法术 (代码详解)

# 🌲 DFS 探险法术
# step: 当前走到第几个盘子
# current_sum: 已经放了多少克配料
# path: 随身携带的小本本，记录当前每个盘子放了多少克
def dfs(step, current_sum, path):
    # 🎯 10 个盘子都放完了
    if step == 10:
        if current_sum == n: # 如果美味值刚好等于 n
            # 📸 关键点：必须用 path[:] 拍一张照片复制存入，不能直接存 path！
            solutions.append(path[:]) 
        return # 结束当前尝试，返回上一步
        
    # 🧂 依次尝试放入 1克、2克、3克
    for i in range(1, 4):
        path.append(i) # 记录在本本上
        
        # 走向下一个盘子，总和增加 i
        dfs(step + 1, current_sum + i, path)
        
        # 回溯！把刚写上的克数擦掉，准备尝试下一种克数
        path.pop() 

# 🗺️ 读取目标美味值
n = int(input())

# ✂️ 究极剪枝：如果美味值要求太离谱，直接宣告失败
if n < 10 or n > 30:
    print(0)
else:
    solutions = [] # 大篮子，准备接收成功的配方
    
    # 从第 0 个盘子开始，当前和为 0，小本本是空的 []
    dfs(0, 0, [])
    
    # 📢 第一行输出：配方的总数量！
    print(len(solutions))
    
    # 📜 逐行打印每一个配方
    for sol in solutions:
        # 用空格胶水把配方里的数字连接成完美的字符串并输出
        print(' '.join(map(str, sol)))

🌟 Python 重点复习小贴士

  • 💡 for i in range(1, 4) 的边界： 永远不要忘记，Python 的 range 函数是“包头不包尾”的。想要生成 1、2、3，结尾必须写成 4 哦！
  • 💡 为什么不用全局变量记录和？ 在 Python 递归中，把 current_sum 作为参数一直往下传递，比使用全局变量更加安全和直观，每次回溯时，current_sum 的值自然就回到了上一层的状态，不用手动去减！

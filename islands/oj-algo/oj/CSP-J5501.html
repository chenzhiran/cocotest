📜 任务清单 (题目描述)

勇敢的小卒从起点 A(0,0) 出发，只能每次向右或向下走一格，目标是抵达终点 B(n,m)。

河面上有一匹“冰马”站在 C(hx,hy)。冰马所在的点，以及它一步能跳到的 8 个“日”字格子，都会变成危险的禁区。小卒绝不能踏上这些格子！

计算任务： 小卒从 A 到 B，一共有多少条安全的路线？

📥 魔法输入阵：
(终点坐标 n=6, m=6，冰马坐标 hx=3, hy=3)
6 6 3 3
📤 奇迹输出阵：
(一共有 6 条安全路线)
6

🧠 魔法道具箱 (核心图解：递推算法)

这道题的核心是递推 (Dynamic Programming)。由于小卒只能往右或往下走，所以到达某个格子 (i, j) 的路径数，只与它左边的格子 (i-1, j) 和上边的格子 (i, j-1) 有关！

👆 冰马在 C 点，它控制的 P1~P8 以及 C 点本身都是无法踏入的黑洞！

✨ 递推的三个关键步骤：

  1. 拉起警戒线 (标记障碍)： 首先，我们要把冰马所在的位置 (hx, hy) 以及它跳跃的 8 个偏移量位置，在地图（数组）上标记为不可通行 (blocked)。
  2. 定义状态魔法： 使用一个二维数组 dp[i][j]，表示从起点走到坐标 (i, j) 的路径总数。如果起点没被封锁，我们就设 dp = 1。
  3. 铺设安全通道 (状态转移)： 从左到右、从上到下遍历每一个格子。如果这个格子被“冻结”了，它的路线数就是 0。如果没有被冻结，它的路线数就是：
dp[i][j] = dp[i-1][j] + dp[i][j-1]
(当前格子的路线数 = 从左边走过来的路线数 + 从上面走过来的路线数)

📘 C++ 魔法秘籍 (知识点)

  • 🛡️ 超级大容器 long long： 棋盘变大时，路径的条数会成倍增加，极容易超过 int 的最大值（约21亿）。所以记录路径的 dp 数组一定要用 long long 类型防止数字“溢出”！
  • 🗺️ 方向偏移数组： 我们用 dx[] 和 dy[] 两个数组来存马步的 8 个方向，配合 for 循环生成控制点，代码既整洁又不容易写错。
  • 🚧 边界防守： 计算马控制的格子时，务必要检查 x >= 0 && x <= n，防止数组越界报错（Segmentation Fault）。

💻 C++ 终极法术 (代码详解)

#include <iostream>
#include <vector>
using namespace std;

int main() {
    int n, m, hx, hy;
    cin >> n >> m >> hx >> hy;

    // 🛡️ 标记障碍格子 (大小为 n+1 行, m+1 列，全填 false)
    vector<vector<bool>> blocked(n + 1, vector<bool>(m + 1, false));

    // 🐴 标记冰马所在位置
    if (hx >= 0 && hx <= n && hy >= 0 && hy <= m)
        blocked[hx][hy] = true;

    // 🗺️ 8 个马步的偏移量
    int dx[] = {1, 2, 2, 1, -1, -2, -2, -1};
    int dy[] = {2, 1, -1, -2, -2, -1, 1, 2};
    for (int k = 0; k < 8; ++k) {
        int x = hx + dx[k];
        int y = hy + dy[k];
        // 🚧 检查是不是在棋盘内部
        if (x >= 0 && x <= n && y >= 0 && y <= m)
            blocked[x][y] = true;
    }

    // 🛑 如果起点一开始就被冰马冻住了，那无路可走！
    if (blocked[0][0]) {
        cout << 0 << endl;
        return 0;
    }

    // ✨ DP 数组，用 long long 防止数字过大溢出
    vector<vector<long long>> dp(n + 1, vector<long long>(m + 1, 0));
    dp[0][0] = 1; // 设起点路径数为 1

    // 🏃 开始推导每一格的路线总数
    for (int i = 0; i <= n; ++i) {
        for (int j = 0; j <= m; ++j) {
            if (blocked[i][j]) continue; // 冰雪禁区不可进入
            
            // 从上面走下来
            if (i > 0) dp[i][j] += dp[i - 1][j];
            // 从左边走过来
            if (j > 0) dp[i][j] += dp[i][j - 1];
        }
    }

    // 🏆 输出最终终点的路线数
    cout << dp[n][m] << endl;
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • 📐 二维数组的创造： 在 Python 里生成二维数组要注意避坑，必须用 [ * (m + 1) for _ in range(n + 1)] 的列表推导式。千万不能用 [ * (m + 1)] * (n + 1)，否则修改一行会把所有行都改掉！
  • 🐎 元组记录方向： 用一个包含多个元组的列表 [(1, 2), (2, 1)...] 来存储马步偏移量，遍历时直接 for dx, dy in moves:，写起来超级优雅。
  • ♾️ 自动防溢出： Python 的整数是自动扩展的，无论路径数量有多大，都不会像 C++ 那样发生整型溢出，这是 Python 魔法的隐藏福利！

💻 Python 终极法术 (代码详解)

def main():
    # 读入四个参数：棋盘长宽，冰马坐标
    n, m, hx, hy = map(int, input().split())

    # 🛡️ 生成二维列表标记障碍格子
    blocked = [[False] * (m + 1) for _ in range(n + 1)]

    # 🐴 标记冰马所在位置，前提是马在棋盘内
    if 0 <= hx <= n and 0 <= hy <= m:
        blocked[hx][hy] = True

    # 🗺️ 8 个马步的相对偏移量
    moves = [(1, 2), (2, 1), (2, -1), (1, -2),
             (-1, -2), (-2, -1), (-2, 1), (-1, 2)]
             
    for dx, dy in moves:
        x, y = hx + dx, hy + dy
        # 🚧 越界检查
        if 0 <= x <= n and 0 <= y <= m:
            blocked[x][y] = True

    # 🛑 如果起点被阻塞，直接输出 0
    if blocked[0][0]:
        print(0)
        return

    # ✨ DP 二维数组
    dp = [[0] * (m + 1) for _ in range(n + 1)]
    dp[0][0] = 1  # 起点路线为 1

    # 🏃 核心递推过程
    for i in range(n + 1):
        for j in range(m + 1):
            if blocked[i][j]:
                continue # 是障碍就跳过
            
            # 累加上面的格子带来的路径
            if i > 0:
                dp[i][j] += dp[i - 1][j]
            # 累加左边的格子带来的路径
            if j > 0:
                dp[i][j] += dp[i][j - 1]

    # 🏆 输出终点的安全路线数
    print(dp[n][m])

if __name__ == "__main__":
    main()

💡 技巧和重点总结 (划重点啦)

  • 🌟 DP的灵魂是初始化： 起点的状态 dp = 1 是整个水流的发源地。如果不设为 1，或者起点一开始就被马给冻住了没有特殊判断，最终算出来就会全都是 0。
  • 🌟 边界！边界！还是边界： 在棋盘问题中，计算偏移坐标（如马步）时，永远不要忘了加上 x >= 0 && x <= n 这样的判断，否则会改写到不属于地图的内存区域导致崩溃！
  • 🌟 画图是破题的关键： 递推关系不是死记硬背的。像这道题，只要你在纸上画出坐标系，一看就知道一个点只能从“上边”或“左边”走过来，状态转移方程自然就浮现了！

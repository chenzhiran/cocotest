📜 任务清单 (题目描述)

给出一个高精度正整数 n，你需要从中删去恰好 k 个数字。

要求剩下的数字保持原来的左右相对顺序不变，使得重新组合出来的非负整数尽可能地小。

通关特别警示规则：

  • 只能进行删除操作，不能更改数字的大小，也不能打乱没有被删除的石板的左右位置。
  • 删除数字后得到的新数字前面可能会出现多余的 0（前导零）。
  • 重要约束： 最终输出答案时，绝对不能输出前导零！例如 0012 应该精简输出为 12。
  • 如果把所有非零石板删除后，最后剩下的全都是 0，那么请输出一个孤零零的 0。

  • 正整数 n 非常庞大，必须作为字符串读取和处理，切不可直接存入标准整型。

🧠 魔法道具箱 (核心图解：喜新厌旧的递增序维护)

要想让剩下的数字整体最小，核心的贪心大智慧是：让高位（左边）的数字尽量小！

也就是说，当我们从左往右遍历数字时，如果发现紧挨着的左边石板比当前的右边石板大，那左边的高位石板就是让数值变大的“罪魁祸首”，必须果断把它扣掉！

这种一边遍历一边把较大的左邻居赶走的数据结构模型，就叫做单调栈。

第 1 步： 遇到 1，此时栈为空，直接放入。

第 3 步： 遇到 3。哎呀！栈顶的 4 居然比 3 大！趁着删除次数 k > 0，立刻将 4 弹射爆破！(k 变为 1)。之后 3 顺利搬入。

✨ 单调栈删数四步法

  1. 🔸 miniatures 单调大扫描： 从左往右依次审阅大数 n 的每一位数字字符。
  2. 🔸 逆向狂轰炸： 只要当前栈不为空、且栈顶的数字大于当前正准备入栈的数字、并且手里还有剩余的删除次数 k，说明触发了优化契机。立刻弹出栈顶元素，同时让 k 自减 1。重复此检查直到条件不满足，再把当前数字压入栈。
  3. 🔸 尾部切削法： 如果整个大数都扫描完了，手里的删除次数 k 仍然大于 0（比如输入 123456 且 k=2，一路上都是递增的，没有触发任何弹栈），这说明末尾的数字最大，直接强制将栈末尾的 k 个元素削去。
  4. 🔸 精整前导零： 将栈中残留的字符拼接成串。寻找第一个不是 '0' 的字符位置。将其后面的部分切片输出。若全都是 '0'，则只需大方打印一个 '0'。
核心思想：高位大数如流沙，留小除大靠当下。若逢逆序即弹栈，前导清零开密码。

📘 C++ 魔法秘籍 (知识点)

  • 字符串模拟栈： C++ 中的 std::string 具备绝佳的栈特异性。我们可以直接使用 st.back() 查看末尾，用 st.pop_back() 弹出末尾，用 st.push_back() 压入，免去了额外开辟存储的烦恼。
  • 尾部动态裁剪 resize： 如果扫描结束仍有没用完的删除额度，利用 st.resize(st.size() - k) 可以瞬间切掉字符串尾部不需要的较大高危字符。
  • 前导零精确定位 finder： 利用内置的强大的字符串检索工具 st.find_first_not_of('0')，能一网打尽所有前导零。如果找不到非零位置（返回 string::npos），则说明整个序列全是由零伪装的，直接输出 0。

💻 C++ 终极法术 (经典单调栈版)

#include <iostream>
#include <string>
using namespace std;

int main() {
    // ⚡ 提速法阵：解除输入输出流解绑，大数据下不卡常
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    string n;
    int k;
    cin >> n >> k;

    string st; // 🔮 用标准的 string 动态容器来完美模拟“数字石板栈”
    
    // 🔍 从左往右单向扫描每一个石板字符
    for (char c : n) {
        // 💥 贪心风暴：当栈非空、栈顶数字大于当前字符、且依然拥有删除额度时
        while (!st.empty() && st.back() > c && k > 0) {
            st.pop_back(); // 弹出并丢弃当前不合格的高位大数
            --k;           // 消耗一次删数咒机会
        }
        st.push_back(c);   // 顺理成章将当前较小的潜力新星压入栈顶
    }

    // ✂️ 尾部收拾残局：如果数字本身是单调递增的导致 k 没用完，直接削去末尾最大的 k 位
    if (k > 0) {
        st.resize(st.size() - k);
    }

    // 🧼 破除前导零幻象：寻找第一个不是 '0' 的神圣文字位置
    size_t pos = st.find_first_not_of('0');
    
    if (pos == string::npos) {
        // 如果漫山遍野全都是 '0'，按照古老训诫，只输出一个单独的 0
        cout << 0 << "\n";
    } else {
        // 🏆 从第一个非零勇士的位置切片，输出核心真实的黄金秘码
        cout << st.substr(pos) << "\n";
    }

    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • 标准列表做栈： Python 的标准列表 stack = [] 天然就是最顶级的栈。使用 stack[-1] 可以在 O(1) 时间内查看栈顶，用 stack.pop() 可以完美弹出。
  • 切片大杀器： 针对还没删够的情况，利用 Python 极其霸道的负数切片技术 stack = stack[:-k]，一行代码就能利落地斩断尾部的冗余数字。
  • 一键洗零法 lstrip： 拼接好字符串后，利用特有的 lstrip('0') 可以直接从左边洗刷掉所有的前导零字符。随后配合三目运算符表达式判断空串，代码极其简练优美。

💻 Python 终极法术 (极简单调栈版)

import sys

def main():
    # 📥 极速读入大数线索，并顺手洗掉可能存在的边缘空格
    n = sys.stdin.readline().strip()
    k = int(sys.stdin.readline().strip())

    stack = [] # 🔮 建立空列表，用来扮演吞噬大数的单调结构
    
    # 🔍 顺序审阅高精度数字字符串里的每一个字符
    for ch in n:
        # 💥 贪心反冲：只要栈里有货、栈顶比当前字符大、且还有多余的删数天命
        while stack and stack[-1] > ch and k > 0:
            stack.pop()  # 毫不留情地驱逐当前的高位大石板
            k -= 1       # 扣减掉珍贵的删除次数
        stack.append(ch) # 把当前字符迎入栈中安放

    # ✂️ 如果遇到一路单调递增的石板导致 k 仍有剩余，利用切片技术果断截断尾部
    if k > 0:
        stack = stack[:-k]

    # 🧼 一剑拂尘：通过 join 成型后，使用 lstrip 方法狂扫一切属于左侧的前导零
    result = ''.join(stack).lstrip('0')
    
    # 🏆 最终大检阅：如果洗完之后空无一物，说明答案就是 0；否则打印真实风采
    print(result if result else '0')

if __name__ == "__main__":
    main()

💡 技巧和重点详细总结 (石门破译者的反思日记)

  • 🌟 为什么局部贪心能推导出全局最优？： 很多初学者做这道题容易陷入狂暴的“排列组合搜索”深渊。然而，数字的大小比对有一条铁律：高位（左侧）数字的大小具有绝对的压倒性特权！ 比如 1xxx 永远比 2xxx 小，哪怕后面再大也无法逆天改命。因此，我们在每一个局部步骤里，都千方百计地让高位（左边）的数字尽可能变小，这种“局部死磕高位”的贪心原则，最终通过单调栈的连环弹射，奇迹般地缔造出了全局最小数的终极形态。
  • 🌟 绝妙的时间复杂度表现： 虽然代码里包含了一个 while 循环嵌套在 for 循环的内部，但这绝对不是平方级 O(n^2) 的慢速算法！仔细观察可以发现：大数 n 中的每一个数字字符，在其一生的旅途中，最多只会被压入栈一次，也最多只会被弹出栈一次。这意味着两个循环指针在整个法术执行期间都是单向不回头的。因此，整体算法的时间复杂度被完美定格在惊人的线性 O(n)！哪怕字符长度达到 250 位甚至 250,000 位，程序也能在 1 毫秒内瞬间秒杀破阵，威力无敌！

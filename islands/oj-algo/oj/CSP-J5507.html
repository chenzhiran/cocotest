📜 任务清单 (题目描述)

斐波那契数列（Fibonacci Sequence）有一个非常简单的生长规则：

  • 第一个数和第二个数都是 1。
  • 从第三个数开始，每个数都是它前面两个数之和。
  • 前几个数看起来像这样：1, 1, 2, 3, 5, 8, 13, 21...

计算任务： 输入一个正整数 n（确保 n ≥ 3），请你算出第 n 个斐波那契数是多少！

📥 魔法输入阵：
(想知道第 3 个数字是多少)
3
📤 奇迹输出阵：
(1 + 1 = 2)
2

🧠 魔法道具箱 (核心图解：滚动变量)

要算第 n 个数，难道要把前面的所有数字都存在一个超长的数组里吗？当然不用！仔细想想，我们每次计算新数字时，只需要用到它前面的那 2 个数字，更早的数字已经没用了！

新数字 (k3) = 前一个数 (k2) + 再前一个数 (k1)

因此，我们可以使用“滚动变量”魔法。只需要准备 3 个小盒子（变量），像接力赛一样，每次算完新数字后，就把位置往后挪一步！

📘 C++ 魔法秘籍 (知识点)

  • ⚙️ 变量的初始赋值： 在 C++ 中，我们要为三个接力队员 k1, k2, k3 分配空间。因为前两个数字固定是 1，所以在最开始就可以直接写 int k1 = 1, k2 = 1;。
  • 🔄 更新顺序千万别错： 在滚动变量时，必须先算新值 k3 = k1 + k2，然后再更新旧值 k1 = k2，最后更新 k2 = k3。如果先写了 k1=k2，原来的 k1 就弄丢啦！
  • 🛡️ 隐藏的溢出陷阱： 虽然这道题的输入示例较小，但要知道，斐波那契数列增长极其可怕（第 47 个数就超过了 C++ int 的最大值 21亿）。如果以后遇到 n 很大的题，记得要把 int 换成超大容量的 long long 哦！

💻 C++ 终极法术 (代码详解)

#include <iostream>
using namespace std;

int main() {
    int n;
    cin >> n;
    
    // 🏁 初始化：k1 和 k2 代表前两个数字，初始都是 1
    // k3 用来装马上要计算出来的新数字
    int k1 = 1, k2 = 1, k3;
    
    // ⏳ 开始递推！因为前 2 个数字已知，我们直接从第 3 个数开始算起
    for(int i = 3; i <= n; i++) {
        // ✨ 第一步：新数字等于前两个老数字相加
        k3 = k1 + k2;
        
        // 🔄 第二步：变量大滚动，为下一次计算做准备！
        // 原来的第 2 个数，退居二线变成了"再前一个数"
        k1 = k2;
        // 刚才算出的新数字，晋升成了"前一个数"
        k2 = k3;
    }
    
    // 🏆 循环结束时，k3 里装的就是我们要找的第 n 个数啦！
    cout << k3 << endl;
    
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • ✨ 无敌的多元赋值魔法： 在 C++ 中我们需要借助一个临时变量 k3 来完成交接，但在 Python 中，一句 a, b = b, a + b 就能同时完成新值计算和变量滚动的全套动作，简直是为递推量身定做的语法糖！
  • ♾️ 自带无限手套的数字： Python 里面的整数类型是可以根据数字大小自动扩展的！无论斐波那契数字膨胀得有多大，你都不用操心它会像 C++ 那样“内存溢出”，Python 会稳稳地帮你算下去。

💻 Python 终极法术 (代码详解)

def main():
    # 📥 读取想要查询的第 n 个数字
    n = int(input())
    
    # 🏁 初始化：a 对应第一个数(1)，b 对应第二个数(1)
    a, b = 1, 1
    
    # ⏳ 我们需要滚动计算的次数是 n - 2 次
    # 因为只要执行 1 次滚动，b 就变成了第 3 个数
    for _ in range(3, n + 1):
        # 🔄 Python 最强一招：同步计算并滚动！
        # 等号右边的 b 赋值给了新的 a
        # 等号右边的 a + b 赋值给了新的 b
        a, b = b, a + b
        
    # 🏆 循环结束后，b 就精准地停在了第 n 个数上
    print(b)

if __name__ == "__main__":
    main()

💡 技巧和重点总结 (划重点啦)

  • 🌟 降维打击的空间优化： 这是算法中最常见的一种优化方式。如果当前状态只和前有限个状态有关，坚决不要开大数组（比如 arr），使用滚动变量能把空间复杂度从 O(n) 直接压缩到极致的 O(1)！
  • 🌟 边界与初始化是递推的灵魂： 所有的递推公式都需要一个坚实的“起点”。对于斐波那契来说，前两个 1 就是基石。忘记给初始变量赋值，递推大厦就会立刻崩塌。
  • 🌟 C++的更新陷阱： 新手写滚动变量最容易犯的错就是：k1 = k2; k2 = k1+k2;。注意看，当执行第二句时，k1 的旧值已经被覆盖了！所以必须引入 k3（或者临时变量）来暂存新计算出的和。

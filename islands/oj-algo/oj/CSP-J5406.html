📜 任务清单 (题目描述)

你有 n 根魔法火柴，必须把它们全部用完，拼出一个 A + B = C 的等式。大门会根据你能拼出多少个不同的等式来给出奖励！

炼金规则：

  • A、B、C 都是用火柴拼成的非负整数。如果数字不是 0，它的最高位不能是 0（比如不能拼出 05）。
  • 符号 + 需要 2 根火柴，符号 = 也需要 2 根火柴。
  • 注意：如果 A 和 B 不一样，A + B = C 和 B + A = C 算是两个不同的等式哦！

🧠 魔法道具箱 (核心思路：枚举与边界剪枝)

我们要在所有的数字组合里寻找满足条件的 A 和 B。但是数字有无限大，难道我们要一直找下去吗？不！计算机魔法需要确立边界。

✨ 为什么 A 和 B 只要猜到 999 就够了？

我们可以算一笔账：拼一个数字最省火柴的就是数字 1（只要 2 根）。要拼出一个四位数，最便宜的也就是 1111，需要 2 × 4 = 8 根火柴。

假设 A 是个四位数（至少 8 根），B 最少是 0（6 根），那 C 也是个四位数（至少 8 根）。再加上加号（2 根）和等号（2 根）。
总花费最少最少也要：8 + 6 + 8 + 4 = 26 根火柴！
可是题目说我们最多只有 24 根火柴呀！所以，绝对不可能出现四位数！A、B、C 的范围最大都在 0 到 999 之间。

✨ 施法步骤：

  1. 打造计费器： 写一个函数 cost(x)，专门用来计算数字 x 需要几根火柴。利用取余数拆解数字的每一位。
  2. 预先算账： 为了加速，我们先把 0 到 1999 所有数字需要的火柴数提前算好存进数组里。这叫“空间换时间”！
  3. 双重循环探险： 让 A 从 0 循环到 999，B 从 0 循环到 999。算出 C = A + B。如果 cost(A) + cost(B) + cost(C) + 4 == n，我们就成功找到了一个等式！

📘 C++ 魔法秘籍 (知识点)

  • 📊 常量字典 need： 直接把 0~9 的火柴数写成数组，查询的时候 need[x % 10] 就能秒出结果！
  • ✂️ 剪枝魔法 continue： 如果单单是 A 的火柴数加上符号的 4 根就已经超过了拥有的总数 n，那 B 根本不用试了，直接 continue 跳过，这会让程序跑得飞快！
  • 🗂️ 预处理容器 vector<int> pre(2000)： 提前把 0~1999 的花销算出来存好，后面查询的时候就是 O(1) 的速度！

💻 C++ 终极法术 (代码详解)

#include <iostream>
#include <vector>
using namespace std;

// 🔢 每个数字所需的火柴数 (0到9)
const int need[10] = {6, 2, 5, 5, 4, 5, 6, 3, 7, 6};

// 🧮 计费器：计算一个完整的整数 x 需要多少根火柴
int cost(int x) {
    if (x == 0) return 6; // 如果直接是数字 0，需要 6 根
    int sum = 0;
    while (x > 0) {
        sum += need[x % 10]; // 把最后一位数字所需的火柴加上
        x /= 10;             // 砍掉最后一位，继续算下一位
    }
    return sum;
}

int main() {
    int n;
    cin >> n;
    
    // 📝 预处理：把 0~1999 的火柴数全部算好存进表格里
    vector<int> pre(2000);
    for (int i = 0; i < 2000; ++i) {
        pre[i] = cost(i);
    }
    
    int ans = 0; // 记录成功的等式数量
    
    // 🔍 开始双重循环寻找 A 和 B
    for (int A = 0; A <= 999; ++A) {
        int ca = pre[A];
        if (ca + 4 > n) continue; // ✂️ 简单剪枝：光是 A 和符号就超标了，跳过！
        
        for (int B = 0; B <= 999; ++B) {
            int cb = pre[B];
            if (ca + cb + 4 > n) continue; // ✂️ 剪枝：A 和 B 加上符号超标了，跳过！
            
            int C = A + B;
            if (C > 1999) continue; // C 最大不超过 1998 (999+999)
            
            int cc = pre[C];
            // 🎯 终极判定：火柴数量刚刚好等于 n！
            if (ca + cb + cc + 4 == n) {
                ++ans;
            }
        }
    }
    
    cout << ans << endl; // 宣告找到的等式数量！
    return 0;
}

🌟 C++ 重点复习小贴士

  • 💡 为什么要有 + 4？ 别忘了题目说，加号和等号分别需要 2 根火柴。很多同学算漏了符号的火柴，导致怎么也算不对哦！
  • 💡 拆分数字的套路： x % 10 取最后一位，x /= 10 去掉最后一位，这个套路在处理数字各位信息时是绝对的黄金法则！

🐍 Python 魔法秘籍 (知识点)

  • 📜 列表推导式： 使用 pre = [cost(i) for i in range(2000)]，一行代码就能把 0 到 1999 所有数字需要的火柴数量算出来并变成一个超级查询本！这体现了 Python 的优雅。
  • 🧮 整除运算符 //： 在 Python 中，砍掉数字的最后一位必须要用 x //= 10，这样才能保证结果依然是干净的整数，而不是带有小数点的浮点数。

💻 Python 终极法术 (代码详解)

# 🔢 每个数字所需的火柴数 (0到9)
need = [6, 2, 5, 5, 4, 5, 6, 3, 7, 6]

# 🧮 计费器：计算一个完整的整数 x 需要多少根火柴
def cost(x):
    if x == 0:
        return 6 # 数字 0 单独处理
    s = 0
    while x > 0:
        s += need[x % 10] # 累加最后一位数字的火柴数
        x //= 10          # 砍掉最后一位 (注意要用 // 整除)
    return s

def main():
    n = int(input())
    
    # 📝 预处理：利用列表推导式瞬间造出一个 0~1999 的查询字典！
    pre = [cost(i) for i in range(2000)]
    
    ans = 0 # 成功等式的计数器
    
    # 🔍 嵌套循环，尝试所有的 A 和 B (0 到 999)
    for A in range(1000):
        ca = pre[A]
        if ca + 4 > n:
            continue # ✂️ 如果 A + 符号就超标了，直接下一位！
            
        for B in range(1000):
            cb = pre[B]
            if ca + cb + 4 > n:
                continue # ✂️ A + B + 符号超标了，跳过！
                
            C = A + B
            if C > 1999:
                continue # 保险起见，防止超出列表范围
                
            cc = pre[C]
            # 🎯 判断最终火柴数是否完美等于 n
            if ca + cb + cc + 4 == n:
                ans += 1
                
    print(ans) # 输出结果

if __name__ == "__main__":
    main()

🌟 Python 重点复习小贴士

  • 💡 为什么要有预处理？ Python 的执行速度相对 C++ 慢一点点。在这个双重循环里，我们要循环 1000 × 1000 = 1,000,000 次。如果每次都去调用 cost 函数拆数字，时间就会很危险。但是通过查列表 pre[A]，速度能提升非常多，这就是用空间换时间的顶级智慧！

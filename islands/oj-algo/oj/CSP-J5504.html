📜 任务清单 (题目描述)

地牢里有 2n 枚棋子：左边全是白子 o，右边全是黑子 *，最右边有两个空位 --。

移动规则：

  • 每次必须选中相邻的两枚棋子（颜色不限），保持左右顺序一起跳到两个空位上。
  • 跳跃时必须跨过至少一枚棋子（不能紧贴着空位平移）。
  • 目标：把棋子变成黑白相间的阵列，并且两个空位要回到最左端。

(一侧棋子的数量 n)

🧠 魔法道具箱 (核心图解：递推降维打击)

面对庞大的棋盘，无脑乱跳肯定会陷入死胡同！这道题的标准解法是：把大问题缩小成小问题 (分治与递推)。我们来看看如何通过固定套路的“两连跳”，把规模为 n 的问题，降级成 n-1 的问题！

✨ 递推的奥秘

你发现了吗？经过上面两步，最右边形成了一个完美的 o* 后缀！而前面的部分变成了 ooo***--，这简直和一开始的 oooo****-- 结构一模一样，只是规模变成了 n=3！

只要我们不断重复这个“两连跳”，每次都能剥离出一个排好的 o* 放后面，直到问题缩小到 n=3 这个无法继续用此套路拆分的基础形态。最后我们只要为 n=3 准备好 4 步固定的“人工走法”，游戏就通关啦！

📘 C++ 魔法秘籍 (知识点)

  • 🧵 灵活的 string 操作： 我们用字符串来模拟一整排棋子，初始化用 string(n, 'o') + string(n, '*') + "--" 瞬间搞定。
  • 🔄 swap 法术： 移动棋子其实就是交换字符。把第 i 和 i+1 位的字符，与空位交换，只需调用两次 swap 即可。
  • 🛑 Base Case (基础情境) 的硬编码技巧： 在算法竞赛中，当递推缩小到某个极小规模（如这里的 n=3）时，由于边界条件的限制无法继续套用公式。这时候直接输出预先推算好的几个步骤（附带上已经排好的后缀 ss），是非常聪明且高效的做法！

💻 C++ 终极法术 (代码详解)

#include <iostream>
#include <string>
using namespace std;

int n;
string s;

// 📢 输出当前棋盘状态
void output() {
    cout << s << endl;
}

// 🏃 移动一对棋子的魔法函数
void movechess(int start, int end) {
    // 交换第一个棋子和第一个空位
    swap(s[start], s[end]);
    // 交换第二个棋子和第二个空位
    swap(s[start + 1], s[end + 1]);
    // 每次移动完必须立刻汇报状态
    output();
}

int main() {
    cin >> n;
    
    // ✨ 初始化阵列并打印
    s = string(n, 'o') + string(n, '*') + "--";
    output();

    int len = n;
    // 🌀 核心递推：只要规模还能再缩，就一直循环
    while (true) {
        // 步骤1：把边界的 o* 搬到末尾空位
        movechess(len - 1, 2 * len);
        
        len--; // 问题规模缩小！
        
        // 🛑 如果缩到了 n=3，触发退出机制，不能再用套路了
        if (len == 3) break;
        
        // 步骤2：把空位右侧的 ** 搬回刚产生的空位
        movechess(len, 2 * len);
    }

    // 🎁 Base Case 人工补全阶段：处理 n=3 的情况
    string ss;
    // 把之前已经排好的后缀字符串给生成出来
    for (int i = 0; i < n - 4; i++) ss += "o*";
    
    // 按 n=3 的固定走法输出，并在末尾贴上后缀 ss
    cout << "ooo*o**--*" << ss << endl;
    cout << "o--*o**oo*" << ss << endl;
    cout << "o*o*o*--o*" << ss << endl;
    cout << "--o*o*o*o*" << ss << endl;

    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • 🎭 列表与字符串的切换： 因为 Python 的字符串是“不可变”的，不能直接执行 s = '-'，所以我们一开始用 list() 把它变成了可以随意修改的列表。输出时再用 ''.join(s) 把它拼回漂亮的字符串。
  • ✨ 优雅的多元赋值： 交换两个位置的值？在 Python 里连临时变量都不需要！直接 s[start], s[end] = s[end], s[start] 搞定。
  • 🧵 字符串乘法： 快速生成重复内容，一句 'o*' * (n - 4) 就生成了所有排好的后缀组合！

💻 Python 终极法术 (代码详解)

n = int(input())

# ✨ 初始化序列并转为列表方便修改
s = list('o' * n + '*' * n + '--')
print(''.join(s)) # 打印初始状态

def movechess(start, end):
    # 🔄 Python 特色的同行多变量交换
    s[start], s[end] = s[end], s[start]
    s[start + 1], s[end + 1] = s[end + 1], s[start + 1]
    
    # 每次操作后汇报阵型
    print(''.join(s))

len_ = n
# 🌀 核心递推：不断剥离出排好的 o* 后缀
while True:
    # 第一步：交界 o* 搬到末尾
    movechess(len_ - 1, 2 * len_)
    
    len_ -= 1 # 规模减小！
    
    # 🛑 遇到 n=3 的最小无法继续套用公式的局面，果断退出
    if len_ == 3:
        break
        
    # 第二步：把多出的 ** 搬回刚才造成的空位
    movechess(len_, 2 * len_)

# 🎁 终极清场：处理留下的 n=3 残局
# 先生成之前由于循环递推排好的、安放在后面的那堆 o*
ss = 'o*' * (n - 4)

# 直接按 n=3 的标准 4 步走法输出，并拼接上后缀
print('ooo*o**--*' + ss)
print('o--*o**oo*' + ss)
print('o*o*o*--o*' + ss)
print('--o*o*o*o*' + ss)

💡 技巧和重点总结 (划重点啦)

  • 🌟 找规律是构造题的灵魂： 遇到操作类的题目，先在草稿纸上模拟小数据（如 n=4, n=5）。当你发现经过两步操作后，原数组的核心部分“形似神似”最初的状态时，这就暗示你可以用递推思想来解决。
  • 🌟 打破常规的 Base Case： 很多初学者执着于写一个完美的公式解决所有情况，但这往往很难甚至不可能。当公式推导到 n=3 出现冲突时，直接人工枚举、硬编码 (Hardcode) 少数特殊情况，才是最高效解题的大佬行为！
  • 🌟 后缀分离保护法： 在手工补全最后四行打印时，代码非常聪明地把后方已经排好序的 ss 提取了出来直接拼上去。这样做完全剥离了递推生成的成果与手工步骤，逻辑互不干扰，极其清晰！

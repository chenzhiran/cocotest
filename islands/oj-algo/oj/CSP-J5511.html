📜 任务清单 (题目描述)

你负责摆放装有 铀 (U) 和 铅 (L) 两种元素的盒子。按照要求，将 n 个盒子放成一行。

安全规则： 必须满足 至少有三个相邻的盒子装有铀 (U)。（即序列中必须出现 UUU）

计算任务： 给定盒子的数量 n (1 < n < 30)，计算符合安全规则的摆放方式总数。

📥 魔法输入阵：
(一共要摆放 4 个盒子)
4
📤 奇迹输出阵：
(只有 3 种：LUUU, UUUU, UUUL)
3

🧠 魔法道具箱 (核心图解：正难则反)

正面去算“包含至少 3 个 U”的情况非常复杂，因为它们可能在开头、中间、结尾，还可能出现 4 个、5 个 U 相邻，极其容易算重！

这时候我们要用一种绝妙的魔法思维：正难则反 (Complementary Counting)。

  • 总共有两种元素，n 个盒子，完全自由摆放的总情况数是：2^n 种。
  • 如果设 g(n) 是 “绝对没有 3 个连续 U” 的不合格摆放总数。
  • 那么符合要求的摆放数 f(n) 就是：f(n) = 2^n - g(n)

✨ 如何计算不合格的 g(n)？

要保证一个长度为 n 的序列里没有 UUU，那么这个序列的最后几个盒子只能有以下 3 种合法的结尾形态：

👇 没有 UUU 的序列，结尾只能是这 3 种样子 👇
长 n-1 且安全
L
➜ 结尾是 L。前面 n-1 个盒子只要安全即可，有 g(n-1) 种。
长 n-2 且安全
L
U
➜ 结尾是 LU。前面 n-2 个盒子只要安全即可，有 g(n-2) 种。
长 n-3 且安全
L
U
U
➜ 结尾是 LUU。前面 n-3 个盒子只要安全即可，有 g(n-3) 种。

把这三种情况加起来，就得到了递推魔法公式：

$$g(n) = g(n-1) + g(n-2) + g(n-3)$$

(这就是著名的“泰波那契/Tribonacci”数列变体！)

📘 C++ 魔法秘籍 (知识点)

  • ⚡ 位运算巧算 2 的次方： 求 2^n 不要去包含 <cmath> 再用 pow(2, n)！因为 pow 返回的是浮点数，容易有精度误差。直接使用位运算 1 << n，速度极快且完全是整数操作！
  • 🛡️ 初始化基础状态： 遇到往前查 3 个状态的递推，一定要手动把 g(0), g(1), g(2) 的值初始化好：
    g(0)=1 (空盒子，0个UUU，算1种安全状态)
    g(1)=2 (U, L 都没达到3个，全安全)
    g(2)=4 (UU, UL, LU, LL 都没有3个U，全安全)

💻 C++ 终极法术 (代码详解)

#include <iostream>
#include <vector>
using namespace std;

// 🎨 计算“没有连续3个U”的不合格摆放数 g(n)
int g(int n) {
    // 🏁 处理基础边界情况
    if (n == 0) return 1;
    if (n == 1) return 2;
    if (n == 2) return 4;
    
    // 创建记忆魔法书 w，大小为 n+1
    vector<int> w(n + 1, 0);
    w[0] = 1;
    w[1] = 2;
    w[2] = 4;
    
    // 🌀 核心递推：当前状态由前三个状态相加而得
    for (int i = 3; i <= n; i++) {
        w[i] = w[i - 1] + w[i - 2] + w[i - 3];
    }
    
    return w[n];
}

// 🎨 最终法术 f(n)：总情况 - 不合格情况
int f(int n) {
    // 1 << n 就等同于 2 的 n 次方，超级高效！
    int total = 1 << n; 
    return total - g(n);
}

int main() {
    int n;
    cin >> n;
    
    // 按照题目规模 1 < n < 30 执行法术
    if (n < 30) {
        cout << f(n) << endl;
    }
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • ✨ 空间压缩的艺术： 和前面练习的斐波那契数列一样，这里 g(n) 只需要用到它前面的 3 个状态，所以我们完全可以用 a, b, c = b, c, a+b+c 这种超级炫酷的滚动变量来写，连列表都不需要开！
  • ⚡ 强大的位运算： Python 中求 2 的 n 次方，既可以写 2 ** n，也可以写 1 << n。两者都很棒，但位运算的逼格更高！

💻 Python 终极法术 (代码详解)

# 🎨 计算“没有连续3个U”的不合格摆放数 g(n)
def g(n):
    # 🏁 基础边界情况
    if n == 0: return 1
    if n == 1: return 2
    if n == 2: return 4
    
    # 🚀 使用滚动变量魔法，a, b, c 分别代表 g(0), g(1), g(2)
    a, b, c = 1, 2, 4
    
    # 🌀 从第 3 项循环到第 n 项
    for _ in range(3, n + 1):
        # Python 多元赋值：计算出新值后，老变量集体向后滚动！
        a, b, c = b, c, a + b + c
        
    # 循环结束时，c 就停在了 g(n) 的位置
    return c

def f(n):
    # 总情况(2的n次方) 减去 不合格情况
    return (1 << n) - g(n)

def main():
    n = int(input())
    if n < 30:
        print(f(n))

if __name__ == "__main__":
    main()

💡 技巧和重点总结 (划重点啦)

  • 🌟 逆向思维 (正难则反)： 当题目里出现“至少含有 X”这种关键词，且正面分类讨论极度复杂时，第一时间思考用“总数 - 完全不含 X 的数量”来解答！这不仅能简化代码，更能防止因为考虑不周导致的算重算漏。
  • 🌟 状态结尾强制分类： 在推导 g(n) 时，我们通过强行规定最后几个盒子的元素（即以 L, LU, LUU 结尾），将 g(n) 完美地拆解成了前面已经算过的三个更小的状态。这就是“划分无重叠状态空间”的 DP 灵魂！
  • 🌟 超出类型的危机预警： 注意题目中的 n < 30，这意味着 2^{29} 的数量级在 5 亿多，仍然在 32 位整型 int (最高约21亿) 的安全范围内。如果题目规定 n = 50，请务必在 C++ 中毫不犹豫地换上 long long！

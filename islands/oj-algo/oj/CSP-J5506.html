同学们围成了一个神奇的圆圈传球。由于圆圈是首尾相连的，我们可以用“动态规划”的魔法，算出球在每个人手中流转的所有可能！

📜 任务清单 (题目描述)

林间空地上，n 位同学围成一个圈，小蛮站在 1号位 并拿着发光球。每次传球只能传给左边或右边的相邻同学。

传球规则：

  • 相邻关系是环形的：1 号的相邻是 2 号和 n 号；n 号的相邻是 n-1 号和 1 号。
  • 你需要计算：从小蛮（1号）开始，恰好传了 m 次后，球又回到小蛮手里的“不同传球序列”一共有多少种。
📥 魔法输入阵：
(3 位同学成环，传球 3 次)
3 3
📤 奇迹输出阵：
(2 种路径：1→2→3→1 和 1→3→2→1)
2

🧠 魔法道具箱 (核心图解：破环成链与状态转移)

这是经典的动态规划 (DP) 问题。要计算第 t+1 次传球后，球在某个人手里的方法数，我们只需要知道第 t 次传球后，球在他的左边邻居和右边邻居手里的方法数，然后把它们加起来！

👇 “破环成链”：为了看清 1 号的邻居，我们把圈展开 👇
n-1
n
(1的左邻居)
1
(目标)
2
(1的右邻居)
3
↘ -- ↙
传到 1 号的方法数 = (球从 n 号传来的方法) + (球从 2 号传来的方法)

✨ 状态转移方程式

如果用二维数组 dp[t][i] 表示“传了 t 次后，球在 i 号同学手里的方法数”，那么魔法公式如下：

$$dp[t+1][i] = dp[t][left(i)] + dp[t][right(i)]$$

注意边界魔法： 当 i = 1 时，左邻居是 n；当 i = n 时，右邻居是 1。

🔄 空间优化术：滚动数组

你发现了吗？计算第 t+1 步的数据时，我们只依赖第 t 步的数据，更早的数据完全用不上了！所以我们不需要一个庞大的二维矩阵，只需要两个一维数组（当前步 cur 和下一步 nxt）交替“滚动”使用，就能把空间复杂度从 O(nm) 降到 O(n)。

📘 C++ 魔法秘籍 (知识点)

  • 🛡️ 超级大容量 long long： 当人数增加且传球次数很多时，方法数会呈指数级爆炸。使用 vector<long long> 能防止数字溢出变成负数。
  • 🧹 快速清空数组 fill()： 每一轮计算新的一步时，nxt 数组必须先清零。fill(nxt.begin(), nxt.end(), 0) 是一句非常高效的清理魔咒。
  • ⚖️ 三目运算符巧解边界： int left = (i == 1) ? n : i - 1; 这一行代码既优雅又简洁地解决了首尾相连的环形边界问题。

💻 C++ 终极法术 (代码详解)

#include <iostream>
#include <vector>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;
    
    // 🚀 滚动数组：cur 记录当前步，nxt 记录推导出的下一步
    vector<long long> cur(n + 1, 0), nxt(n + 1, 0);
    
    // 🏁 初始化：第 0 步时，球在 1 号小蛮手里，有一种情况
    cur[1] = 1; 

    // ⏳ 开始传球！一共传 m 步
    for (int step = 1; step <= m; ++step) {
        // 🧹 每算新的一步前，把下一步的记录板擦干净
        fill(nxt.begin(), nxt.end(), 0);
        
        // 遍历每个同学，看看他手里的球能传给谁
        for (int i = 1; i <= n; ++i) {
            if (cur[i] == 0) continue; // 如果这个人手里没球的方法数，直接跳过加速
            
            // 🧮 计算左右邻居的真实编号 (处理环形边界)
            int left = (i == 1) ? n : i - 1;
            int right = (i == n) ? 1 : i + 1;
            
            // ✨ 把当前位置积累的方法数，分别传递给左边和右边
            nxt[left] += cur[i];
            nxt[right] += cur[i];
        }
        
        // 🔄 滚动替换：把推导好的 nxt 变成下一次循环的 cur
        swap(cur, nxt);
    }
    
    // 🏆 输出 m 次传球后，球回到 1 号小蛮手里的方法数
    cout << cur[1] << endl;
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • ✨ 快速列表重置： 在每次循环中写 nxt = * (n + 1) 就能瞬间创建一个全新的、装满 0 的数组用来记录下一步的状态，非常符合 Python 的简洁美学。
  • 🔄 极简的数组更替： 循环末尾的 cur = nxt 直接将变量指针重新绑定，完成了滚动数组中最核心的状态交接。
  • 🛡️ 大数无忧： Python 最棒的一点是整数大小不受限制！你完全不需要像 C++ 那样去操心 int 和 long long，就算方法数大到冲破天际，Python 也能精准计算。

💻 Python 终极法术 (代码详解)

def main():
    # 📥 读入同学数量 n 和 传球次数 m
    n, m = map(int, input().split())
    
    # 🚀 创建当前步的状态列表，下标从 1 到 n
    cur = [0] * (n + 1)
    
    # 🏁 初始化：一开始球在 1 号手里，只有这 1 种确定情况
    cur[1] = 1
    
    # ⏳ 传球过程，重复 m 次
    for _ in range(m):
        # 🧹 准备一个新的列表，用来装载传完这一步后的结果
        nxt = [0] * (n + 1)
        
        # 扫描每个人的手牌
        for i in range(1, n + 1):
            if cur[i] == 0:
                continue  # 优化：手里完全没有可能的球，跳过
                
            # 🧮 环形边界处理：Python 单行 if-else 魔法
            left = i - 1 if i > 1 else n
            right = i + 1 if i < n else 1
            
            # ✨ 当前 i 号的路线数，分裂传递给左和右
            nxt[left] += cur[i]
            nxt[right] += cur[i]
            
        # 🔄 滚动交接：旧的一步作废，新的一步成为当前步
        cur = nxt
        
    # 🏆 游戏结束，看看 1 号同学手里积累了多少种路线
    print(cur[1])

if __name__ == "__main__":
    main()

💡 技巧和重点总结 (划重点啦)

  • 🌟 万物始于初态： DP 最重要的一步是初始化。这里 cur = 1 是所有可能性的“火种”，如果全初始化为 0，无论循环多少次算出来的也都是 0。
  • 🌟 逆向推导 vs 正向贡献： 这个题的题解用的是“我把球传出去”（把 cur[i] 贡献给 nxt[left] 和 nxt[right]），这种“贡献法”在很多状态转移里比“我从哪里接球”要想得更清楚自然。
  • 🌟 奇偶性小彩蛋： 在纸上画一画你会发现：当总人数 n 是偶数时，传奇数次球（m 是奇数），球绝对不可能回到最初的 1 号手里（答案必然是 0）。这就像是在国际象棋黑白格子里走动一样，走奇数步一定会换颜色！

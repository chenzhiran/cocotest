📜 任务清单 (题目描述)

冰川上有一串正整数宝珠，从左到右排成队。每当左边的宝珠数字 严格大于 右边的宝珠数字时，就会产生一道“逆序刻痕”！

数学定义： 如果 $i < j$ 并且 $a_i > a_j$，那么这就是一个逆序对。

你的任务： 算出这串数字里一共有多少个逆序对！注意，宝珠最多有 500,000 个，刻痕的总数可能会非常非常大哦！

📥 魔法输入阵：
(6颗宝珠)
6
5 4 2 6 3 1
📤 奇迹输出阵：
(经过仔细计算，一共产生了11道刻痕！)
11

🧠 魔法道具箱 (核心思路：分治雪锯 / 归并排序)

如果用两个手指头一个一个去比对，50 万颗宝珠会让你比对到天荒地老（这叫 $O(n^2)$，会超时的）！

我们需要请出超级魔法——“分治雪锯” (归并排序)！把宝珠不断从中间锯开，直到每一边都只有 1 颗，然后在“拼合”它们的时候，顺便数一数刻痕！

看这里！拼合时的神奇计数法：
假设左边和右边都已经各自从小到大排好队了：
左半边：
5
8
右半边：
2
6
💡 发现刻痕！
我们拿左边的 5 和右边的 2 比。发现 5 > 2！
因为左边是排好序的，既然 5 都比 2 大，那 5 后面的 8 肯定也比 2 大！
所以，右边的 2 一下子跳过了左边的 2个数字 (5 和 8)。
产生逆序对：+2！ 简直是批量计算，太快啦！🚀

📘 C++ 魔法秘籍 (知识点)

  • 🪚 分治递归 merge_count： 我们自己写一个函数，它能把数组劈成两半，然后把左右两边产生的逆序对加起来。
  • 🛡️ 防爆保护罩 long long： 50万个数字，如果完全是倒序的（比如 6 5 4 3 2 1），逆序对会有大约 1250亿 个！普通的 int 只能存 21 亿，瞬间就会爆炸。所以记录逆序对数量的变量一定要用 long long！
  • 📦 临时辅助口袋 temp： 归并排序需要一个一模一样大的空口袋 temp 来临时存放排好序的宝珠，排完后再倒回原来的数组里。

💻 C++ 终极法术 (代码详解)

#include <iostream>
#include <vector>
using namespace std;

// 🪚 分治雪锯魔法：不仅能排序，还能返回一共有多少逆序刻痕！
long long merge_count(vector<int> &a, int l, int r, vector<int> &temp) {
    if (l >= r) return 0; // 如果只有1颗宝珠，刻痕是 0
    
    int mid = (l + r) / 2; // 找到中间劈开
    
    // 先算左边一半的刻痕，再算右边一半的刻痕
    long long cnt = merge_count(a, l, mid, temp) + merge_count(a, mid + 1, r, temp);
    
    // 开始拼合左右两半！
    int i = l;       // 左半边读取指针
    int j = mid + 1; // 右半边读取指针
    int k = l;       // 临时口袋放入指针
    
    while (i <= mid && j <= r) {
        if (a[i] <= a[j]) {
            // 左边小，正常顺序，乖乖放进临时口袋
            temp[k++] = a[i++];
        } else {
            // 💡 发现逆序刻痕！左边当前数字大于右边！
            // 那么左边剩下的所有数字（从 i 到 mid）都比 a[j] 大！
            cnt += (mid - i + 1); 
            temp[k++] = a[j++]; // 把右边较小的放进临时口袋
        }
    }
    
    // 把没放完的尾巴全部扫进临时口袋
    while (i <= mid) temp[k++] = a[i++];
    while (j <= r) temp[k++] = a[j++];
    
    // 🔄 把临时口袋里排好序的宝珠，倒回原来的数组里
    for (int p = l; p <= r; ++p) a[p] = temp[p];
    
    return cnt; // 返回总刻痕数
}

int main() {
    // ⚡ 加速读入咒语
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n;
    cin >> n;
    
    vector<int> a(n);
    for (int i = 0; i < n; ++i) cin >> a[i];
    
    vector<int> temp(n); // 📦 准备临时空口袋
    
    // 施展分治魔法，从第 0 个算到第 n-1 个
    long long ans = merge_count(a, 0, n - 1, temp);
    
    // 🎉 喊出总刻痕数量！
    cout << ans << '\n';
    
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • 🪚 分治递归与切片： arr[:mid] 和 arr[mid:] 就像是直接用刀把蛋糕切成两半。虽然切片会产生新的小列表，但在 Python 里写起来非常直观清晰！
  • 🛡️ 无限大口袋： Python 最棒的一点就是它的整数（int）天生自带防爆属性！不管逆序对是 10 个还是 1250亿 个，Python 都能轻松装下，完全不用担心溢出。
  • 🌪️ 批量大扫除 extend： 当我们把两半合并完后，可能有一半还没走完。用 merged.extend(left[i:]) 能一口气把剩下的宝珠全部扫进新列表里，比一个一个 append 快多了！

💻 Python 终极法术 (代码详解)

import sys # 召唤基础工具箱

# 🪚 分治雪锯魔法：返回排好序的新列表，以及逆序刻痕数量！
def merge_count(arr):
    if len(arr) <= 1:
        return arr, 0 # 如果只有1颗或0颗宝珠，刻痕是 0
        
    mid = len(arr) // 2 # 从中间切开！
    
    # 递归！分别去算左半边和右半边的刻痕
    left, inv_left = merge_count(arr[:mid])
    right, inv_right = merge_count(arr[mid:])
    
    merged = [] # 准备一个空列表放排好序的宝珠
    i = j = 0
    inv = inv_left + inv_right # 总刻痕数先把左右两边的加上
    
    # 拼合左右两边
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            merged.append(left[i])
            i += 1
        else:
            # 💡 发现逆序刻痕！左边大于右边！
            merged.append(right[j])
            # 因为 left 是排好序的，left[i] 后面的所有数字都比 right[j] 大！
            inv += len(left) - i  # 批量加上逆序对数量！
            j += 1
            
    # 🌪️ 用 extend 把剩下的尾巴全部扫进来
    merged.extend(left[i:])
    merged.extend(right[j:])
    
    return merged, inv

def main():
    # 🌪️ 用吸尘器把所有字都吸进来
    data = sys.stdin.buffer.read().split()
    if not data:
        return

    n = int(data[0])
    arr = list(map(int, data[1:1 + n]))
    
    # 🪄 施放分治魔法！我们只需要返回的刻痕数量（第二个返回值 ans）
    _, ans = merge_count(arr)
    
    # 🎉 大声喊出总刻痕数量！
    sys.stdout.write(str(ans))

if __name__ == "__main__":
    main() # 🚀 启动魔法！

🌟 重点复习小贴士

  • 💡 为什么一定要边排序边数？ 如果不排序，我们每次看到右边有个小数字，还得回头去左边一个个找有几个比它大。排好序后，因为有了“顺序”，我们一眼就能看出来左边剩下的全部都比它大，一步到位！这就是算法的魅力！
  • 💡 严格大于才算！ 如果两颗宝珠数字一样（比如 left[i] == right[j]），按照规则不算逆序，所以要走 left[i] <= right[j] 的路线，乖乖把它放进合并后的数组里，不增加刻痕。

📜 任务清单 (题目描述)

给定 n 个开区间 (a, b)，目标是选择尽量多的区间，使得这些区间没有公共点（互不重叠）。请输出最多可以选出的区间数量。

大殿调度法则：

  • 每个实验占用一个开区间 (a, b)，其中 a 是起点，b 是终点。
  • 因为是开区间，所以如果上一个实验在时刻 b 结束，下一个实验在时刻 b 开始（即没有公共点），这两个实验是可以无缝衔接的！
📥 魔法输入阵：
(4个巫师的申请时段)
4
1 3
2 4
3 5
6 8
📤 奇迹输出阵：
(最大不冲突的实验数)
3

数据规模限制：

  • 1 ≤ n ≤ 100,000
  • -1,000,000 ≤ a < b ≤ 1,000,000

🧠 魔法道具箱 (核心图解：结束时间优先的贪心艺术)

要挑选出最多的不冲突实验，最聪明的策略是：谁先结束，就先安排谁！

为什么？因为一个实验越早结束，留给后面实验的剩余时间就越多！这就叫区间贪心策略。

⏳ 样例时间轴模拟（按结束时间升序排序后） ⏳
1. 实验 (1, 3)
选它！✅ 结束于3
2. 实验 (2, 4)
冲突 ❌ (2 < 3)
3. 实验 (3, 5)
选它！✅ 结束于5
4. 实验 (6, 8)
选它！✅ 结束于8

* 注：因为是开区间，实验 (3, 5) 的起点 3 刚好等于上一次的结束时间 3，完美衔接，不算重叠！

✨ 贪心算法三步破咒法

  1. 🔸 排个序： 使用结构体或元组存储区间的起点和终点，然后以终点（结束时间）为基准进行升序排序。
  2. 🔸 定个界： 记录当前选中的最后一个区间的结束时间 e。初始时，由于没有任何实验被选，我们将它设为一个极小的值。
  3. 🔸 扫一遍： 从前往后遍历所有区间。如果当前区间的开始时间大于或等于 e，说明不冲突！立即把该区间纳入麾下，并将计数器加 1，同时更新 e 为当前区间的结束时间。
核心思想：优先挑选结束早的区间 → 留出最多的未来时空 → 达成全局最大数量解

📘 C++ 魔法秘籍 (知识点)

  • 封装利器 struct： 使用结构体打包区间的 start 和 end，让数据结构整洁清晰。
  • 自定义排序 std::sort： 通过编写回调函数 cmp，告诉 C++ 编译器我们需要按照结构体的 end 升序排列，这是掌握高级竞赛语法必备的基础。
  • 高效输入输出优化： 由于 n 达到了 100,000，建议在正式比赛中加上 cin.tie(NULL); 提升读入效率，避免超时。

💻 C++ 终极法术 (区间贪心版)

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

// 🔮 定义区间结构体，用于存储巫师实验的起始和结束点
struct Int {
    int start;
    int end;
};

// ⚖️ 比较函数：按照区间的结束点进行升序排序
bool cmp(Int a, Int b) {
    return a.end < b.end;
}

int main() {
    // ⚡ 优化输入流性能，助你快人一步
    ios::sync_with_stdio(false);
    cin.tie(NULL);
    
    int n; 
    cin >> n;
    
    vector<Int> a(n); 
    for (int i = 0; i < n; ++i) {
        cin >> a[i].start >> a[i].end;
    }
    
    // ⏱️ 贪心核心：按结束时间从小到大进行排序
    sort(a.begin(), a.end(), cmp);
    
    int count = 0;      // 🏆 记录所选不重叠区间的最大数量
    
    // 🧙‍♂️ 避坑提示：由于坐标范围可至 -1,000,000，初始结束时间需设得足够小
    int e = -2000005;   
    
    // 🔍 遍历所有区间，施展贪心选择法
    for (int i = 0; i < a.size(); ++i) {
        // 如果当前区间的起点大于或等于上一个选择区间的终点（开区间衔接）
        if (a[i].start >= e) {
            e = a[i].end; // ⏳ 更新上一个选择区间的结束时间
            count++;      // 🌟 成功收入囊中
        }
    }
    
    cout << count << endl;
    
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • sys.stdin.read 极速快读： 处理十万级的大规模输入时，用自带的循环 input() 会慢得像蜗牛。利用 sys.stdin.read().split() 一口气把所有魔法字符读进内存切片，效率直接起飞！
  • 优雅的匿名函数 lambda： Python 不需要单独定义排序比较函数。直接使用高级语法 intervals.sort(key=lambda x: x)，一句话就能实现按照区间终点排序，精简而优雅。

💻 Python 终极法术 (快读匿名排序版)

import sys

def main():
    # 📥 使用极速快读法，一口气抓取大殿内所有的魔法数据
    input_data = sys.stdin.read().split()
    if not input_data:
        return
        
    n = int(input_data[0])
    intervals = []
    
    # 📦 将一维数组每两个一组，打包成 (start, end) 的元组放入列表
    idx = 1
    for _ in range(n):
        start = int(input_data[idx])
        end = int(input_data[idx+1])
        intervals.append((start, end))
        idx += 2
        
    # ⏱️ 贪心核心：使用 lambda 表达式，以区间的结束时间(索引为1的值)进行升序排序
    intervals.sort(key=lambda x: x[1])
    
    count = 0
    # 🧙‍♂️ 避坑提示：坐标范围最低到 -1000000，初始结束时间应设得比它更小（负无穷大）
    end_time = float('-inf')
    
    # 🔍 依次审阅每一位巫师的实验申请
    for start, end in intervals:
        # 如果当前实验的起点不早于当前的结束界限
        if start >= end_time:
            end_time = end  # ⏳ 刷新当前时间的终点界限
            count += 1      # 🌟 成功加入时间表
            
    print(count)

if __name__ == '__main__':
    main()

💡 技巧和重点详细总结 (调度官的避坑笔记)

  • 🌟 为什么不能按起点排序？： 很多初学者直觉上会选择按“起点早”来排序。但是试想一下：如果有一个实验它的起点确实最早，但它一占就是一百天（比如区间是 (-100, 100)），那在这期间别的任何小实验就都没法办了！所以，“早结束”才是硬道理。
  • 🌟 初始值的细节恶魔： 原题目给出的代码中，把上一次的结束时间 e 设为了 -1。在样例都是正数的时候这当然没问题，但看一下数据规模提示：-1,000,000 ≤ a < b ≤ 1,000,000，这意味着时间坐标可以是负数！如果有些区间的范围是 (-10, -5)，你的 e 初始是 -1，那这些优秀的区间就会被错误地漏掉！我们在重构的代码中，已经把它的初始值修正为了安全的更小值（C++ 中为 -2000005，Python 中为负无穷 -inf），帮你完美避开陷阱。
  • 🌟 绝妙的时间复杂度： 排序的耗时是 O(n log n)，而随后的单向循环遍历只需要 O(n)。因此，整体的时间复杂度牢牢锁定在 O(n log n)。对于 n = 100,000 的规模，在 C++ 和 Python 中都可以在零点几秒内瞬间出结果，优雅且极速！

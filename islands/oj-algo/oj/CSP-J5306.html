📜 任务清单 (题目描述)

地图上有 N 个“雪晶坐标”，每个点都有自己的三维位置 (x, y, z)。其中 z 代表高度，每个点的高度都不一样哦！

飞行规则：

  • 必须从高度最低的点出发，严格按照高度 z 从小到大 的顺序飞向下一个点，绝不回头！
  • 两点之间的路程，就是它们在三维空间里的直线距离。

你的任务： 帮 HKE 算出总飞行距离，并保留 三位小数。

(5个坐标点)
5
(乱乱的x, y, z坐标：)
2 2 2
1 1 1
4 4 4
3 3 3
5 5 5
📤 奇迹输出阵：
(从(1,1,1)飞到(2,2,2)...一直飞到(5,5,5)的总距离)
6.928

🧠 魔法道具箱 (核心思路：排序与三维距离公式)

这是一场高空接力赛！要算出总路程，我们要分两步走：

🥇 第一步：整理航线 (排序)。因为必须从低到高飞，所以我们要把所有的点按照高度 z 从小到大排好队！

🥈 第二步：计算距离 (欧几里得公式)。在立体的三维世界里，两颗星星之间的距离怎么算呢？

🌟 三维直线距离魔法公式 🌟

$$ \text{距离} = \sqrt{(x_2-x_1)^2 + (y_2-y_1)^2 + (z_2-z_1)^2} $$
例如样例中的起飞过程：
高度 1
(1,1,1)
✈️ 飞！算距离！
高度 2
(2,2,2)
高度 2
(2,2,2)
✈️ 飞！算距离！
高度 3
(3,3,3)
把每一次“✈️ 飞”的距离加起来，就是总距离啦！

📘 C++ 魔法秘籍 (知识点)

  • 📦 自定义超级背包 struct Point： 以前我们的背包只能装一个数，现在我们需要一个背包同时装下 x, y, z 三个坐标！
  • 🪄 教精灵怎么排队 compareZ： 电脑不知道什么是“按高度排队”，我们需要写一个小裁判函数，告诉它：“比 z 的大小，z 小的站前面！”
  • 📏 精准开方 sqrt()： 在 <cmath> 工具箱里，这个魔法可以算出平方根。
  • ✂️ 裁切小数点 fixed << setprecision(3)： 放在 cout 里面，可以让输出的数字乖乖地保留 3 位小数。

💻 C++ 终极法术 (代码详解)

#include <iostream>
#include <vector>
#include <algorithm>
#include <iomanip>  // ✂️ 小数点裁切工具箱
#include <cmath>    // 📏 开方魔法工具箱
using namespace std;

// 🎒 制作“坐标点”专属超级背包
struct Point {
    int x, y, z;
};

// 🧑‍⚖️ 裁判函数：告诉电脑，谁的 z 小谁就排前面
bool compareZ(const Point &a, const Point &b) {
    return a.z < b.z;
}

int main() {
    // ⚡ 加速咒语
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N;
    cin >> N; // 🖥️ 电脑问：有多少个坐标点？

    vector<Point> points(N);
    for (int i = 0; i < N; ++i) {
        cin >> points[i].x >> points[i].y >> points[i].z;
    }

    // 🪄 开始排队啦！用我们自己写的 compareZ 当规则
    sort(points.begin(), points.end(), compareZ);

    double total = 0.0; // 📏 准备一个带小数点的计步器，初始为 0

    // ✈️ 开始飞行！从第 2 个点 (下标1) 开始，每次和前一个点算距离
    for (int i = 1; i < N; ++i) {
        // 算出 x, y, z 各自的差距 (用 long long 防爆)
        long long dx = points[i].x - points[i-1].x;
        long long dy = points[i].y - points[i-1].y;
        long long dz = points[i].z - points[i-1].z;
        
        // 🌟 三维距离魔法公式：平方相加再开方！
        total += sqrt(dx*dx + dy*dy + dz*dz);
    }

    // 🎉 输出结果，固定保留 3 位小数！
    cout << fixed << setprecision(3) << total << '\n';

    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • 📦 小包裹 Tuple (元组)： 在 Python 里，我们可以把 (x, y, z) 像包糖果一样包在一起放进列表，这就是元组。
  • 🪄 隐形裁判 lambda： sort(key=lambda p: p[2]) 是一句超级炫酷的魔法。它的意思是：“排队的时候，每个人都只看包裹里第 3 个位置（下标为 2 的那个数，也就是 z）的大小！”
  • ✂️ 完美切片 f"{total:.3f}"： 这叫 f-string 魔法，.3f 代表把小数点后面的数字稳稳地裁切成 3 位。

💻 Python 终极法术 (代码详解)

import sys # 召唤高级工具箱！
import math # 📏 算距离要用的数学工具箱

def main():
    # 🌪️ 用超级吸尘器一口气吸入所有的数字
    data = sys.stdin.read().strip().split()
    if not data:
        return

    n = int(data[0]) # 点的数量
    points = []      # 准备一个空列表装所有的坐标小包裹
    
    idx = 1 # 阅读指针，从第1个位置开始读坐标
    for _ in range(n):
        x = int(data[idx])
        y = int(data[idx+1])
        z = int(data[idx+2])
        # 📦 把 x, y, z 包成一个小包裹，塞进列表里
        points.append((x, y, z))
        idx += 3 # 读完三个数，指针往后走 3 步

    # 🪄 排队魔法：告诉它按包裹里的第 3 个数 (也就是 z，下标是 2) 排序！
    points.sort(key=lambda p: p[2])

    total = 0.0 # 📏 准备一个计步器

    # ✈️ 开始飞行！从第 2 个点 (下标1) 开始，计算和前一个点的距离
    for i in range(1, n):
        # 拆开前一个点和当前点的小包裹
        x1, y1, z1 = points[i-1]
        x2, y2, z2 = points[i]
        
        # 算出各方向的差距
        dx = x2 - x1
        dy = y2 - y1
        dz = z2 - z1
        
        # 🌟 三维距离魔法公式：平方相加再用 math.sqrt 开方！
        total += math.sqrt(dx*dx + dy*dy + dz*dz)

    # 🎉 喊出总距离，f"{...:.3f}" 保证只显示 3 位小数！
    print(f"{total:.3f}")

if __name__ == "__main__":
    main() # 🚀 启动魔法！

🌟 重点复习小贴士

  • 💡 千万别用粗暴的三次方再开方！ 公式里面是 平方相加 (dx*dx + dy*dy + dz*dz)。这其实是勾股定理在三维世界的升级版哦！
  • 💡 为什么 C++ 里的 dx 要用 long long？ 坐标最大可能非常大，比如 10000。如果直接 dx * dx，在相加的时候可能会超出普通 int 能承受的极限爆炸掉！用 long long 绝对安全！

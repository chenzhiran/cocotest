📜 任务清单 (题目描述)

天梯一共有 N 级，你站在第 0 级（平地）。

每次你可以向上跳 1 到 K 级。只有恰好落在第 N 级，大门才会开启。

计算任务： 数一数一共有多少种不同的跳法！因为答案可能大得惊人，你需要把最终结果对 100003 取模后输出。

📥 魔法输入阵：
(目标 N=5 级台阶，每次最多跳 K=2 级)
5 2
📤 奇迹输出阵：
(一共有 8 种跳法)
8

🧠 魔法道具箱 (核心图解：递推与滑动窗口)

这道题是经典的动态规划 (DP)。想知道到达第 i 级的方法数，只要知道它前面 K 级的方法数，然后把它们全加起来就行了！

dp[i] = dp[i-1] + dp[i-2] + ... + dp[i-K]

✨ O(N) 的飞跃：滑动窗口求和法

如果每次都用 for 循环往前找 K 个数相加，当 N 和 K 都很大时会超时。我们可以准备一个神奇的滑动窗口，它只框住最近的 K 个数。每往前走一步，就把新产生的数加进窗口，把最旧的数踢出窗口，这样加法永远只需做一步！

🏃 窗口滑动和 S = 2 + 3 = 5 (新 dp 诞生！)
下一步：窗口会向右滑动，加入 dp，踢出 dp。
S = (当前S + 新来的dp) - 踢走的dp = (5 + 5) - 2 = 8。那就是 dp 的值啦！

📘 C++ 魔法秘籍 (知识点)

  • 🛡️ 取模的防护罩： 每次做加法或减法后都要立刻对 MOD 取模。特别是在做减法踢出窗口数字时，可能会产生负数！在 C++ 中，负数取模还是负数，所以我们要写成 (sum - dp[i - 1 - K] + MOD) % MOD，先加上一个 MOD 确保它是正的。
  • 📦 窗口和变量： 使用 long long sum = 0 来维护当前窗口内的总和。虽然取过模，但在加法瞬间可能会超出普通的 int 范围，用大容器更安全。

💻 C++ 终极法术 (代码详解)

#include <iostream>
#include <vector>
using namespace std;

const int MOD = 100003;

int main() {
    int N, K;
    cin >> N >> K;
    
    // 创建阶梯数组，大小为 N+1
    vector<int> dp(N + 1);
    dp[0] = 1; // 🏁 站着不动也是一种姿势 (初始状态)
    
    long long sum = 0; // 维护最近 K 个 dp 值的和 (滑动窗口)
    
    for (int i = 1; i <= N; ++i) {
        // ➕ 第一步：把上一个台阶的方法数加入窗口
        sum = (sum + dp[i - 1]) % MOD;          
        
        // ➖ 第二步：如果窗口“撑破”了 K 的大小，把最远的那一级踢出去
        if (i - 1 - K >= 0) {                    
            // 注意 C++ 中负数取模的技巧：先加 MOD 再取模
            sum = (sum - dp[i - 1 - K] + MOD) % MOD;
        }
        
        // ✨ 第三步：当前窗口的总和，就是跳到这一级的方法数！
        dp[i] = sum;                              
    }
    
    // 🏆 输出第 N 级的方法数
    cout << dp[N] << endl;
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • 🧮 超省心的取模魔法： 在计算踢出窗口的数据时 s = (s - dp[i - 1 - K]) % MOD，即使括号里减出了负数也不用怕！Python 底层的取模运算 % 会自动将负数处理成正确的正数余数，这比 C++ 省事多了！
  • 🚀 滑动窗口的魅力： 传统的 DP 解法会在外层 for i 里面嵌套一个 for j 来往回倒推 K 步。用了单变量 s 充当滑动窗口后，内层循环直接消失，算法速度直线起飞！

💻 Python 终极法术 (代码详解)

MOD = 100003

def main():
    # 读入阶梯数 N 和 最大步数 K
    N, K = map(int, input().split())
    
    # 创建阶梯列表，长度 N+1，全部初始化为 0
    dp = [0] * (N + 1)
    dp[0] = 1  # 🏁 还没出发时，站在原地就是 1 种方法
    
    s = 0 # s 用来记录滑动窗口内的所有方法数之和
    
    for i in range(1, N + 1):
        # ➕ 刚迈出新的一步，把身后紧挨着的那一级方法数加进窗口
        s = (s + dp[i - 1]) % MOD
        
        # ➖ 检查窗口是不是太长了？如果超过了最大步数 K，要把最旧的剔除
        if i - 1 - K >= 0:
            # Python 对负数取模会自动处理成正的范围内，极其聪明
            s = (s - dp[i - 1 - K]) % MOD   
            
        # ✨ 这个窗口和，就是登顶当前这级台阶的最终方法数！
        dp[i] = s
        
    # 🏆 终于登顶啦！输出第 N 级的魔法密码
    print(dp[N])

if __name__ == "__main__":
    main()

💡 技巧和重点总结 (划重点啦)

  • 🌟 灵魂起跑线： 递推第一步永远是想好初始状态。在这里，还没有跨越任何台阶的 dp = 1 是唯一正确的火种，没有它，后面的所有数都会是 0 乘 0！
  • 🌟 滑动窗口的艺术： 把 O(N*K) 的嵌套循环降维打击成 O(N) 的线性时间。你不需要每次都去重新点算过去 K 步的总和，只需要“一进一出”，维护一个动态的包袱。
  • 🌟 模运算的负数陷阱： 记住这个黄金法则 —— 在含有取模运算的题目中，凡是有减法的地方，一定要留心负数情况！C++ 选手要用 (A - B + MOD) % MOD，这是保护程序不出乱码的护身符。

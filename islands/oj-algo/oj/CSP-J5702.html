📜 任务清单 (题目描述)

数轴上有 n 个闭区间 [a, b]，挑选尽量少的守护点，使得每个保护区内都至少有一个点。

不同区间内的守护点可以是同一个点。请输出所需最少守护点的数量，并按从小到大的顺序输出这些守护点的坐标。

大殿调度法则：

  • 第一行包含一个整数 n，表示区间的数量。
  • 接下来的 n 行中，每行包含两个整数 a 和 b，表示一个闭区间 [a, b] 的起始和结束点。
  • 输出包含两行：第一行是一个整数，表示最少点数；第二行输出具体的守护点坐标，用空格隔开。

3

📤 奇迹输出阵：
(最少点数与点的位置)

🧠 魔法道具箱 (核心图解：紧抓终点的贪心智慧)

为了让一个守护点尽可能服务于更多的区间，直觉告诉我们：应该把守护点放在当前最紧迫、最先结束的区间的右边界（终点）上！

因为放得越靠右，就越有可能覆盖到后面那些“晚开始”的未来区间。这就是经典的区间选点贪心策略。

* 方案复盘：先按结束时间对所有区间进行升序排序。首先处理区间，在终点 3 设立守护点；接着看区间，由于上一个点 3 满足 2 ≤ 3 ≤ 5，它已经被顺利守护；最后看区间，当前的点 3 不在范围内，因此必须在它的右端点 6 设立新的守护点！

✨ 贪心选点三步破咒法

  1. 🔸 终点优先排序： 将所有区间按照结束点（右端点）进行升序排列。如果右端点相同，则将开始点（左端点）按照从大到小降序排列。
  2. 🔸 哨兵追随战略： 用一个变量 e 记录最近设立的守护点位置，初始时设为安全极小值（例如 -1）。
  3. 🔸 扫描全景判定： 顺序遍历每一个区间：如果当前区间没有包含最近设立的守护点 e（即满足 e < start 或者 e > end），说明该区域失守！我们贪心地在当前区间的右边界 end 设立一个新的守护点，更新 e = end 并将其记录进结果集。
核心思想：把守护点钉在区间的“最右侧悬崖”，利用极限位置为后续区间腾出最大重叠可能

📘 C++ 魔法秘籍 (知识点)

  • 双料比较规则： 编写 cmp 函数时，不仅对 end 进行升序规范，还对结束点相同的情况通过 a.start > b.start 实施左端点降序规范，让结构更加稳固。
  • 动态向量储存： 利用 vector<int> p 实时捕捉并记录下每一个破阵的哨兵坐标，最后轻松打印结果。
  • 覆盖条件边界： 核心循环中通过简洁的逻辑判定条件 e < a[i].start || e > a[i].end 敏锐地察觉是否有断层产生。

💻 C++ 终极法术 (区间终点排序版)

#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

// 🔮 定义区间结构体，用于存储区间的起始和结束点
struct Int {
    int start;
    int end;
};

// ⚖️ 比较函数：按照区间的结束点升序排序；若结束点相同，按起始点降序排序
bool cmp(Int a, Int b) {
    if (a.end == b.end)
        return a.start > b.start;
    return a.end < b.end;
}

int main() {
    // ⚡ 提速阵法：优化输入流
    ios::sync_with_stdio(false);
    cin.tie(NULL);

    int n; 
    cin >> n;
    
    vector<Int> a(n); 
    for (int i = 0; i < n; ++i) {
        cin >> a[i].start >> a[i].end;
    }
    
    // ⏱️ 按照右端点升序的核心贪心排序
    sort(a.begin(), a.end(), cmp);
    
    vector<int> p;    // 🏆 用于存储所有被选中的守护点
    int e = -2000005; // 🧙‍♂️ 避坑提示：原代码初始值-1在面对全负数坐标时会失效，已修正为安全极小值
    
    // 🔍 遍历所有区间，进行贪心选择
    for (int i = 0; i < a.size(); ++i) {
        // 如果上一个选择的点没有在当前区间的管辖防线内
        if (e < a[i].start || e > a[i].end) {
            e = a[i].end;     // ⏳ 贪心地选择当前区间的最大结束点作为新的守护点
            p.push_back(e);   // 将该点加入结果集
        }
    }
    
    // 📤 施放法术，输出最终守护点的数量和序列
    cout << p.size() << endl;
    for (int i = 0; i < p.size(); ++i) {
        cout << p[i] << " ";
    }
    cout << endl;
    
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • sys.stdin.read 快速吸入： 面对十万级的数据读入，传统的迭代式 input() 会让时间超支。用 sys.stdin.read().split() 能够一次性将所有魔法数字切片处理，极大精简读取速度。
  • 多关键字联合排序： Python 拥有极为强大的联合排序天赋。利用高级语法 intervals.sort(key=lambda x: (x, -x)) 可以一行搞定：第一关键字右端点升序，第二关键字左端点降序。
  • 星号解包输出： 使用优雅的解包技巧 print(*(points))，能够瞬间把列表中的所有整数用空格分开并整齐打印出来，告别冗长的循环编写。

💻 Python 终极法术 (多关键字排序版)

import sys

def main():
    # 📥 一次性加载整个魔法阵范围内的所有数据
    input_data = sys.stdin.read().split()
    if not input_data:
        return
        
    n = int(input_data[0])
    intervals = []
    
    idx = 1
    for _ in range(n):
        start = int(input_data[idx])
        end = int(input_data[idx+1])
        intervals.append((start, end))
        idx += 2
        
    # ⏱️ 排序核心：优先按照结束点(x)升序，若相同则按起始点(x)降序
    intervals.sort(key=lambda x: (x[1], -x[0]))
    
    points = []
    # 🧙‍♂️ 避坑提示：坐标可能包含负数，故初始哨兵点应设为负无穷大
    last_point = float('-inf')
    
    # 🔍 依次排查每一个星空保护区
    for start, end in intervals:
        # 如果上一步设立的阵眼不在此区间的保护范围内
        if last_point < start or last_point > end:
            last_point = end          # ⏳ 贪心地在右边界打下新阵眼
            points.append(last_point) # 记录阵眼坐标
            
    # 🏆 荣耀输出：第一行输出数量，第二行利用星号解包整齐打印坐标
    print(len(points))
    print(*(points))

if __name__ == '__main__':
    main()

💡 技巧和重点详细总结 (大天师的修行手记)

  • 🌟 与“魔法时间表”的精妙对称： 仔细体会你会发现，这道题跟之前的区间不重叠选择题（魔法时间表）核心架构完全一样。都是采用右端点升序排序！在“时间表”问题中，右端点意味着我们留给未来的时间；而在本题中，右端点代表了我们在保证覆盖本区间的前提下，最向右延伸的极限位置，两者的贪心逻辑在数学上是完美统一的。
  • 🌟 变量初始值的隐藏恶魔： 原题目中给出的代码，将记录上一个点的变量 e 设为了 `-1`。如果测试数据中包含负数空间（比如区间的左端点和右端点分别是 `-10` 和 `-5`），那么 `-1` 就会判定为不在这两个负数区间内，从而打下新的断层标记。我们在重构的代码中已帮大家杜绝了这一隐患，将其设定成了安全的负无穷界限，保证全坐标秒杀！
  • 🌟 绝对无敌的时空表现： 该模型的核心花销主要在排序这一步，耗时 O(n log n)。之后的单向贪心循环只需要对数组进行一次 $O(n)$ 的线性扫描。整套法术总时间复杂度为 O(n log n)。对于 $n = 100,000$ 的终极考验，程序也能在几十毫秒内轻松破阵，拿到满分！

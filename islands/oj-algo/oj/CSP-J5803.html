📜 任务清单 (题目描述)

给定一个包含 n 幅画的数组，每幅画由一个特定画家绘制，画家的编号是 1 到 m。

你的任务是找到一个连续的画作欣赏区间 [a, b]，使得该区间能够完美达成以下三大契律：

  • 全员收纳： 区间内必须包含全部 m 位画家的作品，至少各有一幅。
  • 开销最短： 区间的长度 b - a + 1 必须达到全局最小（最省钱）。
  • 起点优先： 如果存在多个最短长度相同的区间，必须选择起点 a 最小（最靠左）的那一组。

大殿输入与输出规则：

  • 第一行：包含两个整数 n（画作总数）和 m（画家的数量），用空格隔开。
  • 第二行：包含 n 个整数，表示从左到右每幅画对应的画家编号。
  • 输出仅一行两个整数，分别表示满足最优条件区间的起点和终点位置（位置坐标从 1 开始计数）。
📥 魔法输入阵：
(12幅画，共5位画家)

2 5 3 1 3 2 4 1 1 5 4 3
📤 奇迹输出阵：

数据规模提示：

  • 1 < n < 1,000,000 (画作多达百万级，传统的平方级双重循环必定爆时无解！)
  • 1 < a_i ≤ m < 2000 (画家总数适中)

🧠 魔法道具箱 (核心图解：像毛毛虫一样爬行的滑动窗口)

* 贪心策略：右指针 R 负责开拓疆土，遇到新画家就将种类数加 1。当窗口内的画家种类数刚好集齐了 m 种时，我们就果断让左指针 L 往右收缩，以此来“挤掉”左侧多余的画作，直到种类数面临减少。在整个贪心收缩的过程中，顺手捕捉下来的最短跨度就是我们要的最终解！

✨ 滑动窗口四步破咒法

  1. 🔸 右端大开拓： 右指针 R 一路向右探索，每当新加入一幅画，就在计数数组里给该画家的作品数加 1。如果是第一次见到该画家，则种类计数器加 1。
  2. 🔸 齐聚刹车判定： 一旦发现种类计数器正好等于 m，说明此时窗口里已经凑齐了所有画家的作品，当前的 [L, R] 区间是一个合法解。
  3. 🔸 左端疯狂压榨： 在满足全员收纳的前提下，我们尝试让左指针 L 往右移动。如果左端点的那幅画在窗口内部还有其他重复的备份（即 count > 1），那这幅画就是冗余的，直接剔除它并让 L 前进。重复此操作，直到左端点的那幅画只剩独苗。
  4. 🔸 刷新备忘录： 此时的窗口是当前右端点下的“最极致精简区间”。我们将它的长度 R - L + 1 与全局最短跨度进行比对，若更短则刷新纪录。之后继续推动右端点。
核心思想：右指针不回头地开疆拓土，左指针随时就地疯狂收缩，确保每一步都是极致高效的线性盘点！

📘 C++ 魔法秘籍 (知识点)

  • 极速 scanf 读入： 面对百万量级的数据规模，使用标准的 cin 会导致输入流耗时过长。换用传统的 scanf 可以在几毫秒内瞬间完成巨量数据的加载。
  • 频次打标数组： 开辟一个大小为 MAXM 的整型计数数组，利用 $O(1)$ 的开销实时追踪和控制当前窗口中每位画家的作品备份数量。
  • 1-Based 坐标映射： 算法内部采用计算机习惯的 0 到 N-1 进行运算，在最终输出结果时，大方地在左右端点下标上各加上 1，完美契合题目要求。

💻 C++ 终极法术 (双指针精简维护版)

#include <cstdio>
#include <algorithm>
using namespace std;

const int MAXN = 1000005;  // 百万级最大画作数量空间
const int MAXM = 2005;     // 最大画家数量空间

int m[MAXM] = {0};         // 画家计数数组：记录当前窗口内每位画家的作品件数
int num = 0;               // 计数器：记录当前窗口内的画家种类数
int n[MAXN];               // 画作长轴数组：存储每幅画对应的画家编号

int main() {
    int N, M;              // N: 画作数量, M: 画家数量
    if (scanf("%d%d", &N, &M) == EOF) return 0;
    
    for (int i = 0; i < N; ++i) {
        scanf("%d", &n[i]);
    }
    
    int L = 0, R = -1;      // 滑动窗口的左右端点指针
    int ansL = 0, ansR = N; // 🏆 记录全局最优区间的左右端点
    int i = 0;              // 遍历右端点的移动索引
    
    // 🔄 第一阶段：持续右移右端点，直到窗口内第一次集齐所有 M 位画家的作品
    while (i < N && num != M) {
        if (m[n[i]] == 0) {
            num++;          // 🌟 发现新画家，种类数加 1
        }
        m[n[i]]++;          // 该画家的作品备份数加 1
        R++;
        i++;
    }
    
    // 📉 第二阶段：尝试就地收缩左端点，挤掉左侧那些有重复备份的多余画作
    while (m[n[L]] > 1) {
        m[n[L]]--;          // 减少左端点画家的计数值
        L++;                // 左指针右移收缩
    }
    
    // 备忘录记录下这第一组达标的合法区间位置
    ansL = L;
    ansR = R;
    
    // 🔄 第三阶段：遍历剩余画作，继续维护和寻找更短的黄金区间
    while (i < N) {
        m[n[i]]++;          // 推动右端点接纳新画作并增加计数
        R++;
        i++;
        
        // 每次右端点走一步，左端点就疯狂跟进收缩到极致
        while (m[n[L]] > 1) {
            m[n[L]]--;
            L++;
        }
        
        // 💡 贪心比对：如果当前挤压出来的区间跨度严格短于之前的记录
        if (ansR - ansL > R - L) {
            ansR = R;       // 刷新最优纪录的右端点
            ansL = L;       // 刷新最优纪录的左端点
        }
    }
    
    // 🏆 输出最终结果，注意题目要求从 1 开始计数，所以输出加 1
    printf("%d %d\n", ansL + 1, ansR + 1);
    
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • sys.stdin.read 爆发快读： 面对百万巨量的数据行，切忌用连续的普通 input()。运用 sys.stdin.read().split() 将全部数据一次性抓进切片大数组，处理速度直接飙升百倍！
  • 高效哈希频次控制： 虽然可以用字典，但由于本题画家的编号范围确定在 2000 以内，直接利用固定的基础列表 * (m + 1) 充当计数池，其底层的连续内存读取效率远远高于哈希表。
  • 单层优雅大循环： 用一个极简的 for 循环让右指针不断步进，内部配合 while 随时收缩左边界，逻辑条理清晰，执行非常丝滑。

💻 Python 终极法术 (全自动迭代快读版)

import sys

def main():
    # 📥 爆发快读法：一口气抓取战场内所有的输入数字片段并进行分块
    input_data = sys.stdin.read().split()
    if not input_data:
        return
        
    n = int(input_data[0])  # 画作总数
    m = int(input_data[1])  # 画家总数
    
    # 📐 将接下来的所有画家编号整齐打包成整型列表
    paintings = [int(x) for x in input_data[2:n+2]]
    
    painter_counts = [0] * (m + 1)  # 🔮 频次计数列表，记录窗口内每位画家的作品数
    distinct_painters = 0          # 计数器：记录窗口内已集齐的画家种类数
    
    l = 0
    ans_l, ans_r = 0, n            # 🏆 记录全局最优区间的左右坐标边界
    min_len = n + 1                # 初始化最短跨度为无穷大
    
    # 🔍 让右指针 r 扮演开拓先锋，一路向右推移
    for r in range(n):
        p = paintings[r]
        if painter_counts[p] == 0:
            distinct_painters += 1  # 🌟 发现新面孔，种类数加 1
        painter_counts[p] += 1      # 该画家的作品数叠加
        
        # 💥 贪心连环大收缩：只要当前窗口内已经完美集齐了 M 位画家的全部作品
        while distinct_painters == m:
            # 💡 判定当前窗口的长度是否刷新了最短记录
            if r - l + 1 < min_len:
                min_len = r - l + 1
                ans_l, ans_r = l, r # 迅速在备忘录上盖章留下最佳根据地
                
            # 尝试驱逐最左端的画作，让窗口左边界右移收缩
            left_p = paintings[l]
            painter_counts[left_p] -= 1
            if painter_counts[left_p] == 0:
                distinct_painters -= 1 # 🛑 独苗被赶跑了，种类数下降，打断收缩循环
            l += 1
            
    # 🏆 荣耀输出最优解。因为位置要求从 1 开始，所以切片下标统一加 1
    print(f"{ans_l + 1} {ans_r + 1}")

if __name__ == '__main__':
    main()

💡 技巧和重点详细总结 (精算调度官的破局心得)

  • 🌟 为什么滑动窗口能完美保障“起点最小”？： 题目有一条细节限制：“如果存在多个最短区间解，选择起点最小的那组”。在我们的滑动窗口推进逻辑中，我们是从左到右自然推进的。当遇到后面出现相同长度的合法区间时，我们的更新条件是 r - l + 1 < min_len（严格小于）。由于是严格小于才刷新，这就意味着后面那些“同样短但起点靠右”的并列区间根本没有机会篡位，从而天然、完美地锁定了最靠左、起点最小的先发优胜者！
  • 🌟 双重循环嵌套为什么没有超时？（复杂度分析）： 很多初学者看到 for 循环内部嵌套了一个 while 循环，就会倒吸一口凉气，下意识地觉得这是平方级 $O(n^2)$ 的慢速算法。完全不用担心！ 让我们用“均摊分析法”来看看双指针的奇妙路线：在整个大循环运行期间，右指针 r 从 0 走到了 n-1，一共前进了 n 步；而左指针 l 同样也是傲然向前，绝不回头，在整个程序里最多也仅仅移动了 n 步。也就是说，两幅画被扫描和处理的总次数是稳稳的 $2n$ 次！因此，整个算法的时间复杂度是无敌且优雅的纯线性 $O(n)$！
  • 🌟 零门槛拿下满分： 百万规模的数据在 $O(n)$ 的降维打击面前形同虚设。程序在几毫秒到几十毫秒内即可瞬间全通通关，毫无卡常压力，稳拿满分！

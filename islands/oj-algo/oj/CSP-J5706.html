📜 任务清单 (题目描述)

给定 C 头奶牛，每头奶牛有一个可以忍受的最小和最大阳光强度范围 [minSPF_i, maxSPF_i]。

有 L 种防晒霜，每种防晒霜有一个固定的阳光强度调节值 SPF_j 以及可使用次数 cover_j。

请你计算在这些条件下，最多可以让多少头奶牛成功涂上合适的防晒霜并享受阳光。

大殿调度法则：

  • 第一行包含两个整数：C（奶牛数量）和 L（防晒霜种类数量），用空格隔开。
  • 接下来的 C 行，每行包含两个整数 minSPF_i 和 maxSPF_i。
  • 接下来的 L 行，每行包含两个整数 SPF_j 和 cover_j。
  • 输出一个整数，表示最多可以享受晒太阳的奶牛数量。
📥 魔法输入阵：
(3头牛与2种防晒霜)
3 2
3 10
2 5
1 5
6 2
4 1
📤 奇迹输出阵：
(成功匹配的最大牛数)
2

数据规模提示：

  • 1 ≤ C ≤ 2500，1 ≤ L ≤ 2500
  • 1 ≤ minSPF_i, maxSPF_i, SPF_j ≤ 1000
  • 1 ≤ cover_j ≤ 1000

🧠 魔法道具箱 (核心图解：先满足最挑剔、最快被晒伤的牛)

这是一道非常精妙的二分图最大匹配/区间贪心问题。要让总数最多，策略是：谁的忍受上限（maxSPF）低，谁就更容易被晒伤，我们就先紧着谁来挑！

在为某一头奶牛选防晒霜时，我们要从符合条件的防晒霜里，优先挑SPF值最小的那个。把高密度的防晒霜留给后面上限更高的奶牛，这就是最优策略。

🧴 霜 SPF 6
3. 奶牛 (上限10)

* 方案复盘：先把奶牛按照最大阳光需求量升序排序，同时把防晒霜按照固定的SPF值从小到大排序。第一步处理奶牛，由于 SPF 4 在其范围内且还有数量，直接分配，成功！第二步处理奶牛，由于 SPF 4 数量减为0，继续往后看，发现 SPF 6 在其范围内，分配成功！至此，2头牛匹配成功，完美拿下解密！

✨ 贪心选霜三步破咒法

  1. 🔸 双路大排序： 将所有奶牛按照右端点（最大阳光强度）从小到大进行升序排列； 同时，将所有防晒霜按照固定的 SPF 防护值从小到大升序排列。
  2. 🔸 逐牛排查法： 从前到后遍历已经排好序的奶牛。对于每一头牛，都从头（也就是从 SPF 值最小的防晒霜）向后开始扫描。
  3. 🔸 首配即最优： 一旦发现某种防晒霜的 SPF 值满足当前奶牛的需求（即满足 SPF >= minSPF 并且 SPF <= maxSPF），且该种防晒霜的库存数量大于 0，说明找到了最完美的解！ 立刻让该防晒霜的库存减 1，计数器加 1，并直接打断内层循环，开心去服务下一头牛！
核心思想：把宝贵的高防晒系数留给后面的耐热大牛，给当前的防晒脆弱牛匹配刚刚好能用的最低配

📘 C++ 魔法秘籍 (知识点)

  • 双结构体精细解耦： 分别建立 cow 和 sunscreen 结构体，使得各自的属性清晰独立。
  • 标准快排组合拳： 编写 com 与 comp 两个自定义排序规则，完美驾驭算法中截然不同的两个排序维度。
  • 循环提阶刹车： 内部判定一旦达成，通过 break; 强行切断内部循环，不仅逻辑完全符合贪心准则，同时也阻断了多余的扫描开销。

💻 C++ 终极法术 (区间右端点排序版)

#include <cstdio>
#include <algorithm>
using namespace std;

int ans; // 🏆 记录最终可以开心地享受到晒太阳的奶牛数量

// 🔮 定义奶牛结构体，存储每头奶牛的阳光强度忍受范围
struct cow {
    int l; // 最小阳光需求
    int r; // 最大阳光需求
} ox;

// 🧴 定义防晒霜结构体，存储每种防晒霜的防护值和剩余库存数量
struct sunscreen {
    int n; // 防晒霜的固定阳光强度
    int s; // 该防晒霜可用的剩余次数
} bb;

// ⚖️ 奶牛比较函数：按照最大阳光强度的右端点升序排列（从小到大）
bool com(struct cow x, struct cow y) {
    return x.r < y.r;
}

// ⚖️ 防晒霜比较函数：按照防晒强度值升序排列（从小到大）
bool comp(struct sunscreen a, struct sunscreen b) {
    return a.n < b.n;
}

int main() {
    int n, m; // n表示奶牛总数，m表示防晒霜的种类数
    if (scanf("%d %d", &n, &m) == EOF) return 0;

    // 输入每头奶牛的阳光强度区间需求
    for (int i = 1; i <= n; i++) {
        scanf("%d %d", &ox[i].l, &ox[i].r);
    }

    // 输入每种防晒霜的固定效能与数量
    for (int i = 1; i <= m; i++) {
        scanf("%d %d", &bb[i].n, &bb[i].s);
    }

    // ⏱️ 施展贪心排序大法：奶牛看上限右端点，防晒霜看系数值
    sort(ox + 1, ox + 1 + n, com);
    sort(bb + 1, bb + m + 1, comp);

    // 🔍 遍历每一头奶牛，贪心地为它们搜索并分配合适的清凉霜
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= m; j++) {
            // 💡 贪心核心：如果当前防晒霜强度适合牛的需求，且库存尚在
            if (bb[j].n >= ox[i].l && bb[j].n <= ox[i].r && bb[j].s > 0) {
                bb[j].s--; // 库存减少一瓶
                ans++;     // 保护成功的数量加 1
                break;     // ✨ 贪心达成，直接退回外层去处理下一头更紧迫的牛
            }
        }
    }

    // 🏆 输出最终在最完美调配下成功享受阳光的奶牛数量
    printf("%d\n", ans);

    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • sys.stdin.read 极速吸入： 面对庞大的多行输入输入流，用自带的循环 input() 会非常耗时。利用一次性切片把所有魔法字符读进内存，效率直接拉满！
  • 字典结构的高效排序： Python 拥有最优雅的字典列表排序天赋。利用高级语法 cows.sort(key=lambda x: x['r'])，一句话就能实现按照任意指定键值的升序排列。
  • 极简对象遍历： 采用简洁的迭代风格，直接利用对字典对象的字段读写来代替生硬的下标数字，大大增强了代码的阅读流畅度。

💻 Python 终极法术 (多维字典排序版)

import sys

def main():
    # 📥 极速快读法：一口气抓取战场内所有的输入片段并进行分块
    input_data = sys.stdin.read().split()
    if not input_data:
        return
        
    c = int(input_data[0])  # 奶牛总数
    l = int(input_data[1])  # 防晒霜种类数
    
    idx = 2
    cows = []
    for _ in range(c):
        min_spf = int(input_data[idx])
        max_spf = int(input_data[idx+1])
        cows.append({'l': min_spf, 'r': max_spf})
        idx += 2
        
    sunscreens = []
    for _ in range(l):
        spf = int(input_data[idx])
        cover = int(input_data[idx+1])
        sunscreens.append({'n': spf, 's': cover})
        idx += 2
        
    # ⏱️ 贪心排序核心：奶牛按上限(r)升序，防晒霜按SPF效能(n)升序
    cows.sort(key=lambda x: x['r'])
    sunscreens.sort(key=lambda x: x['n'])
    
    ans = 0
    
    # 🔍 依次排查每一头对温度敏感的奶牛
    for cow in cows:
        # 从效能最小的防晒霜开始，贪心地往后试探
        for src in sunscreens:
            # 💡 匹配条件：防晒霜效能满足区间，且当前种类的库存充沛
            if src['n'] >= cow['l'] and src['n'] <= cow['r'] and src['s'] > 0:
                src['s'] -= 1  # 消耗一瓶防晒霜
                ans += 1       # 成功营救一头牛
                break          # ✨ 贪心成功，直接刹车，开始为下一头牛寻宝
                
    # 🏆 输出最终成果
    print(ans)

if __name__ == '__main__':
    main()

💡 技巧和重点详细总结 (海滩精算师的复盘笔记)

  • 🌟 核心逻辑的精巧证明： 为什么我们要按 maxSPF（右端点） 升序排序？因为右端点越小的奶牛，它面临的“可选项边界”就越窄，属于极其脆弱、极容易因防晒霜系数过高而被剥夺享受太阳权利的群体。如果我们先满足它，并且贪心地选择当前满足要求的最小值，就能为后面右端点更大、更耐热的奶牛腾出那些高防晒强度的防晒霜。这种“将局势掌控在最危险边缘”的做法就是贪心的灵魂。
  • 🌟 另一种异曲同工的解法： 很多大牛在做这道题时，也会采取按 minSPF（左端点）从大到小降序排列 的策略。在那种情况下，每次就必须贪心地挑选满足条件的 最大 maxSPF 防晒霜 来进行匹配。这两种贪心方向在组合数学上是完美对偶的，都能稳拿满分！
  • 🌟 复杂度表现与安全通关： 排序的耗时是 O(C log C + L log L)，由于两层嵌套循环的最大总开销仅仅为 O(C * L)。由于题目给出的顶格规模 C, L ≤ 2500，计算总量大概在 6,000,000 次以内，在 C++ 和 Python 中都能在几十毫秒内瞬间出结果，极其丝滑！

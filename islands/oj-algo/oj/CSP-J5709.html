🧙‍♂️ 欢迎来到翡翠森林的核心营地，伟大的自然守护者！
有一条长长的林间小道旁边建着许多居民小屋，空地被分成了 1 到 n 号小格子。
每位居民都向你许下了一个心愿：在区间 [b, e] 内至少要种上 t 棵树苗！
为了节省宝贵的树苗，我们需要规划一种重叠度最高的种植方案。
在满足所有居民苛刻要求的前提下，算算最少需要种下多少棵希望之树吧！

📜 任务清单 (题目描述)

小道一共有 n 个区域格子，编号为 1, 2, ..., n。每个区域格子内最多只能种 1 棵树。

一共有 h 位居民提出了他们的种植愿望。第 i 位居民会给出三个整数 b_i, e_i, t_i，表示他希望在闭区间 [b_i, e_i] 之间至少种上 t_i 棵树。

不同居民要求的区间极有可能发生重叠，请你求出能够同时满足所有人愿望时，所需树木的最少总数量。

森林入阵法则：

  • 第 1 行：一个整数 n，表示小道区域的总格子数。
  • 第 2 行：一个整数 h，表示提出要求的居民数量。
  • 第 3 行到第 h+2 行：每行三个整数 b_i, e_i, t_i，代表一个限定区间的起点、终点和最少树木数。
  • 输出仅一行一个整数，代表完全达成目标所需种下的最少树苗总数。
📥 魔法输入阵：
(5个格子与3组愿望)
5
3
1 3 2

📤 奇迹输出阵：
(全满达标的最少树木数)
3

数据范围轻度解析：

  • 1 ≤ n ≤ 30,000 (总格子数较长)
  • 1 ≤ h ≤ 5,000 (居民愿望数)
  • 1 ≤ b_i ≤ e_i ≤ n， 1 ≤ t_i ≤ e_i - b_i + 1

🧠 魔法道具箱 (核心图解：背靠右端点，往左开倒车)

这是一道经典的区间选点贪心扩展题。想要让后面的区间也能白嫖到我们在前面种的树，核心思想是：把树苗尽量种在当前区间的右侧边缘！

因为后面要审阅的区间，其右端点一定比当前区间的右端点更靠右。我们的树越往右种，就越有希望落入后续区间的怀抱中！

✨ 贪心绿化三步破咒法

  1. 🔸 右端升序大排序： 把所有居民的愿望区间，按照右端点（终点 e）从小到大进行升序排列。
  2. 🔸 旧树数量细盘点： 依次遍历排好序的区间。对于当前区间 [b, e]，先用一个循环统计一下在这个格子范围内，之前轮次已经种了多少棵树（记为 k）。
  3. 🔸 逆向补齐留未来： 如果已有树木数 k 小于所要求的 t，说明还需要再补种。我们启动倒车机制：从当前的终点 e 开始向左倒退扫描，遇到没有种树的格子就立刻种下一棵，并让 k 加 1、总计数器加 1。一旦 k 达到了 t 的标准，立刻打断补树循环，去审阅下一个愿望！
核心思想：终点越早越先处理，补数必从右侧倒填，将利益最大化地留给未来的区间

📘 C++ 魔法秘籍 (知识点)

  • 轻量级状态标记： 使用 bool used 布尔动态映射数组，能在 O(1) 的极速时间内判定或修改某个坐标格子上是否已经种了树苗。
  • 自定义结构体排序： 用 struct line 打包起始、终点和需求数，配合自定义比较函数 cmp 进行 std::sort 排序，代码清晰自然。
  • 倒序切入断点： 在补树阶段，通过控制变量循环 for (int j = a[i].e; j >= a[i].s; j--) 实现由右向左倒序切入，完美展现贪心精髓。

💻 C++ 终极法术 (区间右端点升序版)

#include <iostream>
#include <vector>
#include <algorithm>
#include <cstdio>

using namespace std;

// 🔮 定义结构体保存每个居民的要求：s代表起点，e代表终点，v代表需要种的树木数
struct line {
    int s, e, v;
} a;

int n, m, ans = 0;      // n为格子总数，m为居民数，ans为最终种树总数
bool used = {0}; // 🌿 标记布尔阵：记录某个位置是否已经种下了树木

// ⚖️ 比较函数：按照区间的右端点（结束位置）升序排列
bool cmp(line x, line y) {
    return x.e < y.e;
}

int main() {
    // 📥 读入格子总规模以及愿望总数量
    if (scanf("%d%d", &n, &m) == EOF) return 0;
    
    for (int i = 1; i <= m; i++) {
        scanf("%d%d%d", &a[i].s, &a[i].e, &a[i].v);
    }
    
    // ⏱️ 核心贪心第一步：按结束位置由早到晚进行排序
    sort(a + 1, a + m + 1, cmp);

    // 🔍 逐个审阅居民的区间愿望
    for (int i = 1; i <= m; i++) {
        int k = 0; // 统计当前区间内之前已经种了多少棵树
        
        for (int j = a[i].s; j <= a[i].e; j++) {
            if (used[j]) k++;
        }
        
        // 💡 贪心判定：如果已经种下的树木数足够，直接跳过，不用浪费新树苗
        if (k >= a[i].v) continue;

        // 💥 倒车补树：不满足要求，从区间的最右端点开始向左侧空位进行补种
        for (int j = a[i].e; j >= a[i].s; j--) {
            if (!used[j]) {
                used[j] = true; // 在空位上种下一棵崭新的树苗
                k++;            // 当前区间的树木计数加 1
                ans++;          // 王国总树苗消耗加 1
                
                // 🛑 只要数量一够，立刻刹车，绝不多种
                if (k == a[i].v) break; 
            }
        }
    }
    
    // 🏆 输出大功告成的最少树木总数答案
    printf("%d\n", ans);
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • sys.stdin.read 爆发快读： 面对输入项密集的矩阵，在 Python 中使用内置的一口气读取方法 sys.stdin.read().split() 转换为一维大列表，再用迭代器逐个提取，能省下可观的时间开销。
  • 优雅的 lambda 复合元组排序： 采用元组存储后，通过高级一句话语法 req.sort(key=lambda x: x) 即可飞速按照右边界索引 1 处的终点进行原地升序排队。
  • 倒序步长元模型： Python 的 range 函数支持负数步长！通过声明 range(e, s - 1, -1)，可以不费吹灰之力地实现从终点向起点的逆向扫描。

💻 Python 终极法术 (全自动迭代快读版)

import sys

def main():
    # 📥 施放降维吸收术，一口气将整场数据吞入一维切片列表中
    data = sys.stdin.read().strip().split()
    if not data:
        return
        
    it = iter(data)
    n = int(next(it)) # 区域格子总数
    m = int(next(it)) # 居民愿望数
    
    req = []
    for _ in range(m):
        s = int(next(it))
        e = int(next(it))
        v = int(next(it))
        req.append((s, e, v))

    # ⏱️ 贪心核心：按照每个愿望的右端点（结束点）进行原地升序排列
    req.sort(key=lambda x: x[1])

    # 🌿 初始化种植状态列表。因为格子编号是 1 到 n，我们开辟 n + 1 的空间更安全
    used = [False] * (n + 1)  
    ans = 0

    # 🔍 有条不紊地依次审阅每一份森林申请
    for s, e, v in req:
        # 统计当前 [s, e] 闭区间内过去已经种过多少棵树
        k = 0
        for j in range(s, e + 1):
            if used[j]:
                k += 1
                
        # 💡 如果已经自给自足，说明被前面的区间完美罩住了，直接放行
        if k >= v:
            continue

        # 💥 逆向倒开补齐：利用 -1 步长，从当前区间的右边界 e 一路向左撤退补种
        for j in range(e, s - 1, -1):
            if not used[j]:
                used[j] = True  # 在这个风水极佳的空地上种下新树
                k += 1         # 当前愿望的达标进度加 1
                ans += 1       # 总树苗消耗加 1
                
                # 🛑 进度拉满就立刻收手，把左侧宝贵的空位让给后面的缘分
                if k == v:
                    break

    # 🏆 打印输出最低成本下的总绿化数量
    print(ans)

if __name__ == "__main__":
    main()

💡 技巧和重点详细总结 (守护者的复盘日记)

  • 🌟 贪心之所以正确的本质证明： 为什么我们非要按“右端点排序”，还要“从右往左”补树呢？因为在排序后，当前区间之后的所有尚未处理的愿望区间，它们的右端点全都在当前的右端点（e）或者更远的右边。如果我们把树苗种在尽可能靠右的位置，那么这棵树不仅满足了当前的区间，还具有极大的概率落入后续所有区间的覆盖范围内！这是一种让单次投入带来最大化未来复用的高级运筹学大智慧。
  • 🌟 进阶高端优化的展望： 对于这道题，本代码中使用的基础遍历统计和填补方法已经可以通过 100% 的全部数据。但如果格子规模 $n$ 从 30,000 膨胀到 300,000 以上，普通的单层循环就会面临时间上的考验。这时，算法大师们就会祭出树状数组（Fenwick Tree）来在对数时间内维护区间已种树木数量，并搭配并查集（Union-Find）实现一秒跳过已种位置、精准降落到左边下一个空位上的神技！感兴趣的小伙伴可以在通关后尝试用这些高级法宝重组代码哦！
  • 🌟 绝妙的时空运行表现： 排序的耗时是 O(h log h)，由于本算法的遍历上限被很好地约束在 O(h * (e - b))，在题目给出的规模内，C++ 仅用几毫秒即可全对秒杀，Python 也能在零点一秒内瞬间破阵，堪称完美的贪心解法！

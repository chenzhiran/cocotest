📜 任务清单 (题目描述)

在一个凸 n 边形中，通过不相交的对角线将其划分为若干个三角形。求不同的划分方案数目。

规律举例：

  • 三角形（3 边）：不需要对角线，本身就是 1 个三角形。方案数 = 1。
  • 四边形（4 边）：可以连一条对角线。方案数 = 2。
  • 五边形（5 边）：有 5 种不同的三角剖分方法。

计算任务： 输入一个正整数 n，输出凸 n 边形划分三角形的方案数。

📥 魔法输入阵：
(待切割的水晶是：四边形)
4
📤 奇迹输出阵：
(共有 2 种切割方案)
2

🧠 魔法道具箱 (核心图解：化整为零的切割术)

这是一个经典的“分治与递推”问题。为了不重复、不遗漏地计算，我们需要人为地“固定”多边形的一条边作为基准，看看以它为底，能搭出怎样的三角形。

👇 五边形 (n=5) 拆解的奥秘 👇
设底边为红色线。我们在多边形上任意选一个非底边的顶点，与底边连成一个黄色的主三角形。

这个三角形像一把刀，把剩下的图形切成了两部分：
左边剩下一个 j 边形，右边剩下一个 n - j + 1 边形。

✨ 状态转移方程式

假设 h[i] 表示凸 i 边形的三角剖分方案数。当我们选定一个主三角形后，左边多边形的方案数是 h[j]，右边多边形的方案数是 h[i-j+1]。因为左右是相互独立的，根据乘法原理，这种切法下的方案数就是 h[j] * h[i-j+1]。

只要我们把这个主三角形的顶点在剩下的各个点上遍历一次，把所有的乘积加起来，就得到了递推公式：

$$h[i]=\sum_{j=2}^{i-1}h[j]\times h[i-j+1]$$

(边界条件：h = 1 代表一条退化的线段，h = 1 代表三角形本身)

📘 C++ 魔法秘籍 (知识点)

  • ⚙️ 双重循环递推： 与斐波那契只需要看前两项不同，卡特兰数需要用到前面所有算过的项！外层循环 i 代表当前计算几边形，内层循环 j 代表主三角形切分出的左侧是几边形。
  • 🛡️ 容量保护： 卡特兰数的增长速度非常恐怖（如 h 就高达 4.7 亿）。为了防止整数溢出，养成使用 long long 存储 DP 数组的习惯，是一名合格魔法师的必修课！

💻 C++ 终极法术 (代码详解)

#include <iostream>
#include <vector>
using namespace std;

long long catalan(int n) {
    // 🏁 处理无法构成多边形的特殊情况
    if (n < 3) return 0;
    
    // h[i] 代表 i 边形的方案数
    vector<long long> h(n + 1, 0);
    
    // 初始化基础状态
    h[2] = 1; // 2 边形退化为线段，算 1 种基础面
    h[3] = 1; // 三角形本身只有 1 种
    
    // 🌀 外层循环：逐步推导更大的多边形 (从四边形到 n 边形)
    for (int i = 4; i <= n; i++) {
        // 内层循环：枚举主三角形顶点的位置
        // 左边切出 j 边形，右边切出 i-j+1 边形
        for (int j = 2; j <= i - 1; j++) {
            h[i] += h[j] * h[i - j + 1];
        }
    }
    
    return h[n];
}

int main() {
    int n;
    cin >> n;
    cout << catalan(n) << endl;
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • ✨ 自然语言般的循环： Python 中的 for j in range(2, i): 完美对应了数学公式中的 $\sum_{j=2}^{i-1}$，读起来就像在读算式一样顺畅。
  • ⚡ 大数无界： 这道题在评测系统里，如果输入超大的 n，C++ 即使用 unsigned long long 也会爆炸，但 Python 的整数可以无限延展，无需操心数据溢出！

💻 Python 终极法术 (代码详解)

def catalan(n):
    # 🏁 无法构成多边形的边界判断
    if n < 3:
        return 0
        
    # 创建 DP 列表，h[i] 表示 i 边形的方案数
    h = [0] * (n + 1)
    h[2] = 1  # 退化的线段，作为基础乘数
    h[3] = 1  # 三角形本身的方案数
    
    # 🌀 外层：计算 4 到 n 边形
    for i in range(4, n + 1):
        # 内层：枚举主三角形切分出的左右两块多边形的顶点数
        for j in range(2, i):
            h[i] += h[j] * h[i - j + 1]
            
    return h[n]

def main():
    n = int(input())
    print(catalan(n))

if __name__ == "__main__":
    main()

💡 技巧和重点总结 (划重点啦)

  • 🚨 避雷针：原代码勘误！ 在你提供的原 C++ 样例代码中，有一句 return n * Catalan(n - 1) / 2;。这是非常经典的陷阱！ 凑巧的是，这个错误公式在 n=3,4,5 时恰好能得出 1, 2, 5 这个正确结果。但当你测算 n=6 时，错误公式算出来是 15，而真实的卡特兰数应该是 14！在严谨的算法学习中，一定要用标准的卡特兰递推（或者 O(n) 的数学公式 $h[n]=h[n-1]\times\frac{4n-10}{n-1}$）！
  • 🌟 组合数学的王者： 卡特兰数（Catalan Number）是算法竞赛中出现频率最高的数列之一！除了“多边形划分”，它还统治着许多经典场景：括号匹配的合法序列数、出栈序列的总数、二叉树的形态总数等。看到这些题，请立刻回想起这组神秘数字：1, 1, 2, 5, 14, 42...
  • 🌟 分治递推的乘法原理： 当一个大问题被切成独立的两半（左边和右边相互不影响）时，总方案数等于左边的方案数 乘以 右边的方案数。这在很多动态规划区间 DP 问题中都是核心心法。

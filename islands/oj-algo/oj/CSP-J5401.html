📜 任务清单 (题目描述)

你面前有一块 N 行 M 列 的旗布，上面画满了 'W'(白)、'B'(蓝)、'R'(红) 的小方格。

合法的雪山三色旗必须满足：

  • 最上面连续几行全是 白色 (W)，至少 1 行。
  • 中间连续几行全是 蓝色 (B)，至少 1 行。
  • 最下面连续几行全是 红色 (R)，至少 1 行。

你的任务： 算出最少需要重新涂几个方格，才能让旗布变合法！

📥 魔法输入阵：
(4行5列的乱码旗布)
4 5
WRWRW
BWRWB
WRWRW
RWBWR
📤 奇迹输出阵：
(第1行涂白改2个，第2行涂蓝改3个，第3、4行涂红改6个。总共11个！)
11

🧠 魔法道具箱 (核心思路：魔法剪刀 + 穷举法)

这块旗布有 N 行。既然必须是“白-蓝-红”三段，我们就可以拿出 “魔法剪刀” ✂️，在旗布上横着剪两刀，把它分成上、中、下三部分！

💡 解题秘籍： 我们事先不知道剪在哪里最省颜料。那就把所有可能的剪法都试一遍！（也就是枚举所有的分界线）

假设我们这样剪 (第1行白，第2行蓝，第3,4行红)：
WRWRW
变纯白需要涂 2 块
✂️ 咔嚓！第一刀 ✂️
BWRWB
变纯蓝需要涂 3 块
✂️ 咔嚓！第二刀 ✂️
WRWRW
变纯红需要涂 3 块
RWBWR
变纯红需要涂 3 块
总代价 = 2 + 3 + 3 + 3 = 11块！
(把各种剪法的总代价都算出来，挑一个最小的就行啦！)

📘 C++ 魔法秘籍 (知识点)

  • 📝 预处理魔法表： 为了不每次都重新数格子，我们可以一开始就建 3 张表：costW、costB、costR，记录下每一行如果变成白色、蓝色、红色，分别需要改几笔。这叫“预处理”，能大大加快计算速度！
  • ♾️ 无穷大 INT_MAX： 因为我们要找“最小的代价”，一开始就把答案设置为无穷大 INT_MAX (在 <climits> 工具箱里)。只要找到比它小的，就把它替换掉！
  • ✂️ 双重循环剪刀： for(i) 代表白色的行数，for(j) 代表蓝色的行数。剩下的 k = N-i-j 就是红色的行数啦！

💻 C++ 终极法术 (代码详解)

#include <iostream>
#include <vector>
#include <string>
#include <climits> // ♾️ 无穷大魔法 INT_MAX 藏在这里！
using namespace std;

int main() {
    int N, M;
    cin >> N >> M;
    vector<string> grid(N);
    for (int i = 0; i < N; ++i) {
        cin >> grid[i]; // 读入旗布的样子
    }

    // 📝 预处理：算好每一行变成 W, B, R 各需要涂几块
    vector<int> costW(N, 0), costB(N, 0), costR(N, 0);
    for (int i = 0; i < N; ++i) {
        for (char c : grid[i]) {
            if (c != 'W') costW[i]++; // 不是白色的，涂白代价+1
            if (c != 'B') costB[i]++; // 不是蓝色的，涂蓝代价+1
            if (c != 'R') costR[i]++; // 不是红色的，涂红代价+1
        }
    }

    int ans = INT_MAX; // 🏆 最终答案先设为无穷大

    // ✂️ 开始尝试各种剪法！
    // i 是白色行的数量 (至少1行，最多给蓝色和红色留2行，所以 i <= N-2)
    for (int i = 1; i <= N - 2; ++i) {
        // j 是蓝色行的数量 (至少1行，最多给红色留1行，所以 j <= N-i-1)
        for (int j = 1; j <= N - i - 1; ++j) {
            
            int k = N - i - j; // k 就是剩下留给红色行的数量
            int total = 0;     // 记录这种剪法的总代价

            // 第 1 段：前 i 行涂白
            for (int r = 0; r < i; ++r) total += costW[r];
            // 第 2 段：接下来的 j 行涂蓝
            for (int r = i; r < i + j; ++r) total += costB[r];
            // 第 3 段：最后的 k 行涂红
            for (int r = i + j; r < N; ++r) total += costR[r];

            // 💡 如果这次的代价比之前记录的还小，就更新答案！
            if (total < ans) ans = total;
        }
    }

    // 🎉 输出最小的涂改数量
    cout << ans << endl;
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • 📝 预处理魔法： 如果每次尝试剪法都去数字符串里有几个 'W'，那太慢啦！先用 costW 列表把每一行变成白色的代价算出来，后面直接查表就行。
  • ♾️ 无穷大 float('inf')： 在 Python 里，想要一个比所有数字都大的初始值，就用 float('inf')，它就是无尽虚空般的大！
  • ✂️ 切片与求和 sum()： 我们把列表用切片切开，比如 costW[:i] 表示前 i 行涂白的代价，然后外面套一个 sum() 瞬间就能把它们加起来，代码写起来超级清爽！

💻 Python 终极法术 (代码详解)

def main():
    # 读入 N 和 M
    N, M = map(int, input().split())
    
    # 读入旗布的样子，strip() 用来去掉结尾看不见的回车符
    grid = [input().strip() for _ in range(N)]

    # 📝 预处理：算好每一行变成 白、蓝、红 分别需要涂几块
    costW = [0] * N
    costB = [0] * N
    costR = [0] * N

    for i in range(N):
        for ch in grid[i]:
            if ch != 'W': costW[i] += 1 # 不是白的，涂白代价+1
            if ch != 'B': costB[i] += 1 # 不是蓝的，涂蓝代价+1
            if ch != 'R': costR[i] += 1 # 不是红的，涂红代价+1

    ans = float('inf') # 🏆 最终答案先设为正无穷大

    # ✂️ 开始尝试各种剪法！
    # i 是白色行的数量 (至少1行，最多给蓝色和红色留2行，所以到 N-1 为止)
    for i in range(1, N - 1):  
        # j 是蓝色行的数量 (至少1行，所以到 N-i 为止，保证红色 k 至少有1行)
        for j in range(1, N - i):  
            k = N - i - j # k 就是剩下留给红色的行数
            
            # 🧮 神奇的切片求和！
            # sum(costW[:i]) 把前 i 行涂白的代价加起来
            # sum(costB[i : i+j]) 把接下来的 j 行涂蓝的代价加起来
            # sum(costR[i+j :]) 把剩下的行涂红的代价加起来
            total = sum(costW[:i]) + sum(costB[i : i+j]) + sum(costR[i+j:])
            
            # 💡 如果这次的代价比之前记录的还小，就更新答案！
            if total < ans:
                ans = total

    # 🎉 输出最小的涂改数量
    print(ans)

if __name__ == "__main__":
    main() # 🚀 启动魔法！

🌟 重点复习小贴士

  • 💡 千万别漏掉“至少一行”的规则！ 我们在枚举 i 和 j 的时候，边界非常重要。必须保证白、蓝、红每种颜色都至少占据一行，这就是为什么循环的上界是 N-2 和 N-i-1 的原因啦！
  • 💡 算法的时间复杂度： 我们枚举了两次 (双重循环)，算起来大约是 $O(N^2)$。再加上预处理和计算的时间，总共是 $O(N^2 \cdot M)$。题目里 $N$ 和 $M$ 最大只有 50，所以电脑一眨眼（不到千分之一秒）就能算完！

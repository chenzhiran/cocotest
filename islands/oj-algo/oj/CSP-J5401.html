📜 任务清单 (题目描述)

你面前有一块由 N 行 M 列小方格拼成的旗布。每个方格现在的颜色要么是白(W)，要么是蓝(B)，要么是红(R)。

雪山法则规定，合法的三色旗必须是这样的：

  • 顶部若干行（至少一行）全部是白色 (W) ☁️
  • 中间若干行（至少一行）全部是蓝色 (B) 💧
  • 底部若干行（至少一行）全部是红色 (R) 🍎

你可以给任意格子重新上色盖住原来的颜色。你的任务是：算出最少要涂改多少块格子，才能让这面旗帜重新变得合格！

📥 魔法输入阵 (输入样例)：
(4行5列的乱码旗布)
4 5
WRWRW
BWRWB
WRWRW
RWBWR
📤 奇迹输出阵 (输出样例)：
(需要重涂的最少格子数)
11

🧠 魔法道具箱 (核心思路：枚举分割点)

其实这面旗帜就像是一个大蛋糕，我们需要用魔法剪刀 ✂️ 把它横着切成三块（白、蓝、红）。因为 N 比较小（最多才 50 行），我们可以尝试所有的切法，看看哪种切法需要修改的格子最少！

✨ 施法步骤：

  1. 预知代价： 我们先悄悄数一数，如果把第 i 行强行全部变成白色、蓝色或红色，分别需要改几个格子？记录在魔法账本 costW, costB, costR 上。
  2. 挥舞剪刀 (双重循环)： 我们决定：前 i 行涂白，接下来 j 行涂蓝，剩下的 k 行涂红。
    不断调整 i 和 j 的大小（保证红、白、蓝都至少有 1 行）。
  3. 计算总代价： 每切一刀，我们就去魔法账本上把这段区域的代价加起来，找出所有切法里总代价最小的那个！

📘 C++ 魔法秘籍 (知识点)

  • 📊 预处理思想： 与其每次切完再逐个格子去数字符，不如一开始就建好三个 vector<int> 记录每一行变成某种颜色的代价！这叫“磨刀不误砍柴工”。
  • ♾️ 初始无穷大 INT_MAX： 要找最小值，一开始我们要把答案设得非常非常大。召唤 #include <climits>，你就能得到整型最大值 INT_MAX。
  • 🔄 多重循环不迷路： 外层 for 控制白色行数 i，中层 for 控制蓝色行数 j，最底下的红色行数就是 N - i - j 啦！

💻 C++ 终极法术 (代码详解)

#include <iostream>
#include <vector>
#include <string>
#include <climits> // 召唤 INT_MAX 需要的头文件
using namespace std;

int main() {
    int N, M;
    cin >> N >> M;
    
    // 🗺️ 读取原始旗布
    vector<string> grid(N);
    for (int i = 0; i < N; ++i) {
        cin >> grid[i];
    }

    // 📝 预处理账本：算出每一行如果全部变成 白/蓝/红 需要涂改几个格子
    vector<int> costW(N, 0), costB(N, 0), costR(N, 0);
    for (int i = 0; i < N; ++i) {
        for (char c : grid[i]) {
            if (c != 'W') costW[i]++; // 如果不是白色，涂白代价+1
            if (c != 'B') costB[i]++; // 如果不是蓝色，涂蓝代价+1
            if (c != 'R') costR[i]++; // 如果不是红色，涂红代价+1
        }
    }

    int ans = INT_MAX; // 设为一个极大的值，准备打擂台寻找最小值
    
    // ✂️ 开始尝试各种切法！
    // i：顶部白色有多少行 (至少1行，至多N-2行，因为要给蓝红留位置)
    for (int i = 1; i <= N - 2; ++i) {
        // j：中间蓝色有多少行 (至少1行，至多剩下的行数再留1行给红色)
        for (int j = 1; j <= N - i - 1; ++j) {
            int k = N - i - j; // k：剩下给红色的行数
            int total = 0;     // 记录当前这种切法的总代价
            
            // ☁️ 前 i 行涂白
            for (int r = 0; r < i; ++r) total += costW[r];
            // 💧 中间 j 行涂蓝 (下标从 i 开始，到 i+j-1)
            for (int r = i; r < i + j; ++r) total += costB[r];
            // 🍎 后 k 行涂红 (下标从 i+j 开始，到末尾)
            for (int r = i + j; r < N; ++r) total += costR[r];
            
            // 🏆 打擂台，保留最小的代价！
            if (total < ans) ans = total;
        }
    }

    cout << ans << endl; // 输出最终结果，大功告成！
    return 0;
}

🌟 C++ 重点复习小贴士

  • 💡 时间复杂度： 这个算法有 3 层嵌套的循环（枚举 i, j 再加上内部对 r 的求和），大约是 O(N³)。因为 N ≤ 50，最大也就 50 × 50 × 50 = 125,000 次运算，对于现代计算机来说简直是眨眼间的功夫，完全不会超时！
  • 💡 进阶思考（前缀和）： 题目描述里提到了“前缀和”。如果我们提前算出 costW 等数组的累加和，就可以把最里面的 for 循环去掉，直接 O(1) 算出一段区间的代价，这就是时间优化魔法哦！不过对于这道题，直接累加也是没问题的！

🐍 Python 魔法秘籍 (知识点)

  • ♾️ 无量之数 float('inf')： 在 Python 中想要找最小值时，第一步总是把初始答案设为无穷大，咒语就是 float('inf')！
  • 🔪 切片魔法 list[start:end]： 强大的 Python 列表切片功能！不用像 C++ 那样写循环求和，一句 sum(costW[:i]) 就能直接把前 i 行的白色代价加总起来，超级优雅。
  • 📊 一行生成法： 使用 [input().strip() for _ in range(N)] 可以瞬间读完所有的旗布数据并剔除多余的换行符。

💻 Python 终极法术 (代码详解)

def main():
    # 🗺️ 读取行数和列数
    N, M = map(int, input().split())
    
    # 瞬间读入 N 行字符串，并去掉可能带有的回车空格
    grid = [input().strip() for _ in range(N)]

    # 📝 预处理账本：算出每一行如果全部变成 白/蓝/红 需要涂改几个格子
    costW = [0] * N
    costB = [0] * N
    costR = [0] * N
    
    for i in range(N):
        for ch in grid[i]:
            if ch != 'W':
                costW[i] += 1
            if ch != 'B':
                costB[i] += 1
            if ch != 'R':
                costR[i] += 1

    # 设置初始最小值为正无穷大
    ans = float('inf')
    
    # ✂️ 尝试所有剪裁方式！
    # i：白色行数 (至少1行，至多 N-2 行)
    for i in range(1, N - 1): 
        # j：蓝色行数 (至少1行，至多剩下的行数再减去给红色的底线)
        for j in range(1, N - i): 
            k = N - i - j # 红色行数
            if k < 1:
                continue
                
            # 🪄 切片求和魔法：
            # sum(costW[:i]) -> 第 0 到 i-1 行的涂白代价
            # sum(costB[i:i+j]) -> 第 i 到 i+j-1 行的涂蓝代价
            # sum(costR[i+j:]) -> 第 i+j 行到最后的涂红代价
            total = sum(costW[:i]) + sum(costB[i:i + j]) + sum(costR[i + j:])
            
            # 打擂台找最小值
            if total < ans:
                ans = total

    # 📢 输出最终答案！
    print(ans)

if __name__ == "__main__":
    main()

🌟 Python 重点复习小贴士

  • 💡 边界问题很关键： for i in range(1, N - 1) 这个边界要记牢。为什么到 N-1 结束？因为 range 是左闭右开的，实际 i 最大取到 N-2。这就给蓝红各留出了至少一行的生存空间！
  • 💡 切片的妙用： Python 最擅长处理列表！C++ 里累加一整块区域需要写 for 循环，但在 Python 里用 costW[:i] 截取这一段，再套一个 sum() 函数，代码既短又好懂！

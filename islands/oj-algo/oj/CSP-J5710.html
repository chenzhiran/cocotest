📜 任务清单 (题目描述)

田忌和齐王各有 n 匹马，每匹马都有一个固定的速度值，高者获胜。比赛进行 n 场，每匹马只能出战一次。

对决结算酬金机制：

  • 田忌马速 > 齐王马速： 田忌赢得该场，斩获 +200 银币。
  • 田忌马速 = 齐王马速： 双方握手言和，斩获 0 银币。
  • 田忌马速 < 齐王马速： 田忌不幸落败，倒贴 -200 银币。

请你帮助田忌安排比赛的出战顺序，计算并输出他最多能够得到的总银币净收入数量。

(3匹马，第2行为田忌，第3行为齐王)
3

📤 奇迹输出阵：
(精心布局后赚取的最高银币数)
200

数据规模限制：

  • 对于 100% 的数据，满足 1 ≤ n ≤ 2000
  • 每匹战马的速度值均为正整数且满足：0 ≤ 速度值 ≤ 100

🧠 魔法道具箱 (核心图解：田忌赛马的高级双指针贪心)

经典的民间故事告诉我们“用最慢的劣马去消耗对方的王牌”。但是在复杂的动态数据中，这个直觉需要被升级为严密的四指针双向贪心策略：

我们首先把双方的战马速度都从小到大（升序）排序。用两个最左端指针指向各自最慢的马，两个最右端指针指向各自最快的马。

⚔️ 运筹帷幄：四向指针对决树状抉择 ⚔️
情况 A：如果 田忌当前最快的马 > 齐王当前最快的马
👉 策略：那还等什么！直接用最快去干掉齐王的最快，稳稳收下 200 银币！(右指针各自往左收缩)
情况 B：如果 田忌当前最快的马 ≤ 齐王当前最快的马，但 田忌当前最慢的马 > 齐王当前最慢的马
👉 策略：既然最强的拼不过，就看最弱的。如果我最弱的能稳赢你最弱的，那就赶紧让两个最弱的对决，再收 200 银币！(左指针各自往右收缩)
情况 C：以上两个好形势都不满足（也就是高位拼不过，低位也毫无优势）
👉 策略：重头戏来了！既然田忌这匹最慢的马反正也赢不了齐王最慢的，注定无法带来胜利，那就把它派去直接挑战齐王目前最快的马！用最低的代价把齐王的最强王牌换掉，保护田忌其余的主力战马！
核心思想：打得过就狠狠地打，打不过就借力打力，用最废的子弹去消解敌人最强的装甲

📘 C++ 魔法秘籍 (知识点)

  • 标准容器存储： 利用 vector<int> 动态数组存储两边的速度，方便进行后续的高阶快速排序。
  • 高效序列重排： 直接调用内置的 std::sort，在 O(n log n) 的优秀对数时间内完成双方马速的升序大排队。
  • 四指针收缩法： 使用 t_l, t_r 和 k_l, k_r 两个极简整型变量来扮演数组下标指针，避免了复杂的对象构造，极大地加快了运行的速度。

💻 C++ 终极法术 (四指针收缩版)

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    // ⚡ 快读加持，解除输入输出同步羁绊
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    
    vector<int> tian(n), king(n);
    for (int i = 0; i < n; i++) cin >> tian[i];
    for (int i = 0; i < n; i++) cin >> king[i];

    // ⏱️ 将双方战马的速度从小到大进行精确的升序快排
    sort(tian.begin(), tian.end());
    sort(king.begin(), king.end());

    int t_l = 0, t_r = n - 1;  // 📍 田忌最慢马和最快马的左右指针
    int k_l = 0, k_r = n - 1;  // 📍 齐王最慢马和最快马的左右指针
    int ans = 0;               // 🏆 最终田忌净赚取的银币总数

    // 🔄 只要马儿还没全部比完，就继续开展博弈
    while (t_l <= t_r) {
        if (tian[t_r] > king[k_r]) {
            // 💥 局势A：田忌最快马强于齐王最快马，果断强杀强！
            ans += 200;
            t_r--;
            k_r--;
        } else if (tian[t_l] > king[k_l]) {
            // 💥 局势B：田忌最快马无优势，但最慢马能赢齐王最慢马，弱杀弱！
            ans += 200;
            t_l++;
            k_l++;
        } else {
            // 💥 局势C：高低均无胜算，贪心地让田忌最慢的废马去对冲掉齐王最快的王牌
            if (tian[t_l] < king[k_r]) {
                ans -= 200;  // 丢车保帅，虽然这局输了，但为其他主力创造了极大的赢面
            }
            t_l++;  // 田忌用掉了最慢的马
            k_r--;  // 齐王失去了最快的王牌马
        }
    }

    cout << ans << endl;
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • 列表快读切分： 使用高级列表映射语法 list(map(int, input().split()))，一行就能利落地将输入的整行战马速度切分开并整齐打包。
  • 原生原地高效排序： 调用列表自带的成员函数 tian.sort()，该算法底层由高度优化的 Timsort 实现，速度惊人。
  • 多变量并发迭代： Python 完美支持在一行里对多变量进行同步更新（例如 t_l, k_r = t_l + 1, k_r - 1），大大降低了代码的冗余程度。

💻 Python 终极法术 (四指针收缩版)

import sys

def main():
    # 📥 吞入大殿内的数据流
    n = int(sys.stdin.readline().strip())
    tian = list(map(int, sys.stdin.readline().strip().split()))
    king = list(map(int, sys.stdin.readline().strip().split()))

    # ⏱️ 双方马速原地完成完美的升序重排
    tian.sort()
    king.sort()

    t_l, t_r = 0, n - 1  # 📍 田忌最慢马与最快马索引
    k_l, k_r = 0, n - 1  # 📍 齐王最慢马与最快马索引
    ans = 0              # 🏆 银币汇总

    # 🔄 当指针未交错，说明还有马匹等待配对
    while t_l <= t_r:
        if tian[t_r] > king[k_r]:
            # 💥 优先局：田忌快马能赢齐王快马，上上马克中上马！
            ans += 200
            t_r -= 1
            k_r -= 1
        elif tian[t_l] > king[k_l]:
            # 💥 顺风局：快马不行但慢马能赢齐王慢马，以下下马克下下马！
            ans += 200
            t_l += 1
            k_l += 1
        else:
            # 💥 逆风局：高低都无胜算，果断用田忌最慢的马去消耗齐王最快的马！
            if tian[t_l] < king[k_r]:
                ans -= 200
            t_l += 1
            k_r -= 1

    print(ans)

if __name__ == "__main__":
    main()

💡 技巧和重点详细总结 (军师的羊皮纸反思笔记)

  • 🌟 为什么传统的“固定下马对上马”会拿不到满分？： 民间流传的常规故事是死板地让田忌的最慢马去对齐王的最快马。但思考一下：如果田忌的最慢马速度是 10，齐王的最慢马是 5，田忌的最慢马明明可以稳拿下一局获得 200 银币，你却盲目地把它送去给齐王最快的马白白吃掉，这就产生了极大的浪费！所以我们必须采取高低位双向试探，只有在左右都无法得分的情况下，才启动“下马拖上马”的牺牲保帅技能，这才是真正的严密全能贪心。
  • 🌟 负数分数的魔鬼细节： 在某些极端刁钻的测试数据中，齐王的马队可能全线彻底压制田忌（比如田忌全员 10 速度，齐王全员 100 速度）。此时田忌无论怎么巧妙避战，也必须输掉所有比赛，最终总收入会是一个负数。由于我们完美的分类决策能够保证每一局的“亏损和损耗降到最低”，因此循环结束后输出的那个负值依然是全局亏损最少、也就是保留最完美的最高收入解！
  • 🌟 绝妙的时间复杂度表现： 本算法最主要的开销在最初的两次数组排序，需要消耗 O(n log n) 的时间。随后的 while 双指针扫描中，四个指针单向内缩，每个马匹数据只被读取了一次，循环扫描开销仅为线性的 O(n)。因此，整套算法的总时间复杂度达到了完美的 O(n log n)。对于规模高达 n = 2000 的全面大决战，程序也能在 2 毫秒内瞬间破阵，稳拿满分！

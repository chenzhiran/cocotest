5.3 二维递推与组合计数

🕵️‍♂️ 名侦探语录：

“当线索错综复杂，一条单行道走不通时，我们需要一张网！” 从一维升级到二维、甚至三维递推，意味着我们的状态不再仅仅由一个变量决定，而是需要考虑多个维度的制约。准备好迎接更烧脑的多维推导了吗？

一、探索多维世界：解题思路与方法论

在上一节中，我们遇到的大多是像“爬楼梯”或“斐波那契”这样的一维问题。一维问题就像是在一条直线上走路，你唯一需要关心的状态变量就是“当前走到了第几级”。

但在真正的竞赛和复杂问题中，状态往往由多个条件共同决定。比如：“你在一个迷宫里（需要坐标 x 和 y）”，“你挑了几个物品，且总重量不能超过 W（需要物品数 i 和重量 j）”。这就需要引入多维递推的思想！

🧠 破解多维递推的“降维打击”三步曲：

1. 找准维度（状态定义）： 仔细读题，问自己：“我需要同时知道哪些信息，才能完整描述当前的局面？” 每多一个必需的信息，数组就多加一维。比如：dp[数量][限制条件A][限制条件B]。
2. 确立原点（边界初始化）： 找到最微小、最简单、甚至“什么都不做”时的状态，把它作为推导的起点，比如 dp = 1 或 dp[1][1][1] = 1。
3. 剥离最后一步（降维推导）： 不要去想第一步怎么走，而是假设现在已经是一个复杂状态了，思考“我的上一步（或最后一个加入的元素）是从哪个方向过来的？” 只要理清这“最后一下”的操作，就能写出完美的状态转移方程。

二、几何递推：平面分割问题

题目大意： 假设平面上有 n 条封闭曲线（比如圆或椭圆）。如果保证每两条曲线恰好相交于两点，并且没有三条曲线相交在同一个点。请问，这 n 条封闭曲线一共能把平面分成多少个区域？

💡 大白话推导过程（发现几何规律）：
1. 起点： 只有 1 条封闭曲线时，平面被分为“里面”和“外面”共 2块 区域 (a1 = 2)。
2. 加入第 2 条曲线： 它与第 1 条曲线必定交于 2 个点。这 2 个交点把新曲线分成了 2 段，每 1 段都会把一个旧区域“一分为二”。所以新增了 2 块区域，总共 2 + 2 = 4块 (a2 = 4)。
3. 加入第 n 条曲线： 它与前面已有的 n-1 条曲线两两相交，所以会产生 2 × (n-1) 个交点。这 2 × (n-1) 个交点把第 n 条曲线切成了 2 × (n-1) 段。每一段都会“割裂”一个旧区域，从而诞生一个新区域！
结论： 新增的区域数，永远等于新产生的交点数。
an = an-1 + 2 × (n - 1)
曲线数量 (n) 总区域数 (an) 推导过程 (新增区域 = 交点数)
1 2 基础状态
2 4 a1 + 2 × 1 = 2 + 2 = 4
3 8 a2 + 2 × 2 = 4 + 4 = 8
4 14 a3 + 2 × 3 = 8 + 6 = 14

三、三维递推应用：光之法阵（排列组合进阶）

问题背景： 在魔法学院里，年轻的魔导师们正在设计一个光之阵法。有 n 根高度互不相同（高度分别为 1 到 n）的水晶柱排成一排。如果从最左边看，能看到 L 根柱子；从最右边看，能看到 R 根柱子（矮的柱子会被高的挡住）。请问，有多少种不同的排列方案？

🧠 状态设计与推导：
我们需要设计一个三维状态 dp[i][j][k]，表示当前一共排了 i 根柱子，从左边看能见到 j 根，从右边看能见到 k 根的方案数。

巧妙的思维转换：排列组合如果从小排到大，高个子遮挡矮个子，非常难以计算！相反，假设我们先排好了 i-1 根高的，现在把最矮的柱子（高度1）插进去。因为它是最矮的，插进去绝对不会挡住任何人，只会影响它自己能不能被看见。
👉 选择 1：插在最左端。 它是最左边的，所以左侧视野 j 增加了 1，右侧视野不变。这部分方案数来源于之前的状态 dp[i-1][j-1][k]。
👉 选择 2：插在最右端。 它是最右边的，所以右侧视野 k 增加了 1，左侧视野不变。这部分方案数来源于之前的状态 dp[i-1][j][k-1]。
👉 选择 3：插在中间缝隙。 前面有 i-1 根柱子，中间共有 i-2 个缝隙。插在中间被高个子彻底挡死，左右视野都不变！所以有 i-2 种插法，方案数为 dp[i-1][j][k] × (i-2)。
dp[i][j][k] = dp[i-1][j-1][k] + dp[i-1][j][k-1] + dp[i-1][j][k] × (i - 2)
#include <iostream> #include <vector> using namespace std; long long dp[50][50][50]; void init_dp(int max_n) { // 1. 确立原点（边界条件）：1根柱子，左看右看都是1根，只有1种方案 dp[1][1][1] = 1; // 2. 三重循环降维打击： // 遍历柱子总数 i for (int i = 2; i <= max_n; i++) { // 遍历左侧可见数 j for (int j = 1; j <= i; j++) { // 遍历右侧可见数 k for (int k = 1; k <= i; k++) { // 3. 剥离最后一步（递推转移方程） dp[i][j][k] = dp[i-1][j-1][k] // 最矮柱子放最左 + dp[i-1][j][k-1] // 最矮柱子放最右 + dp[i-1][j][k] * (i - 2); // 最矮柱子放中间的(i-2)个位置 } } } } int main() { init_dp(20); // 预处理出所有的状态 int n = 5, L = 2, R = 4; // 例如：5根柱子，左看2，右看4 cout << dp[n][L][R] << endl; // 直接 O(1) 查询 return 0; }

🏆 竞赛实战防爆零小贴士

  • 多维数组的空间开销： 遇到二维或三维的 DP 递推题时，一定要先估算一下内存！例如 long long dp[1000][1000] 会消耗近 8MB 内存。如果维度太高，尝试使用“滚动数组”将第一维的空间优化掉（就像我们在 5.2 节中做的那样）。
  • 逆向思维的美妙： 像“光之法阵”这种排列组合题，如果是按照高度从小到大去排，高个子会遮挡矮个子，导致状态转移非常复杂甚至无法推导。遇到这种情况，务必尝试逆向思维：把“最小/最矮”的元素单独拿出来最后插入！ 往往会有奇效，因为它的插入绝对不会破坏原有的相对遮挡状态。

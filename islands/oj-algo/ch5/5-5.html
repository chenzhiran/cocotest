5.5 高阶递推：卡特兰数 (Catalan)

🕵️‍♂️ 名侦探语录：

“在侦探界，有些看似毫不相干的离奇案件，背后往往隐藏着同一个‘幕后黑手’！”
在组合数学和算法竞赛中，卡特兰数就是这样一个极其神秘的“幕后黑手”。它穿上了无数件马甲，出没于多边形、二叉树、路径规划和数据结构等各个领域。今天，我们就要撕下它的伪装，直击核心！

一、卡特兰数的真面目：定义与公式

卡特兰数 Cn 是一串极其特殊的整数序列。它的前几项是：1, 1, 2, 5, 14, 42, 132, 429...（记住这几个数字，竞赛中一旦手算出了前几项是它们，直接盲猜卡特兰数准没错！）

🧠 卡特兰数的三大核心武器（公式）：

1. 核心递推式（最常用于理解本质）：
将一个规模为 n 的大问题，拆分成两部分：一部分分配 i 个元素，另一部分分配剩下的 n-1-i 个元素。然后将两部分的方案数相乘，最后把所有可能的分配情况累加起来。
Cn = C0×Cn-1 + C1×Cn-2 + ... + Cn-1×C0 (边界 C0 = 1)

2. 线性推导式（最常用于写代码 O(n) 优化）：
通过数学化简，我们可以直接用前一项算出后一项，无需循环累加，速度极快！
h(n) = h(n-1) × (4n - 2) / (n + 1)

3. 组合数公式（最常用于排列组合推导）：
Cn = 1 / (n + 1) × C(2n, n)   (其中 C 是数学中的组合数)

二、撕下伪装：五大经典应用场景精讲

为什么说卡特兰数是“幕后黑手”？因为以下这五个经典问题，虽然描述天差地别，但它们的答案全都是卡特兰数！ 我们以 n = 3 (答案为 5) 为例，通过图解来看看它们惊人的同构性：

💡 为什么这五个场景都是同一个“幕后黑手”？（大白话解析）

👉 1. 凸多边形三角划分： 一个 n+2 边形，固定一条底边，再随便挑一个顶点连成三角形，这个三角形就会把原多边形劈成“左边一个小多边形”和“右边一个小多边形”。这不就是 C(左) × C(右) 吗？
👉 2. 二叉树形态计数 (NOIP真题)： 给定 n 个节点，拔出一个当“根节点”，剩下的 n-1 个节点必须分给左子树和右子树。这不也是把问题劈成两半 C(左子树) × C(右子树) 吗？
👉 3. 合法括号匹配： 一堆括号中，找到最左边的 (，再找到跟它匹配的那个 )。这对括号把整个字符串劈成了“括号包住的内部”和“这对括号的右边”两部分。依然是 C(内) × C(右)！
👉 4. 堆栈出栈序列： 1到n进栈，第 1 个进栈的元素在第 i 个出栈。这就把序列劈成了“在它之前出栈的”和“在它之后出栈的”。
👉 5. 路径计数问题： 在网格中从左下角走到右上角且不越过对角线。找到路径“第一次”碰到对角线的点，路径也被切成了两段！

🔥 NOIP 2015 提高组真题验证：
题目问：结点数为 5 的不同形态的二叉树一共有多少种？
我们直接代入卡特兰数线性递推公式：h(n) = h(n-1) × (4n - 2) / (n + 1)
已知 h(1) = 1, h(2) = 2, h(3) = 5。
当 n = 4：h(4) = 5 × (16 - 2) / 5 = 14
当 n = 5：h(5) = 14 × (20 - 2) / 6 = 14 × 18 / 6 = 42 种！秒杀真题！

三、核心程序模板与实战拆解

为了应对不同规模的数据，我们必须掌握 O(n2) 的循环累加法，以及 O(n) 的极速推导法。

模板一：O(n) 极速推导（强烈推荐，竞赛首选）

直接利用递推公式 h[i] = h[i-1] × (4i - 2) / (i + 1)。速度极快，只需要一重循环！

#include <iostream> using namespace std; long long catalan_fast(int n) { if (n == 0 || n == 1) return 1; long long h[100]; h[0] = 1; h[1] = 1; for (int i = 2; i <= n; i++) { // 核心优化公式，一定要先乘再除，防止除不尽丢失精度！ h[i] = h[i-1] * (4 * i - 2) / (i + 1); } return h[n]; } int main() { cout << catalan_fast(5) << endl; // 输出 42，真题答案 return 0; }
def catalan_fast(n): if n == 0 or n == 1: return 1 h = [0] * (n + 1) h[0] = 1 h[1] = 1 for i in range(2, n + 1): # Python 中使用 // 确保进行整数除法 h[i] = h[i-1] * (4 * i - 2) // (i + 1) return h[n] print(catalan_fast(5)) # 输出 42，真题答案

模板二：O(n²) 循环累加法（原汁原味的定义）

完美再现了“将问题劈成两半，左半边乘以右半边，最后把所有切分点的方案相加”的原始逻辑。

long long catalan_sum(int n) { long long c[100] = {0}; c[0] = 1; // 边界条件很重要 c[1] = 1; // 外层循环算第 i 个卡特兰数 for (int i = 2; i <= n; i++) { // 内层循环模拟一个一个的“切分点” j for (int j = 0; j < i; j++) { // 核心递推：累加 (左边部分 * 右边部分) c[i] += c[j] * c[i - 1 - j]; } } return c[n]; }
def catalan_sum(n): c = [0] * (n + 1) c[0] = 1 # 边界条件很重要 if n >= 1: c[1] = 1 # 外层循环算第 i 个卡特兰数 for i in range(2, n + 1): # 内层循环模拟一个一个的“切分点” j for j in range(i): # 核心递推：累加 (左边部分 * 右边部分) c[i] += c[j] * c[i - 1 - j] return c[n]

🏆 竞赛实战防爆零小贴士

  • 数据爆炸极其恐怖： 卡特兰数的增长速度快得令人发指！当 n = 20 时，结果高达 6,564,120,420，普通的 int 早就爆掉了。因此，写卡特兰数必须无脑使用 long long（C++ 选手牢记）。
  • O(n) 线性推导的“除法”陷阱： 在使用 h[i] = h[i-1] * (4*i - 2) / (i + 1) 时，必须先做乘法，再做除法！ 如果先除，会导致严重的小数截断丢失精度（因为 (4i-2) 往往无法整除 (i+1)，但乘上 h[i-1] 之后就一定能整除）。
  • 取模防坑： 很多题目会要求对 109+7 取模。如果使用 O(n) 的带除法的公式，千万不能直接在除法上取模！ 必须使用“乘法逆元”（费马小定理）来处理除法取模。如果还没学过逆元，老老实实用 O(n²) 的累加法加法取模最安全。

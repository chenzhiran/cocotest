5.6 递推与动态规划的衔接及综合实战

🕵️‍♂️ 名侦探语录：

“最顶级的推理，不仅要算出有多少种可能，还要在无数可能中找出最完美的那条路！”
当单纯的规律难以捕捉，或者面临多重选择时，我们将经典的递推思想升级为动态规划（DP）。今天，我们将揭开这座桥梁的秘密，并挑战两个高难度实战案件！

一、从递推走向动态规划：进阶之路

在前面的章节中，无论是斐波那契数列、上楼梯还是卡特兰数，我们解决的都是“计数问题”（有多少种方案）。在纯递推的世界里，不管你怎么走，条条大路通罗马，我们只需要把所有的可能性加起来（+）。

但如果你在冒险中遇到了“选择”呢？比如：面前有两条路，走左边可以捡到 100 金币，走右边只有 50 金币，问走到终点时最多能有多少金币？这就需要我们在递推的基础上，引入最优化决策（找最大值或最小值）。这就正式跨入了动态规划（Dynamic Programming, 简称 DP）的领地！

💡 大白话总结两者的关系：
动态规划（DP）的本质其实就是“带有记忆化和最优决策的递推”。
• 当状态转移方程里只有加法 + 时，你多半在做纯粹的递推计算。
• 当状态转移方程里出现了 max() 或 min() 时，说明你开始让程序做选择了，这就是动态规划。接下来，让我们通过两个高阶案件，彻底打通这两者的界限！

二、递推视角下的复杂状态设计：骑士赛马

案件背景： 在一个奇幻大陆上，有 n 组骑士参加赛马比赛。比赛不仅分出胜负，还允许并列名次（比如 3 个人并列第一名）。请问最终名次的排列组合一共可能有多少种？结果需要对 10056 取模。

🧠 “名次剥离法”原理解析：
如果比赛没有并列，那答案很简单，就是 n!（全排列）。但现在允许并列，这其实是第二类斯特林数的经典变种！怎么递推呢？

降维打击：我们直接把“第一名”剥离出来！
假设有 m 个人同时并列第一名，我们要从 n 个人里挑出这 m 个人，方案数就是组合数 C(n, m)。
第一名选完后，剩下的 n-m 个人该怎么排？这完全变成了一个规模缩小了的相同的子问题，方案数就是 f(n-m)。

因为并列第一名的人数 m 可以是 1 个，2 个，甚至是所有人（n 个），所以我们把所有可能的 m 累加起来，就得到了终极递推公式：
f(n) = Σ [ C(n, m) × f(n - m) ]    (其中 m 从 1 遍历到 n)
#include <iostream> using namespace std; const int MOD = 10056; long long C[1005][1005]; // 用于存储组合数 long long f[1005]; // 存储最终的排列方案数 void init(int max_n) { // 1. 预处理出所有的组合数 C(n, m) for (int i = 0; i <= max_n; i++) { C[i][0] = 1; for (int j = 1; j <= i; j++) { // 利用杨辉三角递推组合数 C[i][j] = (C[i-1][j-1] + C[i-1][j]) % MOD; } } // 2. 剥离法核心递推 f[0] = 1; // 0个人的排列只有1种(什么都不排) for (int n = 1; n <= max_n; n++) { f[n] = 0; // 枚举并列第一名的人数 m for (int m = 1; m <= n; m++) { long long temp = (C[n][m] * f[n - m]) % MOD; f[n] = (f[n] + temp) % MOD; } } } int main() { init(1000); cout << f[3] << endl; // 3个骑士，答案为 13 return 0; }
MOD = 10056 max_n = 1000 # 1. 预处理组合数 C(n, m) C = [[0] * (max_n + 1) for _ in range(max_n + 1)] for i in range(max_n + 1): C[i][0] = 1 for j in range(1, i + 1): C[i][j] = (C[i-1][j-1] + C[i-1][j]) % MOD # 2. 剥离法核心递推 f = [0] * (max_n + 1) f[0] = 1 # 0个人的排列只有1种 for n in range(1, max_n + 1): # 枚举并列第一名的人数 m for m in range(1, n + 1): f[n] = (f[n] + C[n][m] * f[n - m]) % MOD print(f[3]) # 3个骑士，答案为 13

三、奖励关卡：逆推法与期望 DP 入门

案件背景： 系统将抛出 k 次宝物，宝物共有 n 种且每次抛出概率均等。每次你可以选择“吃”或“不吃”。每个宝物都有不同的分数，但有些宝物必须在吃到某些特定宝物后才能吃（即：有前提集合的限制）。如果你采取最优策略，求平均情况下的最大期望得分？

🧠 逆推法核心逻辑：
如果从第 1 轮开始顺着推，因为后续会掉落什么宝物全看运气，你当前吃的决策会严重干扰后续的概率计算。这叫“前置宝物依赖的不确定性”。

但如果我们使用倒推法（逆推法）：假设时光已经来到了游戏结束后的第 k+1 轮，此时不管什么状态，因为游戏已经结束，期望得分绝对是 0！
我们定义 F[i][j] 表示：当前在第 i 轮，且你包里已经收集到的宝物状态为 j（用二进制“状态压缩”表示集合），从此时算起，到游戏结束你能获得的最大期望得分。
当我们在第 i 轮面临第 l 种宝物掉落时：
👉 如果不吃： 你的状态没变，期望等于下一步的期望 F[i+1][j]。
👉 如果吃（且满足前提）： 你的包里多了一个宝物，状态变成了 j | (1<<(l-1))，期望变成了 F[i+1][新状态] + V[l]。
因为你是绝顶聪明的，你一定会取这两种选择中的 Max 最大值！
F[i][j] += Max( F[i+1][j] , F[i+1][j | (1<<l)] + V[l] ) / n
// k轮，n种宝物。sc数组为分数，need数组存前提条件的二进制集合 // F[105] 存储期望值 for (int i = k; i >= 1; i--) { // 1. 时间倒流：从第 k 轮倒推至第 1 轮 for (int j = 0; j < (1 << n); j++) { // 2. 枚举当前拥有的宝物集合 j (状态压缩) F[i][j] = 0; // 3. 因为抛出 n 种宝物的概率是均等的，所以累加后除以 n for (int l = 1; l <= n; l++) { // 位运算检查：如果 j 中已经包含了 need[l] 需求集合的所有 1 if ((j & need[l]) == need[l]) { // DP的最优化决策：比较“不吃”和“吃掉”取最大值 F[i][j] += max(F[i+1][j], F[i+1][j | (1 << (l-1))] + sc[l]); } else { // 条件不满足，被迫浪费机会，只能不吃 F[i][j] += F[i+1][j]; } } F[i][j] /= n; // 计算期望均值 } } cout << F[1][0] << endl; // 游戏刚开始时（第1轮），什么宝物都没有（状态0）的最大期望
# k轮，n种宝物。sc列表为分数，need列表存前提条件的二进制集合 # F 矩阵存储期望值 for i in range(k, 0, -1): # 1. 时间倒流：从第 k 轮倒推至第 1 轮 for j in range(1 << n): # 2. 枚举当前拥有的宝物集合 j (状态压缩) F[i][j] = 0 # 3. 抛出 n 种宝物概率均等，累加后除以 n for l in range(1, n + 1): # 位运算检查：如果 j 中已经包含了 need[l] 需求的所有 1 if (j & need[l]) == need[l]: # DP的最优化决策：比较“不吃”和“吃掉”取最大值 F[i][j] += max(F[i+1][j], F[i+1][j | (1 << (l-1))] + sc[l]) else: # 条件不满足，被迫不吃 F[i][j] += F[i+1][j] F[i][j] /= n # 计算期望均值 print(F[1][0]) # 游戏刚开始时（第1轮），状态为0的最大期望

🏁 结课大盘点：寻找递推公式的终极方法论

递推与动态规划是信奥赛中最能拉开分数差距的核心板块。在面对千奇百怪的题目时，请死死记住以下“解题三步法”：

  • 【第一步：看数据范围猜维数】 —— n ≤ 105 通常是一维；n ≤ 1000 是二维（如背包、棋盘）；如果 n ≤ 20，大概率是状态压缩 DP（把集合变成二进制整数）。
  • 【第二步：找准起点（基准条件）】 —— 没有初始值，公式就是无本之木。先手动推导规模为 0, 1, 2 时的边界值。特别注意某些概率期望题，终点才是确定的基准值（比如结束时得分为 0），此时必须倒推！
  • 【第三步：狠抓“最后一步”（定关系）】 —— 不要想整个过程多复杂，只去想：“当前这个局面，是由上一秒的哪几种动作演变而来的？” 比如上楼梯的最后一步跨几级、汉诺塔最后一堆怎么移、赛马怎么剥离出第一名。理清最后一步，引入 max/min 进行抉择，状态转移方程就迎刃而解！

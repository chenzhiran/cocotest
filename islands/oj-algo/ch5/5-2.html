5.2 一维线性递推及其经典模型

🕵️‍♂️ 名侦探语录：

“线索往往连成一条单行道，顺着它，你可以直接走到真相的大门前。” 在一维数组的世界里，递推关系就像是串联起来的多米诺骨牌，掌握了前几块骨牌的倒向规律，就能精准预测整个未来的走向！

一、核心案例：斐波那契数列的深度剖析

斐波那契数列（Fibonacci Sequence）是一维递推中最古老也是最经典的代表。它的核心规则极为简单，却蕴含着无尽的奥秘：每一个状态都由它前面的两个状态叠加而成。

F(n) = F(n-1) + F(n-2)
  • 自然界中的神秘映射：
    • 向日葵的螺旋： 当你俯瞰向日葵的花盘，会发现种子排列成了一圈圈的螺旋。神奇的是，顺时针和逆时针的螺旋数目，往往正好是相邻的两个斐波那契数（比如 13 和 21）！这能让种子最紧密地利用空间。
    • 黄金分割比率： 随着数字越来越大，斐波那契数列中相邻两项的比值（例如 34 除以 55），会无限趋近于著名的黄金分割率 0.618。从帕特农神庙的建筑比例到蒙娜丽莎的微笑，再到鹦鹉螺的完美曲线，都暗藏着这个密码。

二、万变不离其宗的“一维变种题”

在信奥赛中，出题人特别喜欢给斐波那契数列穿上各种“马甲”。无论题目描述有多花哨，只要扒下它们的外衣，你会发现底层逻辑全都是那个神奇的公式：当前项 = 前一项 + 前两项！

变种一：楼梯台阶问题

题目大意： 你要爬一段有 n 级台阶的楼梯。你的腿长有限，每次只能选择跨 1 级，或者跨 2 级。请问到达最高点一共有多少种不同的走法？

💡 大白话解析：
想象一下你现在就站在最高点（第 n 级）。你是怎么上来的？因为你每次只能跨 1 步或 2 步，所以你只能从“第 n-1 级”或者“第 n-2 级”跳上来，绝对没有第三种可能！
所以，到达终点的总方案数，自然就等于到达 n-1 级的方案数，加上到达 n-2 级的方案数。这不就是原汁原味的斐波那契吗！

变种二：兔子繁殖问题

题目大意： 有一对刚出生的小兔子，它们需要长 2 个月才能成年。从第 3 个月开始，这对成年兔子每个月都能生出一对新的小兔子（假设兔子永远不死）。求第 n 个月时，一共有多少对兔子？

💡 大白话解析：
第 n 个月时，兔子的总数是怎么来的？首先，上个月（n-1月）的所有兔子都还活着，直接加进来。其次，这个月会有新生宝宝诞生！谁能生宝宝呢？只有那些在 前两个月（n-2月） 就已经存在的兔子，到现在才正好成年具备生育能力。
所以： 第 n 月总数 = (n-1)月的存活量 + (n-2)月老兔子生的新宝宝量

变种三：骨牌铺法问题

题目大意： 你有一大堆尺寸为 1×2 的多米诺骨牌。现在要用它们铺满一个高度为 2、长度为 n 的长方形地板（即 2×n 的网格），请问有多少种铺法？

💡 大白话解析：
铺设地板时，我们直接盯住最右边的末尾处。你只有两种收尾方式：
1. 竖着放 1 块： 它占据了 1 列，那么剩下的 n-1 列不管怎么铺，方案数都是 f(n-1)。
2. 横着放 2 块： 为什么是两块？因为高度是 2，横着放一块的话，它下面或上面必定空出 1×2 的格子，只能再塞一块横着的。这一下就占据了 2 列，剩下的 n-2 列的方案数就是 f(n-2)。
把这两种情况加起来，又是经典的： f(n) = f(n-1) + f(n-2)
C++ 一维递推通用模板
// 无论是楼梯、兔子还是骨牌，核心代码全部长这样！ #include <iostream> using namespace std; long long solve_1D_DP(int n) { if (n <= 2) return n; // 针对台阶和骨牌的边界 long long dp[100]; dp[1] = 1; dp[2] = 2; for (int i = 3; i <= n; i++) { dp[i] = dp[i-1] + dp[i-2]; // 神奇的递推公式 } return dp[n]; }
Python 一维递推通用模板
# 无论是楼梯、兔子还是骨牌，核心代码全部长这样！ def solve_1D_DP(n): if n <= 2: return n # 针对台阶和骨牌的边界 dp = [0] * (n + 1) dp[1] = 1 dp[2] = 2 for i in range(3, n + 1): dp[i] = dp[i-1] + dp[i-2] # 神奇的递推公式 return dp[n]

三、高阶一维递推：错排问题 (Derangement)

问题描述： 假设有 n 个元素，要求这 n 个元素每一个都不能出现在它原来的位置上。最经典的例子就是：有 n 封信和 n 个对应的信封，粗心的邮递员把所有信全部装错了信封，求一共有多少种装法？

🧠 烧脑公式大白话推导：
1. 对于第 n 封信，因为不能装进自己的信封，所以它有 n-1 个错误的目标可以选择（假设它偏偏去了位置 k）。
2. 此时，原本属于位置 k 的那封“信 k”无家可归了。它面临两种抉择：
   👉 情况A： 信 k 说：“你占了我的家，那我就去占你的家！” 于是信 k 去了信封 n。两人完美互换，剩下的事情交给了其余 n-2 封信去互相错排，方法数为 Dn-2。
   👉 情况B： 信 k 偏不去 信封 n，它去了别的信封。因为信 k 绝不进信封 n，我们干脆把信封 n 视作信 k 的“新本命家”（反正规则是不能去本命家）。这就直接变成了一个规模缩小了的 n-1 封信的错排问题，方法数为 Dn-1。
最后别忘了，第 n 封信最初有 n-1 种选择，根据乘法原理，我们得出了绝妙的错排公式：
Dn = (n-1) × (Dn-1 + Dn-2)

代码进阶：空间复杂度优化至 O(1) 的“滚动数组”

仔细观察错排公式，你会发现求解当前的 Dn，仅仅依赖前两项（Dn-1 和 Dn-2），再往前很久的历史数据全都没用了！
既然如此，我们何必开辟一个巨大的数组呢？只需设置三个变量，在循环中不断“轮流接力替换”，就能将空间复杂度直接从 O(n) 降维打击到 O(1)，这就是大名鼎鼎的滚动数组优化思想。

C++ 错排与空间优化
#include <iostream> using namespace std; long long derangement(int n) { if (n == 1) return 0; // 1封信不可能错排 if (n == 2) return 1; // 2封信互相换只有1种可能 long long a = 0; // 扮演 D(n-2) 的角色 long long b = 1; // 扮演 D(n-1) 的角色 long long c = 0; // 扮演当前的 D(n) // O(1) 空间复杂度的滚动数组 for (int i = 3; i <= n; ++i) { c = (i - 1) * (a + b); // 代入错排核心递推公式 a = b; // 把 b 的值移交给 a (历史数据往前滚) b = c; // 把 c 的值移交给 b (准备下一次循环) } return b; }
Python 错排与空间优化
def derangement(n): if n == 1: return 0 # 1封信不可能错排 if n == 2: return 1 # 2封信互相换只有1种可能 a = 0 # 扮演 D(n-2) 的角色 b = 1 # 扮演 D(n-1) 的角色 # Python 支持优雅的多重赋值，连变量 c 都不需要额外声明 for i in range(3, n + 1): # a变成原来的b，b变成公式计算出的新值。一行代码完成滚动！ a, b = b, (i - 1) * (a + b) return b

🏆 竞赛实战防爆零小贴士

  • 数据类型必看： 斐波那契数列和错排问题的增长速度呈指数级爆炸！通常当 n > 46 时，普通 int 就会溢出变成负数。请务必使用 long long (C++) 存储结果！
  • 取模陷阱： 在真题中，由于答案过大，题目通常会要求“答案对 109+7 取模”。切记，要在循环内的每一次加法或乘法之后立刻取模，不要等算完最终结果再取模，那时早就溢出了。
  • 边界初始化： 不要死记 dp 和 dp，要结合题目给的实际场景推导最初的两项。比如有些题目的起步台阶是 dp = 1。

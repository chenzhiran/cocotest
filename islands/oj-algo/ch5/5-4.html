5.4 递推算法的时空优化技术

🕵️‍♂️ 名侦探语录：

“找出真相不仅要准确，还要快、准、狠！优秀的侦探绝不会把时间浪费在无用的线索上，也不会把大脑塞满已经过期的记忆。”
当问题规模急剧膨胀，普通递推会面临“内存撑爆”或“超时计算”的死局。时空优化，就是我们突破性能瓶颈、斩获满分的终极黑魔法！

一、空间优化：滚动数组与内存复用

在竞赛中，有时候题目会要求你计算到第 10 万项甚至 100 万项。如果你老老实实地去开辟一个 int dp 的数组，很容易就会触发 MLE (Memory Limit Exceeded，内存超限)。

但我们仔细观察斐波那契数列或错排问题的递推公式：

F(n) = F(n-1) + F(n-2)
🧠 详细原理解析：
想象你在过河，踩着水面上的石头往前走。为了踏出下一步（求解 F(n)），你只需要知道你现在踩着的石头（F(n-1)）和刚才踩过的石头（F(n-2)）在哪里。

至于你几分钟前踩过的石头（F(n-3), F(n-4)...），它们对你迈出下一步毫无帮助！留在内存里纯属浪费。
优化绝招： 我们只需要设定 3 个变量，在循环中不断地把“新数据”覆盖到“旧数据”上，让这几个变量像车轮一样滚动更新，就能将空间复杂度从庞大的 O(n) 极致压缩到几乎不占内存的 O(1)。

应用示例： 求解斐波那契数列的第 1,000,000 项对 1000000007 取模的结果。使用普通数组会导致内存占用，而滚动数组轻松搞定。

#include <iostream> using namespace std; long long fib_optimized(int n) { if (n == 0) return 0; if (n == 1) return 1; long long a = 0; // 相当于 F(n-2) long long b = 1; // 相当于 F(n-1) long long c = 0; // 用于存储新计算出的 F(n) for (int i = 2; i <= n; i++) { c = (a + b) % 1000000007; // 1. 计算出新项 a = b; // 2. 旧数据滚动：a接替b b = c; // 3. 旧数据滚动：b接替c，为下一轮做准备 } return c; }
def fib_optimized(n): if n == 0: return 0 if n == 1: return 1 a = 0 # 相当于 F(n-2) b = 1 # 相当于 F(n-1) for _ in range(2, n + 1): # Python 支持一行内同时进行计算和赋值更替！ # 此时的 a 变成了旧的 b，而 b 变成了两者的和 (即新的项) a, b = b, (a + b) % 1000000007 return b

二、多阶段决策递推：从三柱到四柱汉诺塔

1. 前置复习：经典的三柱汉诺塔

传统的汉诺塔只有 3 根柱子（A, B, C），要求把 i 个盘子从小到大移到目标柱。它的底层逻辑非常纯粹：大盘子绝不能压在小盘子上。

🧠 三柱推导过程：
要想把最底下那个最大的盘子移到终点 C 柱，它上面的 i-1 个小盘子必须全部乖乖地让开路，并且堆放在备用的 B 柱上。
1. 把顶上 i-1 个盘子从 A 柱移到 B 柱，耗时 h3(i-1) 步。
2. 把最底下的 1 个最大的盘子直接从 A 柱移到 C 柱，耗时 1 步。
3. 再把 B 柱上的 i-1 个盘子移回 C 柱盖在大盘子上，耗时 h3(i-1) 步。
由此得出经典公式：h3(i) = 2 × h3(i-1) + 1
#include <iostream> using namespace std; long long solve_h3(int n) { long long h3[100]; h3[1] = 1; // 1个盘子只需直接移动1步 for (int i = 2; i <= n; i++) { h3[i] = 2 * h3[i-1] + 1; } return h3[n]; }
def solve_h3(n): h3 = [0] * (n + 1) h3[1] = 1 # 1个盘子只需直接移动1步 for i in range(2, n + 1): h3[i] = 2 * h3[i-1] + 1 return h3[n]

2. 进阶挑战：四柱汉诺塔 (Frame-Stewart 算法)

应用场景： 如果现在给你 4 根柱子（A, B, C, D），把 20 个盘子从 A 移到 D，最少需要多少步？因为多了一根柱子可以做中转，步数将大幅减少！

🧠 化四为三的奇妙降维推导：
既然有 4 根柱子，我们可以把所有的 i 个盘子强行分成两拨：上面较小的 j 个，和底下较大的 i-j 个。

阶段一： 我们利用全部 4 根柱子，把顶上那 j 个小盘子安全转移到 B 柱上。这耗费了 h4(j) 步。
阶段二： 重点来了！此时 B 柱上铺满了小盘子。对于剩下的 i-j 个大盘子来说，它们绝对不能放到 B 柱上去（大不能压小）。所以在它们眼中，B 柱相当于“报废”了。它们只能利用剩下的 A、C、D 三根柱子，痛苦地转移到终点 D 柱。这耗费了 h3(i-j) 步。
阶段三： 大盘子落位后，我们再利用全部 4 根柱子，把 B 柱上的那 j 个小盘子重新搬回 D 柱即可。又耗费了 h4(j) 步。

终极难点： 我们到底该取多少个盘子作为上面的 j 呢？答案是不知道！所以我们要写一个 for 循环，遍历所有可能的 j (从 1 到 i-1)，把它们产生的结果做对比，取步数最小的那一个！
h4(i) = min { 2 × h4(j) + h3(i-j) } (其中 1 ≤ j < i)
#include <iostream> #include <algorithm> using namespace std; long long h3[25], h4[25]; int main() { // 1. 必须先预处理出所有 3 柱汉诺塔的答案，作为素材备用 h3[1] = 1; for (int i = 2; i <= 20; ++i) { h3[i] = 2 * h3[i-1] + 1; } // 2. 计算 4 柱汉诺塔的最优解 h4[1] = 1; // 边界：1个盘子还是1步 for (int i = 2; i <= 20; ++i) { h4[i] = 999999999; // 初始化为一个极大值，准备打擂台找最小值 // 核心：遍历所有的切分点 j，寻找让总步数最小的切分方案 for (int j = 1; j < i; ++j) { h4[i] = min(h4[i], 2 * h4[j] + h3[i-j]); } } cout << h4[20] << endl; // 输出 n=20 时的答案：297 return 0; }
h3 = [0] * 25 h4 = [0] * 25 # 1. 必须先预处理出所有 3 柱汉诺塔的答案，作为素材备用 h3[1] = 1 for i in range(2, 21): h3[i] = 2 * h3[i-1] + 1 # 2. 计算 4 柱汉诺塔的最优解 h4[1] = 1 for i in range(2, 21): h4[i] = float('inf') # 初始化为无穷大 # 核心：遍历所有的切分点 j，寻找让总步数最小的切分方案 for j in range(1, i): h4[i] = min(h4[i], 2 * h4[j] + h3[i-j]) print(h4[20]) # 输出 n=20 时的答案：297

三、时间优化的终极黑魔法：矩阵快速幂简介

普通的一维递推就算用了滚动数组，跑完一次 for 循环也需要经历 n 步。当竞赛题目的数据规模丧心病狂地给到了 n ≤ 1018 时，线性 for 循环将直接超时 (TLE) 崩溃。

🧠 时间压缩原理解析：
假设你在玩一个游戏，原本升到 16 级需要每次打一只怪（+1级），一共打 16 次。但如果你拿到了一件神奇道具，可以直接让你的等级翻倍呢？
你只需要：1级 → 翻倍变 2级 → 翻倍变 4级 → 翻倍变 8级 → 翻倍变 16级！原本需要 16 步的操作，现在只需要 4 步！这就是传说中的“快速幂”思想。

对于斐波那契数列，我们可以把它的加法关系打包成一个矩阵乘法：
[F(n), F(n-1)] = [F(n-1), F(n-2)] × 矩阵基底
通过不断对“矩阵基底”执行这种“翻倍”操作（矩阵快速幂），我们能以乘方的速度跨越时间，将令人绝望的 O(n) 时间复杂度瞬间压缩到闪电般的 O(log n)！

应用场景： 解决 n=1018 级别的斐波那契数列求值。

#include <iostream> using namespace std; const int MOD = 1000000007; // 定义 2x2 的矩阵结构体 struct Matrix { long long mat[2][2]; }; // 矩阵乘法函数：将矩阵 a 和 b 相乘 Matrix multiply(Matrix a, Matrix b) { Matrix res; for (int i = 0; i < 2; i++) { for (int j = 0; j < 2; j++) { res.mat[i][j] = 0; for (int k = 0; k < 2; k++) { res.mat[i][j] = (res.mat[i][j] + a.mat[i][k] * b.mat[k][j]) % MOD; } } } return res; } // 矩阵快速幂：求矩阵 a 的 n 次方 Matrix power(Matrix a, long long n) { Matrix res; // 构造单位矩阵（类似于普通数字里的 1） res.mat[0][0] = 1; res.mat[0][1] = 0; res.mat[1][0] = 0; res.mat[1][1] = 1; while (n > 0) { if (n % 2 == 1) res = multiply(res, a); a = multiply(a, a); n /= 2; } return res; } int main() { long long n = 1000000000000000000LL; Matrix base; base.mat[0][0] = 1; base.mat[0][1] = 1; base.mat[1][0] = 1; base.mat[1][1] = 0; Matrix result = power(base, n - 1); cout << result.mat[0][0] << endl; // 光速得出答案！ return 0; }
MOD = 1000000007 # 矩阵乘法函数 def multiply(A, B): res = [[0, 0], [0, 0]] for i in range(2): for j in range(2): for k in range(2): res[i][j] = (res[i][j] + A[i][k] * B[k][j]) % MOD return res # 矩阵快速幂函数 def power(A, n): res = [[1, 0], [0, 1]] # 单位矩阵 while n > 0: if n % 2 == 1: res = multiply(res, A) A = multiply(A, A) n //= 2 return res # 求解第 10^18 项 n = 1000000000000000000 base = [[1, 1], [1, 0]] result = power(base, n - 1) print(result[0][0]) # 光速得出答案！

🏁 侦探结案陈词：递推优化大盘点

递推算法并不只是写个循环那么简单，它是关于“如何优雅地处理数据”的艺术：

  • 【空间优化：滚动数组】 —— 只要递推式只和前几项有关，立刻扔掉历史包袱！空间复杂度从 O(n) 变 O(1)。
  • 【模型优化：多阶段拆解】 —— 遇到四柱汉诺塔这种复杂决策，学会用循环遍历“切分点”，寻找最优组合。
  • 【时间优化：矩阵快速幂】 —— 当 n 超过千万级甚至达到 1018，必须祭出“倍增”神技，将线性时间压缩到对数级别。

💡 竞赛秘籍： 在写任何递推前，先看一眼数据范围。107 用循环，1018 必用矩阵，内存限制 64MB 必须滚动！

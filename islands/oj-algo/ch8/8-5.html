8.5 实数二分 (Binary Search on Real Numbers)

🌌 告别离散，进入连续的星空：

在前面的学习中，我们切木头、分数组，面对的都是整数（离散的数字）。但在真实世界中，一根绳子的长度可以是 2.5 米，也可以是 2.513 米。当我们要寻找一个带有小数点的精确答案时，传统的整数二分模板就不管用了！

实数域的二分查找就像是用显微镜在星空中锁定一颗尘埃，它不再需要繁琐的“加一”或“减一”，而是展现出一种纯粹的逼近美学。

一、实数域二分搜索的框架 (8.5.1)

绝对的区别：不再加一减一！

在整数二分中，为了防止死循环，我们每次缩减区间都会写 left = mid + 1。但在实数二分中，3.1 和 3.2 之间还有无数个小数！如果你加了 1，就会完美错过正确答案。

所以，在实数二分中，如果左边界需要移动，我们直接写 left = mid；右边界移动直接写 right = mid。绝不加减任何数值！

二、精度控制与终止条件 (8.5.2)

📜 如何判定循环结束？

因为实数是无限细分的，left 和 right 永远不可能完全绝对相等。我们只有两种方式来结束循环：

  1. 条件判断法 (Eps 误差法)： 规定一个极小的误差值 eps (比如十的负六次方：1e-6)。只要 right - left > eps，我们就继续循环。
  2. 固定迭代法 (无脑循环法) 🌟推荐🌟： 别管误差了，只要硬性规定循环 100 次！每一次循环区间减半，100 次循环后，区间会被缩小到原来的 2 的 100 次方分之一，精度足以满足任何极其变态的算法题，而且绝对不会因为浮点数误差陷入死循环！

三、求方程近似根：寻找平方根 (8.5.3 实例 1)

假设我们不用自带的数学库，要求你自己计算一个数 N 的算术平方根，要求精确到小数点后 5 位。我们可以直接猜答案！如果猜的数的平方大于 N，说明猜大了；反之则猜小了。

图解 2：寻找 10 的平方根追踪
C++ 计算平方根 (固定 100 次迭代)
#include <iostream> #include <iomanip> // 用于控制输出精度 using namespace std; double findSqrt(double n) { // 处理大于1和小于1的情况，保证边界正确 double left = 0; double right = max(1.0, n); // 🌟黄金推荐：固定循环 100 次，精度远超 1e-15，绝不死循环 for (int i = 0; i < 100; i++) { double mid = left + (right - left) / 2.0; // 实数除法 if (mid * mid <= n) { left = mid; // 猜小了或刚好，去右边找。不加一！ } else { right = mid; // 猜大了，去左边找。不减一！ } } return left; // 最后 left 和 right 几乎黏在一起 } int main() { // fixed 和 setprecision 保证输出固定的小数位数 cout << fixed << setprecision(5) << findSqrt(10.0) << endl; return 0; }
Python 计算平方根 (eps 误差法)
def find_sqrt(n): left = 0.0 right = max(1.0, n) # 误差阈值设为 1e-7，要求精度是 5 位，设到 7 位更保险 eps = 1e-7 # 只要区间长度大于误差，就继续切分 while right - left > eps: mid = left + (right - left) / 2.0 # 必须是真正的除法 / if mid * mid <= n: left = mid # 猜小了，不加一！ else: right = mid # 猜大了，不减一！ return left # 测试：格式化输出保留 5 位小数 print(f"{find_sqrt(10.0):.5f}")

四、进阶挑战：最大化平均值 (8.5.3 实例 2)

题目： 你有 N 个物品，每个物品有一个“价值”和一个“重量”。现在要你挑选出 K 个物品，使得这 K 个物品的 (总价值除以总重量) 的平均值尽可能大。求这个最大平均值。

这道题被称为“0-1分数规划”，极具代表性。我们不知道最优平均值是多少，那就直接二分猜一个平均值 mid！

💡 数学小戏法：

假设我们猜平均值能达到 mid，也就是要求： (价值总和) 除以 (重量总和) 大于等于 mid。

把分母乘过去移项，就会变成一个神奇的公式：每一项的 (价值 减去 mid 乘以 重量) 累加起来必须大于等于 0！

有了这个公式，我们只要把每个物品按 (价值 减去 mid 乘以 重量) 算出一个新分数，把分数最高的 K 个物品挑出来。如果它们的和大于等于 0，说明这个 mid 是可行的！

图解 3：最大化平均值的判定黑盒转换
C++ 最大化平均值 (0-1 分数规划)
#include <iostream> #include <vector> #include <algorithm> using namespace std; // 验证平均值 mid 是否可行 bool check(double mid, const vector<double>& v, const vector<double>& w, int K) { vector<double> scores(v.size()); for (int i = 0; i < v.size(); i++) { // 计算每个物品的新分数：价值 - mid * 重量 scores[i] = v[i] - mid * w[i]; } // 将新分数从大到小排序 sort(scores.rbegin(), scores.rend()); double sum = 0; // 贪心挑选分数最高的前 K 个物品 for (int i = 0; i < K; i++) { sum += scores[i]; } // 如果最大的 K 个累加和大于等于 0，说明这个 mid 是可行的！ return sum >= 0; } double maxAverage(vector<double>& v, vector<double>& w, int K) { double left = 0.0, right = 1000000.0; // 假定一个足够大的上界 // 实数二分，固定迭代 100 次 for (int i = 0; i < 100; i++) { double mid = left + (right - left) / 2.0; if (check(mid, v, w, K)) { left = mid; // 可以达到这个平均值，尝试挑战更高的 (Yes) } else { right = mid; // 达不到，只能降低标准 (No) } } return left; }
Python 最大化平均值 (0-1 分数规划)
def check(mid, values, weights, K): # 计算每个物品的新分数：价值 - mid * 重量 scores = [v - mid * w for v, w in zip(values, weights)] # 将新分数从大到小排序 scores.sort(reverse=True) # 贪心挑选分数最高的前 K 个物品，判断总和是否 >= 0 return sum(scores[:K]) >= 0 def max_average(values, weights, K): left, right = 0.0, 1000000.0 # 假定一个足够大的上界 # 实数二分，固定迭代 100 次以确保精度 for _ in range(100): mid = left + (right - left) / 2.0 if check(mid, values, weights, K): left = mid # 可以达到，尝试更高的平均值 (Yes) else: right = mid # 达不到，降低标准 (No) return left # 测试样例：3个物品挑2个 v = [2.0, 5.0, 2.0] w = [1.0, 2.0, 2.0] print(f"最大平均值: {max_average(v, w, 2):.4f}")

🎉 总结：实数二分的致命口诀

只要是看到题目要求输出带有小数点的最大值或最小值，毫不犹豫使用实数二分！
写代码时请在心里默念：“除以二点零，更新不加一，循环一百次，精度保第一！”

8.4 二分答案 (Binary Search on Answer)

🎯 从“寻找数字”到“猜测结局”：

普通的二分查找，是在一个已知的数组里寻找某个数字是否存在。但有一种更高级的魔法：我们甚至不知道数组是什么，我们直接在答案的可能范围里“瞎猜”！

猜出一个答案后，我们去验证这个答案“行不行得通”。如果行得通，我们就尝试猜一个“更好”的答案；如果行不通，我们就退而求其次。这种思想，被称为二分答案。

一、最优解到判定问题的转化与单调性 (8.4.1 & 8.4.2)

核心理念转化：
与其绞尽脑汁去算“最优解到底是多少？”，不如转化成一个简单的选择题：“如果我要求答案是 X，能不能做到？” 只要写出一个 check(x) 函数来验证即可。

二分的绝对前提：单调性！
你猜的答案范围必须具有“单调性”。比如我们在玩游戏调整难度，难度 1 到 5 你都能通关（全是 Yes），难度 6 及以上你就通关不了了（全是 No）。这种 YYYYYNNNN 的分布，就完美契合了二分查找的条件！

二、整数二分答案的两种经典写法 (8.4.3)

💡 黄金守则复习：

我们依然使用上一节学过的万能模板：使用 ans 变量记录合法解，然后往可能更优的方向继续逼近！绝生死循环！

情况一：求“最小可行解” (NNNNYYYY 模型)

举例：你想以最小的速度吃完所有的香蕉。速度太慢（No），速度够快（Yes）。你要找的是第一个 Yes。

图解 2：最小可行解寻找过程 (找第一个 Y)
C++ 最小可行解 (找第一个 Y)
int findMinFeasible() { int left = MIN_VAL, right = MAX_VAL; int ans = -1; while (left <= right) { int mid = left + (right - left) / 2; if (check(mid)) { // 如果当前速度可以完成任务 (Yes) ans = mid; // 记录这个合法的速度 right = mid - 1; // 贪心地尝试更小的速度 (向左找) } else { left = mid + 1; // 速度太慢了 (No)，必须加快速度 (向右找) } } return ans; }
Python 最小可行解 (找第一个 Y)
def find_min_feasible(): left, right = MIN_VAL, MAX_VAL ans = -1 while left <= right: mid = left + (right - left) // 2 if check(mid): # 如果当前速度可以完成 (Yes) ans = mid # 记录合法答案 right = mid - 1 # 贪心地尝试更小的速度 else: left = mid + 1 # 速度太慢 (No)，必须加快 return ans

情况二：求“最大可行解” (YYYYNNNN 模型)

举例：你想把一根长木头切成相等的小段，并且每段尽可能长。太长了切不出足够数量（No），短一点可以（Yes）。你要找的是最后一个 Yes。

图解 3：最大可行解寻找过程 (找最后一个 Y)
C++ 最大可行解 (找最后一个 Y)
int findMaxFeasible() { int left = MIN_VAL, right = MAX_VAL; int ans = -1; while (left <= right) { int mid = left + (right - left) / 2; if (check(mid)) { // 如果当前长度可以切出 (Yes) ans = mid; // 记录这个合法的长度 left = mid + 1; // 贪心地尝试更长的木头 (向右找) } else { right = mid - 1; // 太长了切不出 (No)，必须缩短 (向左找) } } return ans; }
Python 最大可行解 (找最后一个 Y)
def find_max_feasible(): left, right = MIN_VAL, MAX_VAL ans = -1 while left <= right: mid = left + (right - left) // 2 if check(mid): # 如果当前长度可以切出 (Yes) ans = mid # 记录合法答案 left = mid + 1 # 贪心地尝试更长的木头 else: right = mid - 1 # 太长了切不出 (No)，必须缩短 return ans

三、典型应用：最大化最小值 / 最小化最大值 (8.4.4)

在算法竞赛中，只要题目里出现极其拗口的“最大化最小值”或者“最小化最大值”，99%都是在考察二分答案！

🐄 经典案例：愤怒的牛 (Aggressive Cows)

题目： 一条直线上有若干个隔间，你有 C 头暴躁的牛。你需要把牛放进隔间里，为了防止它们打架，你想让任意两头牛之间的最小距离尽可能大。求这个最大的最小距离。

翻译为人话： 我们不知道答案，所以直接猜一个距离 mid，判断：“我能不能让所有牛之间的距离都至少是 mid？” 如果可以，说明这个距离行得通，我就试着挑战更大的距离！这就变成了一个标准的求最大可行解（找最后一个Y）问题！

图解 4：牛棚分布检查 (check 函数的贪心逻辑)
C++ 愤怒的牛 (最大化最小值)
#include <iostream> #include <vector> #include <algorithm> using namespace std; // 判定函数：以 mid 为最小间距，能否放下 C 头牛？ bool check(int mid, const vector<int>& stalls, int C) { int count = 1; // 第一头牛放在第一个隔间 int last_pos = stalls[0]; for (int i = 1; i < stalls.size(); i++) { // 只有当当前隔间与上一个放牛的隔间距离 >= mid 时，才放牛 if (stalls[i] - last_pos >= mid) { count++; last_pos = stalls[i]; // 更新上一头牛的位置 if (count >= C) return true; // 牛全放进去了！(Yes) } } return false; // 隔间跑完了，牛没放完 (No) } int maxMinDistance(vector<int>& stalls, int C) { sort(stalls.begin(), stalls.end()); // 隔间必须先排序！ // 答案的可能范围：最小距离是 1，最大可能距离是首尾之差 int left = 1; int right = stalls.back() - stalls.front(); int ans = -1; while (left <= right) { int mid = left + (right - left) / 2; if (check(mid, stalls, C)) { ans = mid; // 记录这个成功放牛的间距 left = mid + 1; // 贪心地去挑战更大的间距！ } else { right = mid - 1; // 间距太大了放不下，必须缩小间距 } } return ans; }
Python 愤怒的牛 (最大化最小值)
def check(mid, stalls, C): count = 1 # 第一头牛放在第一个隔间 last_pos = stalls[0] for i in range(1, len(stalls)): # 只有距离 >= mid 时，才放牛 if stalls[i] - last_pos >= mid: count += 1 last_pos = stalls[i] # 更新上一头牛的位置 if count >= C: return True # 牛全放进去了！(Yes) return False # 隔间跑完了，牛没放完 (No) def max_min_distance(stalls, C): stalls.sort() # 隔间必须先排序！ # 答案的可能范围 left = 1 right = stalls[-1] - stalls[0] ans = -1 while left <= right: mid = left + (right - left) // 2 if check(mid, stalls, C): ans = mid # 记录这个成功放牛的间距 left = mid + 1 # 贪心地去挑战更大的间距！ else: right = mid - 1 # 间距太大了放不下，必须缩小 return ans # 测试代码 stalls = [1, 2, 8, 4, 9] C = 3 print(f"最大化的最小间距是: {max_min_distance(stalls, C)}") # 输出 3

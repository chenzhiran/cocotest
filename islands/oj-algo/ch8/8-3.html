8.3 二分查找的常见变种

🎯 当数组里出现“双胞胎”：

基础的二分查找就像一个直男，只要碰到目标数字就立刻高兴地返回了。但问题是，如果数组里有一大堆相同的数字（比如 ，寻找 5），基础二分可能返回中间那个 5，也可能返回旁边那个 5。它的行为是不可预测的！

在实际做题中，我们往往需要精准地找到：第一个 5 出现的位置 或者 最后一个 5 出现的位置。这就需要对二分魔法进行变种升级！

一、查找第一个等于目标值的位置 (8.3.1)

核心思路 (Lower Bound)： 当我们切到中间，发现 arr[mid] == target 时，绝不提前高兴，绝不提前返回！

我们用一个小本本（一个叫 ans 的变量）记录下这个 mid 的位置。然后我们在心里想：“虽然找到了，但左边可能还有更靠前的兄弟呢！” 于是，我们把右边界 right 强行拉过来（right = mid - 1），继续在左半区疯狂逼近！

图解 1：寻找“第一个 5”的贪婪逼近
C++ 第一个等于目标值的位置
int findFirst(const vector<int>& arr, int target) { int left = 0, right = arr.size() - 1; int ans = -1; // 小本本，默认记作找不到 while (left <= right) { int mid = left + (right - left) / 2; if (arr[mid] == target) { ans = mid; // 记录当前找到的位置 right = mid - 1; // 贪心！继续逼近左边寻找更早的 } else if (arr[mid] > target) { right = mid - 1; // 偏大，去左边找 } else { left = mid + 1; // 偏小，去右边找 } } return ans; // 最后小本本上记的就是最左边的下标！ }
Python 第一个等于目标值的位置
def find_first(arr, target): left, right = 0, len(arr) - 1 ans = -1 # 小本本，默认记作找不到 while left <= right: mid = left + (right - left) // 2 if arr[mid] == target: ans = mid # 记录当前找到的位置 right = mid - 1 # 贪心！继续逼近左边寻找更早的 elif arr[mid] > target: right = mid - 1 # 偏大，去左边找 else: left = mid + 1 # 偏小，去右边找 return ans # 最后小本本上记的就是最左边的下标！

二、查找最后一个等于目标值的位置 (8.3.2)

核心思路： 举一反三！当我们发现 arr[mid] == target 时，我们依然用小本本记录下 ans = mid，但这次我们心里想的是：“右边可能还有更靠后的兄弟呢！” 于是，我们把左边界 left 强行拉过来（left = mid + 1），继续在右半区疯狂探测！

图解 2：寻找“最后一个 5”的贪婪逼近
C++ 最后一个等于目标值的位置
int findLast(const vector<int>& arr, int target) { int left = 0, right = arr.size() - 1; int ans = -1; while (left <= right) { int mid = left + (right - left) / 2; if (arr[mid] == target) { ans = mid; // 记录当前找到的位置 left = mid + 1; // 【唯一区别】继续逼近右边！ } else if (arr[mid] > target) { right = mid - 1; } else { left = mid + 1; } } return ans; }
Python 最后一个等于目标值的位置
def find_last(arr, target): left, right = 0, len(arr) - 1 ans = -1 while left <= right: mid = left + (right - left) // 2 if arr[mid] == target: ans = mid # 记录当前找到的位置 left = mid + 1 # 【唯一区别】继续逼近右边！ elif arr[mid] > target: right = mid - 1 else: left = mid + 1 return ans

三、查找第一个大于目标值的位置 (8.3.3)

核心思路 (Upper Bound)： 这次我们要找的数字，严格要求比 target 大！如果数组里有 ，寻找大于 5 的第一个数，答案是下标为 4 的 8。

规则很简单：只要 arr[mid] > target，这个数字就是合法的候选人！我们把它用小本本记下来 ans = mid，但因为我们要找“第一个”，所以我们还要向左逼近 right = mid - 1。如果 arr[mid] <= target，那它太小了不合格，只能去右边找 left = mid + 1。

图解 3：寻找“第一个大于 5”的位置
C++ 第一个大于目标值 (upper_bound)
int findFirstGreater(const vector<int>& arr, int target) { int left = 0, right = arr.size() - 1; int ans = -1; while (left <= right) { int mid = left + (right - left) / 2; if (arr[mid] > target) { // 合格！比 target 大 ans = mid; // 记在小本本上 right = mid - 1; // 去左边找有没有更早的合格者 } else { // 太小或等于都不行 left = mid + 1; // 去右边找更大的 } } return ans; }
Python 第一个大于目标值
def find_first_greater(arr, target): left, right = 0, len(arr) - 1 ans = -1 while left <= right: mid = left + (right - left) // 2 if arr[mid] > target: # 合格！比 target 大 ans = mid # 记在小本本上 right = mid - 1 # 去左边找有没有更早的合格者 else: # 太小或等于都不行 left = mid + 1 # 去右边找更大的 return ans

四、特殊场景：峰值查找 (8.3.4)

⛰️ 不是排好序才能二分！

二分查找的真正精髓在于“抛弃一半”，而不是“数组必须有序”。如果数组是一个“山脉形状”（先升后降），我们要找山顶。只要随意挑一个位置，看看它是上坡还是下坡，就能决定山顶在哪一边！

图解 4：在山脉中寻找峰值
C++ 寻找峰值
int findPeakElement(const vector<int>& arr) { int left = 0, right = arr.size() - 1; while (left < right) { // 注意这里没有等于号 int mid = left + (right - left) / 2; if (arr[mid] < arr[mid + 1]) { // 上坡！说明峰顶在右边 left = mid + 1; } else { // 下坡！说明峰顶在左边 (包含 mid 本身可能就是峰顶) right = mid; } } return left; // 最终 left 和 right 会相遇在峰顶 }
Python 寻找峰值
def find_peak_element(arr): left, right = 0, len(arr) - 1 while left < right: # 注意这里没有等于号 mid = left + (right - left) // 2 if arr[mid] < arr[mid + 1]: # 上坡！说明峰顶在右边 left = mid + 1 else: # 下坡！说明峰顶在左边 (包含 mid 本身) right = mid return left # 最终 left 和 right 会相遇在峰顶

五、终极宝典：万能二分模板 (8.3.5)

📜 拒绝死循环的黄金模板 (记录答案法)

如果你总是在纠结 while(left <= right) 还是 <？ 纠结 right = mid 还是 mid - 1？ 请把下面的通用模板刻在脑子里：

  • 永远使用 while (left <= right)。
  • 内部一旦遇到满足要求/疑似答案的 mid，就用 ans = mid 记下来。
  • 收缩边界时，永远是 left = mid + 1 和 right = mid - 1（斩钉截铁，绝不拖泥带水，彻底杜绝死循环）。
  • 最后返回 ans。

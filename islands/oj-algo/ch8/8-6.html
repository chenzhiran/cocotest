8.6 倍增法初步 (Binary Lifting)

🚀 呈指数级爆发的跳跃：

普通的枚举像走路，一步一步走；二分查找像是从中间一刀切。而倍增法，就像是在施展“瞬间移动”！

它的核心思想是：每次跨越的步长都是前一次的两倍（走 1 步，再走 2 步，再走 4 步，8 步，16 步...）。当发现跳得太远时，就倒退回去，换更小的步长继续跳。它和二分法是同父异母的兄弟，但在某些特殊的场景下，倍增法比二分法更加灵活和强大！

一、用倍增法探索未知边界 (8.6.1 & 8.6.2)

困境：无法二分？
二分查找有一个致命前提：你必须知道搜索的“右边界（Right）”到底在哪。但如果数组是无限长的，或者右边界极大，你根本不知道从哪里开始二分！

💡 倍增策略：探路与回头！
1. 探路阶段： 步长从 1 开始。当前位置符合条件，步长就乘 2 往前跳（1，2，4，8...），直到跳到某个位置不符合条件为止。此时我们瞬间锁定了一个大致的区间！
2. 精搜阶段： 在最后跨越的这个区间里，使用标准的二分查找，精准锁定最终目标。

图解 1：倍增法寻找未知边界的“探路过程”
C++ 倍增探路 + 二分查找
int findUpperBoundary() { int pos = 0; int step = 1; // 1. 倍增探路阶段：每次步长乘 2 while (check(pos + step)) { pos += step; // 向前跳跃 step *= 2; // 步长翻倍：1, 2, 4, 8... } // 此时，pos 是合法的，但 pos + step 是不合法的。 // 真正的边界被锁定在区间 [pos, pos + step] 之间！ int left = pos; int right = pos + step; int ans = pos; // 2. 精确搜索阶段：标准的二分答案模板 while (left <= right) { int mid = left + (right - left) / 2; if (check(mid)) { ans = mid; // 记录合法解 left = mid + 1; // 继续向右探索 } else { right = mid - 1; // 太靠右了，往左退 } } return ans; }
Python 倍增探路 + 二分查找
def find_upper_boundary(): pos = 0 step = 1 # 1. 倍增探路阶段：每次步长乘 2 while check(pos + step): pos += step # 向前跳跃 step *= 2 # 步长翻倍：1, 2, 4, 8... # 此时，真实的边界被锁定在区间 [pos, pos + step] 内 left = pos right = pos + step ans = pos # 2. 精确搜索阶段：标准的二分答案模板 while left <= right: mid = left + (right - left) // 2 if check(mid): ans = mid # 记录合法解 left = mid + 1 # 继续向右探索 else: right = mid - 1 # 太靠右了，往左退 return ans

二、经典实例：快速幂算法 (8.6.3)

💡 为什么要用倍增算乘方？

如果要计算 3 的 11 次方，普通方法是把 3 乘 11 次。如果指数是 10 亿呢？程序会跑很久！

倍增思想（快速幂）的魔法在于：我们不要 1 个 1 个地乘，我们对底数进行平方倍增。把 11 拆成二进制的构成：11 = 8 + 2 + 1。所以我们只需要计算：(3的8次方) 乘 (3的2次方) 乘 (3的1次方)。乘法次数从 11 次瞬间降到了 3 次！极大地提高了效率！

图解 2：快速幂算法拆解 (把 11 次方切分为 8+2+1)
C++ 快速幂算法
long long fastPower(long long base, long long exp) { long long result = 1; while (exp > 0) { // 1. 如果当前二进制的最后一位是 1 if (exp % 2 == 1) { // 把当前倍增的 base 乘入结果中 result = result * base; } // 2. 底数进行自我翻倍 (倍增的核心！) // 比如：3 变成 9，9 变成 81，81 变成 6561... base = base * base; // 3. 指数除以 2，相当于把二进制向右移一位，丢掉处理完的末尾 exp = exp / 2; } return result; }

三、一维倍增与树上倍增简介 (8.6.4)

倍增思想在高级算法中最闪耀的舞台，是用来解决“寻找最近公共祖先 (LCA)”的问题。试想一下，如果一棵家谱树非常深，你要找两个人的共同祖先，一步一步往上爬太慢了。

于是我们提前做一张“倍增小抄表”：记录下每个人往上跳 1 步、2 步、4 步、8 步分别是谁。当需要寻找时，我们直接按照 2 的指数级步长“大跨步”向上飞跃！把时间复杂度从 $O(N)$ 降到了 $O(\log N)$。

图解 3：树上倍增寻找共同祖先 (大步跨越)

🌲 魔法扩展提示：

树上倍增和一维倍增的底层原理是建立一个二维数组 f[i][k]。它代表：从位置 i 开始，向前跳跃 2的k次方 步后到达的位置。

它的递推公式极其优美：跳 2的k次方步，等价于先跳 2的(k-1)次方步，然后再接着跳 2的(k-1)次方步！ 这就是高级算法中常用的“状态转移方程”。

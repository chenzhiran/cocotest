8.2 二分查找（递归/非递归）

🎯 猜数字游戏：

心里想一个 1 到 100 之间的数字，让我来猜。如果我猜 50，你告诉我“太小了”，那么我瞬间就能淘汰掉 1 到 50 所有的数字，接下来我只需要在 51 到 100 之间猜（我可能会猜 75）。

这就是二分查找的灵魂！每一次探测，都无情地抛弃掉一半的错误答案。哪怕是在 40 亿个数字里找目标，最多也只需要猜 32 次！

一、有序序列中的二分查找原理 (8.2.1)

绝对前提： 数组必须是排好序的（从小到大，或从大到小）。如果乱七八糟，就无法判断目标在左半边还是右半边。

三大核心探子：

  • 左边界 (Left)：搜索范围的起点。
  • 右边界 (Right)：搜索范围的终点。
  • 中间位置 (Mid)：左边界和右边界的正中间。每次都拿 Mid 指向的值去和目标对比！

二、递归实现的写法与调用 (8.2.2)

递归实现非常符合人类直觉：把当前的区间切成两半，检查中间的数字。如果不相等，就让函数“自己调用自己”，直接去查左半边或者右半边。

C++ 二分查找 (递归版)
#include <vector> using namespace std; int binarySearchRec(const vector<int>& arr, int left, int right, int target) { // 1. 递归终止条件：范围错乱，说明找不到！ if (left > right) return -1; // 2. 计算中间位置，防止 left+right 溢出 int mid = left + (right - left) / 2; // 3. 中大奖了！直接返回下标 if (arr[mid] == target) { return mid; } // 4. 如果中间数字比目标大，目标只可能在左边 else if (arr[mid] > target) { return binarySearchRec(arr, left, mid - 1, target); } // 5. 如果中间数字比目标小，目标只可能在右边 else { return binarySearchRec(arr, mid + 1, right, target); } }
Python 二分查找 (递归版)
def binary_search_rec(arr, left, right, target): # 1. 递归终止条件：范围错乱，说明找不到！ if left > right: return -1 # 2. 计算中间位置 (整除) mid = left + (right - left) // 2 # 3. 中大奖了！直接返回下标 if arr[mid] == target: return mid # 4. 如果中间数字比目标大，往左边找 elif arr[mid] > target: return binary_search_rec(arr, left, mid - 1, target) # 5. 如果中间数字比目标小，往右边找 else: return binary_search_rec(arr, mid + 1, right, target) # 调用示例 arr = [3, 9, 14, 27, 38, 43, 55, 62, 70] print(binary_search_rec(arr, 0, len(arr)-1, 62))

三、非递归实现（循环写法）与边界控制 (8.2.3)

💡 为什么推荐用循环写二分查找？

虽然递归看起来很优雅，但如果数组长达 10 亿级别，深层的递归调用会消耗栈内存，甚至可能导致程序崩溃（Stack Overflow）。而用一个 while 循环来移动 Left 和 Right 指针，不仅内存零消耗，速度也更快！在竞赛和工程中，我们99%都使用循环写法。

C++ 二分查找 (循环版)
int binarySearchIter(const vector<int>& arr, int target) { int left = 0; int right = arr.size() - 1; // 循环条件：左边界还没有越过右边界 // 必须要有 = 号！否则只有一个元素的区间会被漏掉！ while (left <= right) { int mid = left + (right - left) / 2; if (arr[mid] == target) { return mid; // 找到了 } else if (arr[mid] > target) { // 既然 mid 偏大，说明答案一定在 mid 的左边 // 不要再包含 mid 了，直接将右边界设为 mid - 1 right = mid - 1; } else { // 既然 mid 偏小，答案在右边 // 直接将左边界设为 mid + 1 left = mid + 1; } } return -1; // 循环结束也没找到 }
Python 二分查找 (循环版)
def binary_search_iter(arr, target): left = 0 right = len(arr) - 1 # 必须包含等于号，因为 left 和 right 指向同一个数时也要查！ while left <= right: mid = left + (right - left) // 2 if arr[mid] == target: return mid elif arr[mid] > target: # mid 偏大，抛弃包括 mid 在内的右半部分 right = mid - 1 else: # mid 偏小，抛弃包括 mid 在内的左半部分 left = mid + 1 return -1 # 找不到

四、循环不变式与常见死循环陷阱 (8.2.4)

💀 为什么我的二分查找卡死了？

很多初学者在写二分查找时，最常犯的错误就是把 left = mid + 1 写成了 left = mid！这样一改，一旦当 left 和 right 只相差 1 时，程序就会永远卡在原地，彻底陷入死循环！

图解 4：死循环陷阱揭秘 (致命的截断)

💡 黄金守则（循环不变式）：

我们在设计二分循环时，必须保证：每一次循环结束，搜索区间的长度必须严格缩小！

既然我们已经在这一轮判断过 arr[mid] 不是我们要找的数字了，那么下一轮的搜索范围就完全没有必要再把 mid 包含进去了！因此，正确的写法永远是跳过它：left = mid + 1 或者 right = mid - 1！这不仅缩小了区间，还彻底杜绝了死循环！

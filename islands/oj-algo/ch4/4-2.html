4.2 循环枚举

循环枚举是最基础的枚举形式，通过 for 循环遍历变量的所有可能取值。在编程竞赛中，几乎所有的枚举问题都始于一个或多个嵌套的循环。

4.2.1 单层循环枚举

知识点讲解

  • 适用场景： 解空间由一个变量完全确定，且该变量取值范围连续、规模可控。
  • 核心步骤：
    1. 确定变量的合理范围（下界与上界）。
    2. 用 for 循环遍历该范围内的每一个值。
    3. 在循环体内判断该值是否满足题目条件。
  • 复杂度： 若循环次数为 N，则复杂度为 O(N)。

例题 1：水仙花数

求三位数中，各位数字立方和等于该数本身的数。

💡 思路分析：

  • 确定范围： 三位数为 100 ~ 999。
  • 提取各位数字： 个位 num % 10，十位 (num / 10) % 10，百位 num / 100。
  • 检验条件： 百位³ + 十位³ + 个位³ == 原数。
#include <iostream> using namespace std; int main() { for (int num = 100; num <= 999; ++num) { int a = num / 100; // 百位 int b = (num / 10) % 10; // 十位 int c = num % 10; // 个位 if (a*a*a + b*b*b + c*c*c == num) { cout << num << endl; } } return 0; }
for num in range(100, 1000): a = num // 100 # 百位 b = (num // 10) % 10 # 十位 c = num % 10 # 个位 if a**3 + b**3 + c**3 == num: print(num)

运行结果：153、370、371、407

4.2.2 多层循环枚举

当问题的解由多个相互关联的变量决定时，需要使用嵌套循环遍历所有可能的变量组合。

知识点讲解

  • 嵌套层数： 通常与变量的个数相同。
  • 循环顺序： 最外层循环对应取值范围最小的变量（有利于后续优化）。
  • 复杂度计算： 各层循环次数相乘。例如三重循环各 100 次，则总次数为 1003 = 106。
  • 优化原则：能算出来的就不要枚举！ 若变量之间存在线性关系，应通过等式减少一层循环。

例题 2：百钱百鸡

公鸡 5 钱一只，母鸡 3 钱一只，小鸡 1 钱三只。用 100 钱买 100 只鸡，问公鸡、母鸡、小鸡各多少？

🧠 思路分析：

朴素三重循环： 循环次数 1013 ≈ 106，虽然可过，但不够优雅。

优化分析：

  • 公鸡最多 100 / 5 = 20 只，母鸡最多 100 / 3 ≈ 33 只。
  • 一旦确定了公鸡 x 和母鸡 y，小鸡 z = 100 - x - y 就被唯一确定。
  • 只需检查金额条件，并确保 z 是 3 的倍数。

优化后复杂度： 21 × 34 ≈ 714 次，效率提升数千倍！

#include <iostream> using namespace std; int main() { for (int x = 0; x <= 20; ++x) { // 公鸡数 for (int y = 0; y <= 33; ++y) { // 母鸡数 int z = 100 - x - y; // 小鸡数 // 检查金额是否等于100，且小鸡数为3的倍数 if (z % 3 == 0 && 5*x + 3*y + z/3 == 100) { cout << "公鸡：" << x << " 母鸡：" << y << " 小鸡：" << z << endl; } } } return 0; }
for x in range(0, 21): # 公鸡最多20只 for y in range(0, 34): # 母鸡最多33只 z = 100 - x - y # 小鸡数唯一确定 if z % 3 == 0 and 5*x + 3*y + z//3 == 100: print(f"公鸡：{x} 母鸡：{y} 小鸡：{z}")

运行结果：
公鸡：0 母鸡：25 小鸡：75
公鸡：4 母鸡：18 小鸡：78
公鸡：8 母鸡：11 小鸡：81
公鸡：12 母鸡：4 小鸡：84

拓展例题：勾股数问题

找出 1 ~ N 内所有满足 a2 + b2 = c2 且 a < b < c 的整数解。

✨ 思路分析：

  • 解空间： 三个变量 a, b, c 均不超过 N。
  • 有序约束： a < b < c，可利用该性质减少循环次数。外层枚举 a，内层枚举 b（从 a + 1 开始）。
  • 优化掉一层： c 通过公式 c = √(a2 + b2) 计算，然后检查是否为整数且不超过 N。
  • 复杂度： 约 O(N2)，当 N = 1000 时约 5 × 105 次，完全可接受。
#include <iostream> #include <cmath> using namespace std; int main() { int N; cin >> N; for (int a = 1; a <= N; ++a) { for (int b = a + 1; b <= N; ++b) { int c2 = a*a + b*b; int c = (int)sqrt(c2); // 检验 c 是否为整数，且不超过 N if (c <= N && c*c == c2) { cout << a << " " << b << " " << c << endl; } } } return 0; }
import math N = int(input()) for a in range(1, N + 1): for b in range(a + 1, N + 1): c2 = a*a + b*b c = math.isqrt(c2) # Python 3.8+ 推荐使用 math.isqrt if c <= N and c * c == c2: print(a, b, c)

本节总结

类型 关键技巧 复杂度 典型题目
单层循环 精确确定上下界 O(N) 水仙花数、素数判定
多层循环 利用等式减少循环层数 O(N2) 或更低 百钱百鸡、勾股数
优化思想 能算的不枚举，能剪枝的早排除 大幅降低常数 所有枚举题通用

4.4 排列枚举

👋 魔法师语录：

给定 n 个不同的元素，把它们按照所有的顺序重新排队，就是排列枚举。总共有 n! (n的阶乘) 种排法。
竞赛中常用于解决顺序相关的组合问题，比如密码破译、旅行商路线(TSP)、八皇后排布等。

⚠️ 复杂度警示：阶乘爆炸！

排列数的增长速度极其可怕：

  • 3! = 6
  • 8! = 40,320
  • 10! = 3,628,800
  • 12! ≈ 4.79 × 108 (即将超时！)

竞赛建议：
✔️ n ≤ 10，放心大胆枚举。
⚠️ n = 11 或 12，卡着时间线，需谨慎优化。
❌ n ≥ 13，绝对不能纯暴力全排列，必须换思路！

4.4.1 使用自带魔法（库函数法）

🪄 字典序魔法

知识点讲解

  • C++ 魔法： std::next_permutation 定义在 <algorithm> 中，它将当前序列变成字典序的下一个排列。
  • Python 魔法： 标准库 itertools.permutations 提供迭代器，直接生成所有排列。
  • ⚠️ 致命前提（针对C++）： 使用前必须先升序排序！否则你只能得到从当前顺序往后的排列，前面的就漏掉了。
#include <iostream> #include <algorithm> #include <vector> using namespace std; int main() { vector<int> v = {1, 2, 3}; sort(v.begin(), v.end()); // ⚠️ 必须先排序！ do { for (int x : v) cout << x << " "; cout << endl; } while (next_permutation(v.begin(), v.end())); return 0; }
from itertools import permutations v = [1, 2, 3] # 自动按元素位置生成所有全排列，无需提前排序即可得到所有组合 # 但如果要严格按照数值字典序输出，建议先对 v 排序 for perm in permutations(v): print(' '.join(map(str, perm)))

4.4.2 手写全排列（DFS 回溯）

🌳 图解：排列的递归填坑

知识点讲解

为什么放着魔法库函数不用，非要手写递归？

  • 很多难题需要在排列刚生成一半时就判断不行，从而提前剪枝（比如八皇后）。库函数没法中途停手，会跑死电脑。
  • 手写递归是理解回溯算法(Backtracking)的基石。

核心步骤：

  1. 准备一个数组 a 存放当前填好的排列，准备一个布尔数组 used 标记数字是否已经被用过。
  2. 递归函数 dfs(k) 表示当前正在填第 k 个坑位。
  3. 枚举所有数字：如果没用过，就填入坑位，标记为已用，然后 dfs(k+1) 去填下一个坑。
  4. 最关键一步： 从 dfs(k+1) 回来后，必须把刚填的数字“拔出来”，标记恢复为未使用（这叫回溯），给后面的数字腾出机会尝试。
#include <iostream> using namespace std; int n = 3; int a[20]; bool used[20] = {false}; void dfs(int k) { if (k == n) { // 坑填满了，输出当前排列 for (int i = 0; i < n; ++i) cout << a[i] << " "; cout << endl; return; } // 尝试所有的候选项 1 到 n for (int i = 1; i <= n; ++i) { if (!used[i]) { used[i] = true; // 标记占位 a[k] = i; // 填入坑位 dfs(k + 1); // 去填下一个坑 used[i] = false; // ⚠️ 回溯：拔出数字，恢复未用状态 } } } int main() { dfs(0); return 0; }
n = 3 a = [0] * n used = [False] * (n + 1) # 数字从1开始，大小设为 n+1 def dfs(k): if k == n: print(' '.join(map(str, a))) return for i in range(1, n + 1): if not used[i]: used[i] = True a[k] = i dfs(k + 1) used[i] = False # ⚠️ 回溯：拔出数字，恢复原状 dfs(0)

🚀 进阶实战：带剪枝的排列 —— 八皇后问题

问题： 在 8×8 棋盘上放 8 个皇后，互相不能打架（同行、同列、对角线都不行）。

怎么转化为排列问题？
肯定不能把两个皇后放同一行。所以每行放一个，用 pos[r] 表示第 r 行的皇后放在了第几列。因为同列也不能放，所以 pos 数组就是 1~8 的一个全排列！

剪枝条件： 摆放的过程中，随时检查对角线。如果对角线冲突了，当前分支直接咔嚓✂️，不往下摆了！

(对角线神奇规律：左下到右上，行+列的和固定；左上到右下，行-列的差固定)

4.4.3 两种方法大比拼

方法 优点 缺点 适用场景
库函数
(next_permutation)		 代码极简，一行搞定，不易出错。 无法在中途“踩刹车”剪枝。 小数据量、单纯要求输出全排列的题目。
DFS 回溯 极其灵活，可以随时剪枝，搜得更快。 代码较长，新手容易忘记写“回溯恢复”。 八皇后、复杂约束的旅行商问题、求最优解。

✅ 核心思想：顺位填坑，记录状态，不忘回溯。

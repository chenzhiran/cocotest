4.5 枚举的优化策略

👋 魔法师语录：

暴力枚举虽然直接，但在数据规模稍大时极其容易超时（TLE）。本节将传授给你三大核心“减负”魔法：缩小范围、剪枝 和 改变对象。
掌握这些技巧，能让不可行的枚举变为可行，把“暴力分”拿得稳稳的！

4.5.1 减少枚举范围（数学分析）

🌪️ 图解：用数学等式给循环“瘦身”

思想核心

通过数学分析确定各变量的实际上限，或利用等式关系用计算代替循环，从而将多层循环的层数或范围大幅度缩小。

示例：求 x + y + z = 100 的非负整数解个数

  • 朴素做法： 三重循环，复杂度 1013 ≈ 1.03 × 106。对于 100 尚可，但若 N = 10000，则 1012 必定超时！
  • 优化思路： 既然和是固定的，只要枚举出 x 和 y，那么 z = N - x - y。我们连 z 的循环都不需要写了，只需判断算出来的 z 是否 ≥ 0 即可。
  • 进一步缩小上限： y 的最大值不可能超过 N - x。
  • 复杂度： 降维打击，变为两重循环 O(N2)。
#include <iostream> using namespace std; int main() { int N = 100; int cnt = 0; // 优化1：只需枚举 x 和 y for (int x = 0; x <= N; ++x) { // 优化2：y 的上限受 x 限制，最大不能超过 N - x for (int y = 0; y <= N - x; ++y) { int z = N - x - y; if (z >= 0) cnt++; } } cout << "解的个数：" << cnt << endl; // 5151 return 0; }
N = 100 cnt = 0 for x in range(N + 1): for y in range(N - x + 1): # 优化：y 上限受 x 限制 z = N - x - y if z >= 0: cnt += 1 print("解的个数:", cnt) # 5151

📣 实战口诀：能算出来的，就不要枚举。 找出变量间的等式或不等式关系，用数学推导缩小循环范围。

4.5.2 剪枝 (Pruning)

✂️ 图解：果断剪掉无用的分支

思想核心

在枚举过程中，若发现当前状态已经不可能产生有效解，立即放弃该分支，不再继续深入（跳过循环或退出递归）。

示例：哥德巴赫猜想验证（偶数 = 素数 + 素数）

给定一个偶数 n，找出所有 n = p + q 且 p, q 均为素数的拆分。

  • 朴素做法： 枚举 i 从 2 到 n/2，判断 i 和 (n - i) 是否都是素数。但判断素数很耗时。
  • 剪枝优化： 如果 i 本身就不是素数，那么 n - i 根本就不需要检查了！直接 continue 跳入下一轮。
#include <iostream> #include <cmath> using namespace std; bool isPrime(int n) { if (n < 2) return false; for (int i = 2; i * i <= n; ++i) if (n % i == 0) return false; return true; } int main() { int n = 100; for (int i = 2; i <= n / 2; ++i) { if (!isPrime(i)) continue; // ✂️ 剪枝：i 不是素数则直接跳过 int j = n - i; if (isPrime(j)) { cout << n << " = " << i << " + " << j << endl; } } return 0; }
import math def is_prime(n): if n < 2: return False for i in range(2, int(math.sqrt(n)) + 1): if n % i == 0: return False return True n = 100 for i in range(2, n // 2 + 1): if not is_prime(i): continue # ✂️ 剪枝：跳过非素数 i j = n - i if is_prime(j): print(f"{n} = {i} + {j}")

✂️ 剪枝的常见形式

类型 说明 示例
可行性剪枝 当前状态已违反约束条件，不可能达到目标。 背包搜一半已超重；八皇后放下去发现已经冲突。
最优性剪枝 当前状态即使跑到底，结果也不可能比目前已知最好的解更优。 找最短路时，当前走的距离已经超过了已知的一条路。
对称性剪枝 利用图形或数据的对称性，避免做重复的搜索。 排列组合中人为规定 a1 < a2 来去重。

4.5.3 改变枚举对象（改变搜索维度）

💡 图解：熄灯问题的“顺藤摸瓜”递推法

思想核心

当直接枚举所有可能状态会导致指数爆炸时，尝试**只枚举一部分关键状态**，剩下的部分通过规则**递推或计算**唯一确定得出。这是解决“连带效应”问题的核心技巧。

经典示例：熄灯问题 (POJ 2811)

一个 5 × 6 的灯阵，按下按钮会改变自己及上下左右 5 盏灯的状态。求熄灭所有灯的按法。

  • 直接枚举： 30 个按钮按或不按，230 ≈ 109 种，必然超时。
  • 关键观察： 如果第一行的按法确定了，那么第一行那些还亮着的灯，**必须且只能通过按第二行对应的按钮来熄灭**！因为不能再去动第一行了。
  • 连锁反应： 同理，第二行定死后，第三行的按法也被唯一确定……一直递推到第五行。最后只需检查第五行是否全灭。
  • 优化结果： 从枚举 30 个格子降维到**仅枚举第一行的 6 个格子** (26 = 64 种)。瞬间完成！
#include <iostream> #include <cstring> using namespace std; int light[5][6]; // 初始灯状态 int press[5][6]; // 按钮按法 // 顺藤摸瓜：根据第一行，递推其余行 bool guess() { for (int r = 1; r < 5; ++r) { for (int c = 0; c < 6; ++c) { // 计算上方灯受到的所有影响总和 int state = light[r-1][c] + press[r-1][c]; if (r-2 >= 0) state += press[r-2][c]; if (c-1 >= 0) state += press[r-1][c-1]; if (c+1 < 6) state += press[r-1][c+1]; // 如果上方灯仍是奇数(亮)，这一行必须按 press[r][c] = (state % 2); } } // 检查最后一行是否全灭 for (int c = 0; c < 6; ++c) { int state = light[4][c] + press[4][c] + press[3][c]; if (c-1 >= 0) state += press[4][c-1]; if (c+1 < 6) state += press[4][c+1]; if (state % 2 != 0) return false; // 还有灯亮着，此方案失败 } return true; } int main() { // (此处省略读入初始 light 数组的代码) // 仅枚举第一行的 64 种按法 (状态压缩) for (int mask = 0; mask < (1 << 6); ++mask) { memset(press, 0, sizeof(press)); for (int c = 0; c < 6; ++c) press[0][c] = (mask >> c) & 1; if (guess()) { cout << "Solution found!" << endl; break; } } return 0; }
# 初始化(此处假设数据已输入) light = [[0]*6 for _ in range(5)] press = [[0]*6 for _ in range(5)] def guess(): # 递推第2~5行的按法 for r in range(1, 5): for c in range(6): state = light[r-1][c] + press[r-1][c] if r-2 >= 0: state += press[r-2][c] if c-1 >= 0: state += press[r-1][c-1] if c+1 < 6: state += press[r-1][c+1] press[r][c] = state % 2 # 检查最后一行 for c in range(6): state = light[4][c] + press[4][c] + press[3][c] if c-1 >= 0: state += press[4][c-1] if c+1 < 6: state += press[4][c+1] if state % 2 != 0: return False return True # 仅枚举第一行的 64 种状态 for mask in range(1 << 6): for r in range(5): for c in range(6): press[r][c] = 0 for c in range(6): press[0][c] = (mask >> c) & 1 if guess(): print("Solution found!") break

本节总结

优化策略 核心思想 典型应用
减少枚举范围 数学分析，利用等式用计算代替循环 百钱百鸡、方程整数解
剪枝 (Pruning) 提前判断，发现不可能的分支立刻退出 素数拆分验证、搜索与背包问题
改变枚举对象 只枚举关键的局部，其余由规则递推得出 熄灯问题、部分复杂的棋盘覆盖问题

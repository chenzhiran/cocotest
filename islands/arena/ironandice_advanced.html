一、🧊 水晶属性 (Hero & Base)

基础坐标与队伍

hero.x / hero.y：水晶中心的坐标。当你在蓝队时 X 坐标会自动镜像，代码无需额外适配。

战术解读：基于水晶的坐标进行加减（如 hero.x + 10），可以构建出始终围绕基地周围的动态防御圈，防止敌人突破盲区。

hero.color：队伍颜色："red" 或 "blue"。

baseX = hero.x baseY = hero.y hero.build("archer", baseX + 10, baseY) if hero.color == "red": hero.build("archer", "A")
auto baseX = hero.x; auto baseY = hero.y; hero.build("archer", baseX + 10, baseY); if (hero.color == "red") { hero.build("archer", "A"); }

时间与生命值

hero.time：战斗经过时间（秒），可用于时间轴策略。

战术解读：非常有用的全局计时器。你可以利用 if hero.time > 60: 在特定的游戏阶段（如后期高压波次）改变你的防守重点或更换塔的类型。

hero.health / hero.maxHealth：当前生命值与最大生命值。

战术解读：当 hero.health 较低时，可以触发“紧急避险”逻辑，例如无视冷却限制频繁释放 burst（爆炸）技能来保命。

hero.type：始终为 "base"（本关卡中）。

if hero.time > 30: hero.build("cannon", "B") if hero.health < hero.maxHealth * 0.3: hero.special("burst", hero.x + 5, hero.y)
if (hero.time > 30) { hero.build("cannon", "B"); } if (hero.health < hero.maxHealth * 0.3) { hero.special("burst", hero.x + 5, hero.y); }

二、⚡ 法力值与资源 (Mana)

hero.mana：当前法力值，上限 100。建造/升级每次固定消耗 25 点。

战术解读：法力不会因为建造失败而扣除。建议在主循环开头时刻检查 hero.mana >= 25，一旦满足就立刻寻找合适的位置投资防御塔，切忌让法力溢出导致浪费。

  • 每秒自动恢复 + 击败怪物奖励
  • 接近上限立刻使用，避免浪费
if hero.mana >= 25: hero.build("archer", "A")
if (hero.mana >= 25) { hero.build("archer", "A"); }

三、🏗️ 建造与升级 (Build & Upgrade)

建造指令

hero.build(towerType, place)：在预设点位 ("A" 至 "I") 建造。同类型再次建造 = 升级。

hero.build(towerType, x, y)：在指定坐标格子中心建造，执行成功后会**返回生成的塔对象**。

战术解读：将建造方法的返回值（塔对象）存入变量是一个极佳的习惯。这样你可以在建造完成的瞬间，立刻使用后续 API 对这座特定塔的行为进行微调（如更改它的瞄准目标）。如果法力不足或坐标非法，建造会被忽略且不消耗法力。

hero.build("archer", "A") hero.build("cannon", "D") # 再次建造同类型即可升级 hero.build("archer", "A") # 获取返回的塔对象并设置策略 myTower = hero.build("mage", 15, 40) if myTower: hero.setTargeting(myTower, "strong")
hero.build("archer", "A"); hero.build("cannon", "D"); // 再次建造同类型即可升级 hero.build("archer", "A"); // 获取返回的塔对象并设置策略 auto myTower = hero.build("mage", 15, 40); if (myTower) { hero.setTargeting(myTower, "strong"); }

合法性检查 & 塔类型列表

hero.canBuildAt(x, y)：检查格子是否可建造。

战术解读：使用循环配合 canBuildAt，可以在你的领地上自动寻找空位“铺满”防御塔，而不用硬编码记住每一个坐标。

hero.towerTypes（或 hero.availableTowerTypes）：获取所有可用防御塔类型列表。

if hero.canBuildAt(15, 40): hero.build("archer", 15, 40) types = hero.towerTypes for t in types: params = hero.getTowerParameters(t)
if (hero.canBuildAt(15, 40)) { hero.build("archer", 15, 40); } auto types = hero.towerTypes; for (auto &t : types) { auto params = hero.getTowerParameters(t); }

获取塔参数 & 设置瞄准策略

hero.getTowerParameters(towerType, level=1)：返回塔在指定等级的详细参数（attackDamage, attackRange, attackCooldown, cost 等）。

hero.setTargeting(tower, targetingType)：设置塔的目标选择策略。

战术解读：支持的策略有 "close"(优先打最近), "weak"(优先打血少,适合弓箭塔补刀收割), "strong"(优先打血多,适合集火高血量怪物), "left"/"right"(优先打偏左或偏右的敌人)。合理搭配这些策略，能让你的火力网不再产生“伤害溢出”。

四、🗼 防御塔查询与属性 (Towers Info)

全局视野

hero.findTowers()：返回所有防御塔（己方 + 敌方）。

hero.findMyTowers() / hero.findEnemyTowers()：分别返回己方和敌方塔列表。

战术解读：通过 findEnemyTowers() 分析对手最薄弱的防线方位，进而考虑用 teleport 传送技能把怪物集中扔到对方防御较弱的路线上。

塔对象属性

每个塔对象 tower 拥有以下属性：
tower.x, tower.y, tower.attackDamage, tower.attackCooldown, tower.attackRange, tower.level, tower.type

战术解读：你可以遍历自己的防御塔列表，根据 tower.level 动态决定是否要继续原地升级。例如“只要等级低于3级的弓箭塔才优先升级”，以此保证法力均匀分配，防止单一塔等级过高但边缘防线薄弱。

myTowers = hero.findMyTowers() for t in myTowers: # 筛选出等级低于3级的弓箭塔进行投资 if t.type == "archer" and t.level < 3: hero.build("archer", t.x, t.y) print(f"{t.type} Lv{t.level} 伤害{t.attackDamage}")
auto myTowers = hero.findMyTowers(); for (auto &t : myTowers) { // 筛选出等级低于3级的弓箭塔进行投资 if (t.type == "archer" && t.level < 3) { hero.build("archer", t.x, t.y); } printf("%s Lv%d 伤害%.1f\n", t.type.c_str(), t.level, t.attackDamage); }

六、🌀 技能系统 (Special Abilities)

释放技能 & 冷却检查

hero.special(specialType, x, y)：释放技能。可用字符串为 "pull", "push", "burst", "teleport"。免费，不消耗法力。

hero.isReady(specialType)：检查该技能是否已冷却完毕。

战术解读：这四个技能有相互独立的冷却时间（Cooldown），不要只用一个！要在不同怪物群中使用不同的技能形成组合技。

if hero.isReady("pull"): hero.special("pull", 30, 40) elif hero.isReady("burst"): hero.special("burst", hero.x + 5, hero.y)
if (hero.isReady("pull")) { hero.special("pull", 30, 40); } else if (hero.isReady("burst")) { hero.special("burst", hero.x + 5, hero.y); }

查看活跃技能区域

hero.findActiveAnomalies()：返回当前所有活跃的技能区域对象。包含字段：name(技能名), force(推拉强度), x, y, endX, endY, duration(剩余持续时间)。

战术解读：进阶指挥官必修课。通过查询活跃的魔法异常区域，你可以知道哪里正在释放 "pull" 吸收怪物，从而避免在同一个坐标重复释放其他冲突的技能浪费冷却时间，或者等该区域结束后再跟进下一个连招。

七、👾 怪物情报 (Monsters)

类型与参数

hero.monsterTypes：所有怪物类型列表。

hero.getMonsterParameters(monsterType)：返回怪物详细参数：attackDamage, attackRange, maxSpeed, mana, flying。

战术解读：特别注意 mana 这个参数，它代表你击杀这只怪物后会获得的奖励法力。当遇到高回报的“财神”怪物时，可以调整防线策略（比如放慢它的移速）优先吃掉它。

波次信息与实时查找

hero.getWaves() / hero.getWave(n) / hero.getWaveNumber()：获取全部波次、第 n 波怪物列表、当前波次编号（0 起始）。

hero.getCurrentWave() / hero.getNextWave()：实时获取当前波次与下一波怪物列表。

战术解读：通过 hero.getNextWave() 提前预判。如果你发现下一波里有大量带 flying: true 属性的怪，就可以提前预留法力，在怪群出现前将多座法师塔（防空主力）建造完毕，实现防守节奏的无缝衔接。

hero.getBosses()：返回 Boss 信息二维数组 [名称, 出现时间]。

hero.findMonsters()：返回当前场上所有存活怪物对象。

# 提前针对飞行怪物部署防空塔 nextWave = hero.getNextWave() for mtype in nextWave: if hero.getMonsterParameters(mtype).flying: hero.build("mage", "C") break monsters = hero.findMonsters() for m in monsters: print(m.type, "位置:", m.x, m.y)
// 提前针对飞行怪物部署防空塔 auto nextWave = hero.getNextWave(); for (auto &mtype : nextWave) { if (hero.getMonsterParameters(mtype).flying) { hero.build("mage", "C"); break; } } auto monsters = hero.findMonsters(); for (auto &m : monsters) { printf("%s 位置: %.1f, %.1f\n", m.type.c_str(), m.x, m.y); }

八/九/十、📏 对手与工具方法

🔴 洞察敌情 (Enemy Base)

hero.getEnemyHero()：获取对手水晶引用，可访问其 x, y, health, maxHealth。

📏 距离计算 (Distance & Nearest)

hero.distanceTo(target)：返回你的水晶到指定目标（塔、怪物）的距离。

hero.findNearest(targets)：从对象数组中，返回距离你水晶最近的那个目标。

战术解读：极为强大的筛选器。直接传入整个 hero.findMonsters() 数组，它会自动计算并返回离你老家最近的危险分子。利用这个返回值，你可以极其精准地将 burst（爆炸）或 teleport（传送）释放在威胁最大的目标头上。

⏸️ 节奏控制 (Wait)

hero.wait(t)：等待 t 秒，期间不执行代码的后续逻辑。

战术解读：谨慎使用！在竞速赛中，挂起等待意味着你无法在这一段时间内建造或释放技能。建议使用基于 hero.time 的条件判断来代替长时的阻塞等待。

# 找到离基地最近的怪，如果距离小于 15 则果断弹飞 nearest = hero.findNearest(hero.findMonsters()) if nearest: if hero.distanceTo(nearest) < 15: hero.special("burst", nearest.x, nearest.y)
// 找到离基地最近的怪，如果距离小于 15 则果断弹飞 auto nearest = hero.findNearest(hero.findMonsters()); if (nearest) { if (hero.distanceTo(nearest) < 15) { hero.special("burst", nearest.x, nearest.y); } }

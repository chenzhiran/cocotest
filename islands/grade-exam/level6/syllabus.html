倒立的森林 树与二叉树

🌳

什么是“树”？

在计算机里，树是倒着长的。
根 (Root) 在最上面，通过树枝 (Edge) 向下连接叶子 (Leaf)。 它没有回路，层级分明，就像家族族谱或者电脑的文件夹结构。

🤖 完全二叉树

强迫症之树。
除了最后一层，其他层都长满了。最后一层的叶子也都靠左边排得整整齐齐。

⚖️ 二叉排序树 (BST)

守规矩之树。
对于每个节点：左孩子 < 爸爸 < 右孩子。
找数据超级快，也就是二叉树搜索算法的基础。

🍇 哈夫曼树

最优二叉树。
权值大的叶子离根最近。带权路径长度(WPL)最短。常用于压缩数据。

🚶 树的遍历

前序：根->左->右
中序：左->根->右
后序：左->右->根

🤐 哈夫曼编码 (Huffman Coding)

让常用的字（比如 'e', 'a'）编码短，不常用的字编码长。
基于哈夫曼树生成，是无损压缩的鼻祖（ZIP文件的核心思想）。

A:0, B:10, C:110...

🌒 格雷编码 (Gray Code)

相邻的两个数，二进制表示只有一位不同。
用于减少数字电路的错误。

普通: 00 -> 01 -> 10 (变2位)
格雷: 00 -> 01 -> 11 (变1位)

迷宫探险 搜索算法

🕵️ DFS 深度优先搜索

不撞南墙不回头

策略： 只要有路就一直往深处走，直到走不通了再回退（回溯）。
工具： 栈 (Stack) 或 递归。

[ 一条路走到黑的探险者 ]

📡 BFS 宽度优先搜索

地毯式搜索

策略： 先把周围的一圈找完，再找外一圈。像水波纹一样扩散。
工具： 队列 (Queue)。

[ 层层扫描的雷达 ]

城市基建 栈与队列

🥞 栈 (Stack)

特点：先进后出 (FILO)

就像洗盘子，最后放上去的盘子，最先被拿走。或者像弹夹。

🚇 队列 (Queue)

特点：先进先出 (FIFO)

就像在食堂排队打饭，先来的人先吃。

🎡 循环队列

特点：首尾相接

为了节省空间，把队列这一长条弯成一个圆环。内存不浪费！

预言家笔记 简单动态规划 (DP)

Dynamic Programming

DP 不是一种算法，而是一种思想。
它的核心是：不要重复计算已经算过的问题。
如果我们记住了“过去”，就能更聪明地规划“未来”。

🪙 0/1 背包问题

小偷有一个能装 W 公斤的背包。面前有 N 个宝物，每个有重量和价值。
策略： 对于每个宝物，只有两种选择：拿 (1) 还是 不拿 (0)。
目标：在不撑破背包的情况下，价值最大化。

📏 一维 DP (爬楼梯)

每次能爬 1 阶或 2 阶。爬到第 N 阶有多少种方法？
公式： dp[i] = dp[i-1] + dp[i-2]
想知道现在的状态，只需要回头看前两步。

创世蓝图 面向对象 (OOP)

🏗️

类 (Class) vs 对象 (Object)

面向对象 让你像上帝一样编程。

  • Class
    类是蓝图/模具。
    例如：“狗”这个概念。它有属性（毛色、名字）和方法（叫、跑）。
  • Object
    对象是实物。
    例如：你家里那只叫“旺财”的具体的小狗。
